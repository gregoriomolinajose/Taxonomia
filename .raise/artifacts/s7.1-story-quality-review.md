### Observations (no action needed)
- **Error Handling (JSON Fallbacks):** En `FormEngine_UI.html` línea ~630, se implementó un bloque de contención `try/catch` para la hidratación asíncrona de variables relacionales: 
  ```javascript
  try {
      initialValues = Array.isArray(data[field.name]) ? data[field.name] : JSON.parse(data[field.name]);
  } catch(e) { initialValues = (typeof data[field.name] === 'string') ? [data[field.name]] : []; }
  ```
  Esto sobrevive a strings planos, arrays válidos o JSON dañados enviados por el Endpoint, garantizando *Fault Tolerance* (Regla de Oro 5.3).
- **Type Honesty (Coerción Idempotente):** Las llaves foráneas (`id_dominio`) provenientes del caché local pueden ser tratadas como Números por V8 si la sintaxis del ID en la base de datos es puramente numérica. La función helper `_normalizeId(e.id_nodo_hijo)` inyectada por el motor asegura coerción a String uniforme antes de los mapeos `.filter()` de las Aristas (Edges). Previno un bug de truthiness de JS puro `123 === '123' (false)`.
- **Garbage Memory Check:** El componente in-line modal (`_buildSearchableMulti`) recrea su DOM (`createElement`) al vuelo y bindea closures aislados a cada input field usando sus copias profundas, con lo cual no interfiere en el global scope y escapa del *Memory Stale Contamination* al momento de reciclar el formulario.
