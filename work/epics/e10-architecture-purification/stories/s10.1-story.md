# User Story
**Como** Desarrollador Principal Avanzado
**Quiero** encapsular las utilerías dispersas de interfaz gráfica bajo un módulo estructurado estricto (`window.UI_Factory`) e inyectar Type Guards
**Para** eliminar las variables globales sueltas, prevenir corrupciones asíncronas de memoria, e impulsar la base de Vanilla JS hacia un isomorfismo apto para Testing Automatizado.

# Acceptance Criteria
### Scenario 1: Erradicación del Global Namespace V8
**Given** que utilerías UI existen salvajemente en `window` (`window._populateSelectOptions`, etc.)
**When** un usuario compila y ejecuta el Frontend MDM
**Then** ningún método procedimental flotará huérfano; todos estarán supeditados a `window.UI_Factory` o un controlador semejante. 

### Scenario 2: Defensividad en Métodos Públicos
**Given** el nuevo Namespace de control Visual
**When** métodos de inyección de opciones (`populateSelectOptions`) sean atacados o expuestos a fallos pasivos que les envíen parámetros `null` o `undefined` en variables iterables
**Then** el validador estricto interno tipo `Array.isArray()` desestimará la operación en O(1) previniendo los desbordes de `TypeError` en el navegador.
