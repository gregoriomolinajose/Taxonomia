# User Story
**Como** Arquitecto de UI
**Quiero** purgar los estilos en línea emulados del DOM y optimizar el token parser (`flattenTokens`)
**Para** asegurar que la vista de datos (DataView) consuma estéticamente sus variables del sistema global de temas de forma agnóstica, eliminando dependencias quemadas y aligerando la hidratación asíncrona de la Taxonomía MDM.

# Acceptance Criteria
### Scenario 1: Unificación del Shadow-DOM
**Given** que `DataView_UI.html` contiene 660 líneas de reglas CSS estáticas localizadas (`<style>`)
**When** un usuario despliega la aplicación y los Partial HTML son inyectados
**Then** todo el CSS será pre-descargado exclusivamente desde `CSS_App` (Single Source of Truth), extirpando por completo la etiqueta `<style>` local del módulo vista.

### Scenario 2: Supresión de Fallos en Renderizado JSON
**Given** la recursión asíncrona `flattenTokens` que convierte JSON Theme Tokens a CSS
**When** el parseador procesa la condicional `if (key === 'default')`
**Then** abortará el resto de la subrutina (`return`) para prevenir que las mismas variables de color nativas se impriman duplicadas bajo un prefijo falso (`--text-default`), ahorrando re-cálculos del motor Chromium.
