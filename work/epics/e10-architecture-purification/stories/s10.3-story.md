# User Story
**Como** QA / Arquitecto de Rendimiento
**Quiero** disponer de una suite asíncrona de Profiling E2E que abra y destruya masivamente modales MDM anidados en la vista Cliente.
**Para** someter a estrés el recolector de basura (Garbage Collection) de V8/Chromium, y descartar categóricamente la existencia de fugas de memoria (Memory Leaks) causadas por Event Listeners huérfanos y nodos Ionic residuales.

# Acceptance Criteria
### Scenario 1: E2E Profiler Utility
**Given** un navegador Chrome ejecutando el SPA Taxonomía
**When** el equipo de ingeniería invoque `window.__runMemoryProfile_E2E(50)` desde la consola
**Then** un script automatizado inyectará secuencialmente 50 renders de `FormEngine` iterativos e interrogará el DOM Tree para comprobar que tras lanzar los métodos `dismiss()`, no quede ni un solo elemento huerfano tipo `ion-select-option` suspendido en la memoria RAM tras un delay razonable.
