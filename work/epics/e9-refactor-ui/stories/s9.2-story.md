# User Story
**Como** Arquitecto Frontend MDM
**Quiero** desacoplar el Modal Stack del objeto global \`window\` y asignarle un nivel máximo de profundidad
**Para** proteger la memoria del navegador durante la creación in-line recursiva (Garbage Collection) y estabilizar la UI sin desbordes de Z-Index.

# Acceptance Criteria
### Scenario 1: Abstracción de Estado LIFO
**Given** que FormEngine_UI necesita apilar componentes visuales (Modales)
**When** el usuario hace click en generar nuevo registro
**Then** el modal se empuja a una clase \`ModalStackController\` inyectada o encapsulada, y no al objeto arcaico \`window.formModalStack\`.
**And** las peticiones de cerrado (\`pop()+\` dismiss) están garantizadas contra referencias nulas.

### Scenario 2: Máxima Profundidad (Max Depth Z-Index Guard)
**Given** un usuario que inicia una cadena recursiva ("Crear padre de padre de padre...")
**When** un modal intenta montarse pero el LIFO ya tiene 3 o más elementos activos
**Then** se activa un \`ion-alert\` interceptor recomendando terminar el flujo actual
**And** la creación del 4to form se detiene matemáticamente previniendo Memory/DOM leaks.
