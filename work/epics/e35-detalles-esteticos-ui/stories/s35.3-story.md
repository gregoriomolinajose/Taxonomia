# Story S35.3: Cobertura Automatizada UI/Back para Eventos Reactivos

## User Story (Connextra)

**Como** desarrollador de infraestructura UI
**Quiero** contar con un entorno de pruebas automatizadas (Tests) para el componente `SearchableSingle` y el orquestador `RelationBuilder`
**Para** garantizar que la fuga de memoria H10, los eventos `ionBlur` móviles y las asincronías de EventBus (DATASTORE::CHANGED, FormEngine::RecordHydrated) no sufran regresiones en futuras manipulaciones de código.

## Acceptance Criteria (Gherkin)

**Scenario 1: Auto-hidratación vía EventBus asíncrono**
**Given** que el componente RelationBuilder se carga vacío y una recarga global es desencadenada
**When** el EventBus lanza `DATASTORE::CHANGED`
**Then** el relationBuilder invoca `updateConfig` y repuebla internamente sus opciones con el dataset más reciente.

**Scenario 2: Limpieza de RAM e Inmunidad a Fugas**
**Given** un componente SearchableSingle montado en el DOM
**When** el ciclo de vida del *Drawer* descarta (remove) el contenedor desde documento
**Then** las suscripciones de `AppEventBus` son debidamente desmanteladas (unsubscribed) impidiendo fugas H10.

**Scenario 3: Excepciones táctiles en UI Móvil**
**Given** un navegador emulado con tamaño de Mobile (`innerWidth <= 768px`)
**When** un blur ocurre en el parent box (input principal) y el Modal toma control
**Then** la validación estricta de blur NO cierra la persiana preventivamente.
