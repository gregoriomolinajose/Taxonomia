# Scope: S35.3 Pruebas Automatizadas: Reactividad y Componentes UI

## In Scope
- Creación de un Runner y Specs en el Suite de Pruebas de AppScript (`TEST_Suite.gs`, o equivalente) para emular el flujo PWA en el Backend si aplica (comprobación de integridad del payload).
- Configuración de Specs en Javascript para verificar las inyecciones táctiles de `UI_Component_SearchableSingle`.
- Pruebas que emiten `DATASTORE::CHANGED` en `AppEventBus` simulado para probar la recarga en caliente de `RelationBuilder`.

## Out of Scope
- Escribir pruebas para cualquier componente del formulario ajeno a SearchableSingle/RelationBuilder.
- Alterar la lógica existente del sistema en producción (la historia se enfoca en puro Testing, el refactor ya ocurrió).

## Done Criteria
- [ ] Test Suite configurado validando las clausulas de `ionBlur` condicionado.
- [ ] Test Suite asegurando el registro de *unsubscribe* (prevención de Zombie Leaks).
- [ ] Test Suite validando la hidratación asíncrona.
- [ ] Todo el código de pruebas compila sin errores a través de la tubería de Clasp / GAS.
