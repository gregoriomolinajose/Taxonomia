# S38.3: Extracción Segura (Reader Pipeline) de la matriz de Google Sheets al Frontend

## User Story
Como Arquitecto de Taxonomía
Quiero que el Motor ETL backend extraiga los datos brutos desde la URL de la Plantilla de Drive
Para tener los registros disponibles en el Frontend Javascript listos for validación masiva y carga asíncrona (Chunking).

## Acceptance Criteria
- Se registra un nuevo method `extractDataFromDrive(entityName, sheetUrlOrId)` en `Engine_ETL.gs`.
- El método se expone al Frontend a través de `API_Universal.gs` mediante una nueva accion `etl_extract_sheet_data`.
- El Lector obtiene la matriz 2D de `SpreadsheetApp`, remueve la cabecera, y devuelve las filas como Objetos `[ {campo: "val"}, ... ]`.
- Omite celdas totalmente vacías o rows en blanco para no ensuciar la base de datos.
- Responde nativamente al botón "3. Ejecutar Extracción Híbrida" de la UI.
