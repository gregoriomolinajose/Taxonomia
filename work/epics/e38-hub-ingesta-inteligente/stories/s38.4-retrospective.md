# S38.4: Tolerancia e Ignorado de Inconsistencias - Retrospective

## Context
- **Epic**: E38 - Hub de Ingesta Inteligente
- **Story**: S38.4
- **Completed**: 2026-04-15

## What Went Well
- **Abstracción DRY (Don't Repeat Yourself):** Logramos centralizar el "Chunker" asincrónico que fragmenta sábanas de datos extrayéndolo a un sub-método `_dispatchChunks` protegido y reciclándolo tanto para la carga de CSVs nativos (`processFile`) como para arreglos cruzados desde Google Sheets (`processPayload`).
- **Seguridad y Calidad de Código:** Pasamos limpiamente la puerta de Calidad mitigando cualquier riesgo de Memory Leaks, ya que evitamos almacenar ciegamente los 800+ registros en Memoria RAM para simplemente empujarlos gradualmente y desecharlos con un `DataStore.set(..., null)` post-inserción.
- **Sanitización Transaccional:** Se suprimió la amenaza de inyecciones de datos "temporales" (fechas robadas) logrando ignorar iterativamente todo llaveo con prefijos técnicos `sys_`.

## What Could Be Improved
- **Alertas Vivas (UX):** El componente de "Loading" en Ionic reacciona muy rápido y aunque logramos un puente natural, en lotes rápidos puede desaparecer de la pantalla repentinamente. Una pequeña demora visual de 500ms al final mejoraría su percepción visual de robustez para el usuario promedio (Eye-Candy feedback).

## Learnings (Knowledge Graph updates)
- **Tolerancia API Frontend-Backend:** En Apps Script, enviar un array de 1000 diccionarios a través de la canalización IPC `google.script.run` lanza excepción general nativa. Fraccionar arreglos de `Math.ceil(data/50)` resulta un bloque conservador idílico para librarse del límite "Maximum Payload Size Exceeded" en los servidores de Google Workspace.
