# S38.4: Tolerancia e Ignorado de Inconsistencias (Chunker Frontend) - Plan

## Overview
- **Story ID:** S38.4
- **Size:** M

## Tasks

1. **Refactorizar lógica central del Chunker (`DataEngine_ETL.client.js`)**
   - Extraer el loop asincrónico a un método `_dispatchChunks(payload, entityName, progressCallback)`.
   - Modificar `processFile` para que llame a este despachador puro.

2. **Crear Extractor Tolerante `processPayload` (`DataEngine_ETL.client.js`)**
   - Recibir el Array nativo proveniente de `window._etlPayloadCache` / Drive.
   - Iterar llaves ignorando silenciosamente aquellas que posean prefijo `sys_`.
   - Pasar el mapa sanitizado a `_dispatchChunks`.

3. **Inyectar Flujo Reactivo Completo (`DataView_UI.client.js`)**
   - Enlazar la respuesta asíncrona validada de `onDriveSync` a la función `processPayload`.
   - Invocar el renderizador Modal (indicador de Progreso del UI de Extracción).
   - Engatillar regeneración con `DataStore.set(..., null)` post-operación y repintar DOM.

## Execution Order
Task 1 y 2 van de la mano. Terminan en Task 3 para amarrar la interfaz cliente.
