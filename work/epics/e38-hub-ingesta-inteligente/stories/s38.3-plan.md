# S38.3: Hub ETL Extractor Híbrido - Plan

## Overview
- **Story ID:** S38.3
- **Size:** M

## Tasks

1. **Definir `extractDataFromDrive` en Backend (`Engine_ETL.gs`)**
   - Implementar el lector que acepta ID o URL completa.
   - Extraer `SpreadsheetApp.openById().getSheets()[0].getDataRange().getValues()`.
   - Iterar, aplicar el mapeo de Headers (fila 0) a diccionario, descartando las filas vacías estériles.

2. **Exponer el Hub a `API_Universal_Router` (`API_Universal.gs`)**
   - Agarrar la acción `etl_extract_sheet_data`.
   - Llamar al `_guardAbac('read', entityName, null)` para prevenir robos laterales de bases de datos.
   - Ejecutar la Extracción y mandar el JSON de Data al fronend de manera stringificada `success`.

3. **Inyectar Receptores Reactivos en el cliente (`DataView_UI.client.js`)**
   - Enlazar la interfaz Callback `onDriveSync` al Request Router `DataAPI.call()`.
   - Incluir Loader asincrónico para UX.
   - Depositar la sabana final en memoria global (`window._etlPayloadCache`) y mandar un Toast verde informando éxito.

## Execution Order
Lineal: Task 1 -> Task 2 -> Task 3. No hay dependencias recíprocas bloqueantes.

## Risks & Mitigations
- **Riesgo:** Un usuario pega un Doc/URL de Drive que no es Gsheets, arrojando excepción de Apps Script fatal.
- **Mitigación:** Asegurar un bloque try-catch limpio que devuelva en DataAPI un error amigable "El archivo introducido es inaccesible o no es una Hoja de Cálculo."
