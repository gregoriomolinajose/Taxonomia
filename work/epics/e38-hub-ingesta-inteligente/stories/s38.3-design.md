# S38.3: Hub ETL Extractor Híbrido - Design

## Problem & Value
- **Problem**: La interfaz actual puede generar planillas vacías, pero no tiene un mecanismo de subida "cruzada" donde lea dinámicamente un Documento en la nube proveído por URL y extraiga sus registros asíncronamente simulando un CSV en memoria.
- **Value**: Permite a los usuarios llenar largas tablas de configuración colaborativamente en Excel/Drive y luego ingestar todo masivamente sin necesidad de descargar a disco y subir ("Zero Touch Extract").

## Approach
Implementaremos un extractor pasivo (`extractDataFromDrive`) en `Engine_ETL.gs`. No habrá mutaciones a base de datos todavía. La función simplemente actúa como un parseador de la matriz de Google Drive hacia Arreglos de Objetos clave-valor basándose en los Headers, y lo dispara de vuelta.
- **Components**: `Engine_ETL.gs` (nuevo logic chunk), `API_Universal.gs` (nuevo pathing ABAC), `DataView_UI.client.js` (Trigger UI y Toast handling).

## Examples
**Carga útil URL**: `https://docs.google.com/spreadsheets/d/1U4x_M9.../edit`
**Response Esperado del Extractor**:
```json
{
  "status": "success",
  "data": [
    { "nombre": "Fulano", "email": "f@f.com" },
    { "nombre": "Mengano", "email": "m@f.com" }
  ]
}
```

## Acceptance Criteria
- [x] **MUST**: `extractDataFromDrive` extrae limpiamente valores descartando filas muertas.
- [x] **MUST**: El Frontend procesa el resultado e imprime la longitud en un Toast y asigna a memoria global los objetos.
- [x] **MUST NOT**: Romper esquemas existentes o guardar a la base de datos (Esto sólo lee).
