# Story S18.2: Rule Evaluator Engine

## User Story
Como Interceptor Transaccional (Middleware)
Deseo evaluar toda petición de escritura o lectura contra las directivas ABAC embebidas en los `APP_SCHEMAS`
Para garantizar que nadie mute entidades bajo la jerarquía de un nodo del cual no sea "Dueño" o un nodo de contexto protegido, aislando la lógica de seguridad del flujo tradicional MVC.

## Acceptance Criteria
**Scenario: Mutación Ilegal Bloqueada**
`Given` un usuario con rol "Analista" que intenta invocar `Engine_DB.saveRecord("Equipo", payload)`
`And` su `___ABAC_CONTEXT___` demuestra que no es el "Owner" del Equipo X.
`And` el App Schema de "Equipo" define explícitamente `abacRules: { edit: 'ownerOnly' }`.
`When` el Controller recibe la petición.
`Then` la operación se detiene abortando el ciclo de vida de la Base de datos y retornando una Excepción Transaccional limpia `403 Forbidden` al frontend.

**Scenario: Regla Opcional Bypass**
`Given` un schema donde `abacRules` no está definido en absoluto o está marcado como `all`.
`When` el Engine evalúa la transacción.
`Then` se omite el salto y el flujo original de I/O de la DB sigue normalmente.
