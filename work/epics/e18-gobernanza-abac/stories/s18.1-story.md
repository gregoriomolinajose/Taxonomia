# Story S18.1: Session Topology Resolver

## User Story
Como Motor de Gobernanza ABAC
Deseo calcular el árbol topológico exacto de un usuario al inicio de su sesión
Para que el sistema de Frontend y Backend puedan filtrar peticiones y vistas con latencia cero usando la matriz de pertenencia en tiempo real.

## Acceptance Criteria (Gherkin)
**Scenario: Un usuario estándar inicia sesión**
`Given` que el empleado "A" figura como "Scrum Master" del equipo "X".
`When` el subsistema de Auth resuelve la identidad del Session Token.
`Then` se genera un payload `___ABAC_CONTEXT___` que incluye exclusivamente el ID del equipo "X" y los nodos inferiores adjuntos bajo "Owner".

**Scenario: Delegación de Rol (RTE)**
`Given` que el empleado "B" es el "Release Train Engineer" del Tren "Y".
`When` se evalúa su topología de sesión.
`Then` el arreglo de "Owner" incluye todos los Equipos debajo de ese Tren "Y", garantizando que posea permisos indirectos por herencia (Se completará lógicamente en S18.3, pero la estructura de datos matriz deberá poder guardarlo aquí).
