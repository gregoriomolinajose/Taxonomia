# Scope S18.4: Contextual UI Renderer (Frontend)

## Objective
Conectar las librerías Frontend (particularmente `UI_DataGrid` y las vistas modales) con la bóveda criptográfica del `__ABAC_CONTEXT__`, impidiendo la renderización visual de directivas y verbos sin permiso activo. También adaptar el motor ABAC en backend para inyectar un diccionario matriz de permisos al Cliente para evaluaciones front-side.

## In Scope
- Adaptación de `Engine_ABAC.gs` para retornar también la matriz `permissions: { Entidad: 'NIVEL_ACCESO' }` además de los topologicos.
- Lógica interceptora en el frontend: `UI_DataGrid.renderRowActions()`.
- Lógica de Hidding de botones de acción masiva de creacion.

## Out of Scope
- Backend enforcement CUD (Ya tratado exitosamente en S18.2).
- Creación de perfiles Custom de permisos desde GUI (Se posterga S18.5).

## Definition of Done
- Los botones de editar desaparecen en Registros donde el usuario es solo Read-Only.
- Las pruebas manuales revelan que cambiar el array del console DOM en tiempo real muta el render de la tabla.
