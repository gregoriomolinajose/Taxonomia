# Implementation Plan: S18.1 Session Topology Resolver

## Overview
- **Story**: S18.1 (Session Topology Resolver)
- **Size**: XS
- **Goal**: Inyectar de forma invisible y sin latencia extra el grafo exacto de la sesión (Qué equipos/trenes lidera o integra el usuario actual).

## Tareas (Tasks)

### Task 1: Motor Resolutor en Backend (Engine_ABAC.gs)
- **Archivos**: `[NEW] src/Engine_ABAC.gs`
- **Descripción**: Crear la función pura `Engine_ABAC.resolveTopologyFor(email)` responsable de iterar sobre el grafo cacheado y devolver `{ ownerOf: [], memberOf: [] }`.

### Task 2: Modificación del Gateway MVC (Code.js)
- **Archivos**: `[MODIFY] src/Code.js`
- **Descripción**: Invocar `Engine_ABAC` dentro de `doGet(e)` usando `Session.getActiveUser().getEmail()`, stringificar limpiamente el JSON y setearlo en `template.__ABAC_CONTEXT__`.

### Task 3: Inyección Cero-Latencia en SPA (Index.html)
- **Archivos**: `[MODIFY] src/Index.html`
- **Descripción**: Agarrar la variable cruda inyectada por el servidor `<?= __ABAC_CONTEXT__ ?>` incrustándola nativamente en una etiqueta script protegida `window.__ABAC_CONTEXT__ = JSON.parse(...)`.

### Task 4: Integración y Verificación (Manual)
- **Archivos**: Ninguno
- **Descripción**: Correr un E2E test validando en `devTools > Console` que la variable exista.
