# Scope S18.2: Rule Evaluator Engine (Middleware)

## Objective
Desarrollar la capa Interceptora (`Guard`) dentro del Controlador backend que blinde de forma irreversible los permisos lógicos de mutación antes de tocar el CRUD de Base de Datos.

## In Scope
- Adaptar `Action.gs` (Controller principal) u otra función de Gateway para pasar peticiones por el `Engine_ABAC.validate(...)`.
- Creación de un sub-motor asíncrono o síncrono encargado de discernir verbos de acción ('edit', 'create', 'delete').
- Modificación del Diccionario `Schema_Engine.gs` (O localmente) simulando campos de configuración `topologyRules`.

## Out of Scope
- Re-renderización de la Interfaz Visual `UI_DataGrid` para ocultar opciones. (Propio de S18.4).
- Ascenso de rangos matemáticos para PMs/RTEs. (Propio de S18.3).

## Definition of Done
- Se comprueba tirando transacciones "sucias" por la consola que el Firewall aborta automáticamente intentos ilegales contra Entidades Reguladas, usando el Payload S18.1 como llave.
