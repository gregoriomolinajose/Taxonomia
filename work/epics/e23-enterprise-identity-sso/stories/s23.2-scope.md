# Story 23.2 Scope

## Objective
Registrar y proveer la vista nativa del panel "Config_Workspace" para inyectar credenciales u IDs estáticos al sistema de seguridad (Identity SSO backend de la S23.1).

## In Scope
- Definición de "Config_Workspace" en `Schema_Engine.gs` (APP_SCHEMAS).
- Campos mínimos necesarios: Identificador Único, Nombre Dominio (Domain Name), Contacto Admin y Status (Activo/Inactivo).
- Inclusión temporal de un `oauth_mode` si se prevé que el inquilino exija Service Accounts exclusivas o si usa delegación global.
- Rol restrictivo a SYS_ADMIN.

## Out of Scope
- Conectar lógicamente el Multi-Directorio OAuth *simultáneamente* a la S23.1. Ese paso integrador puede venir más adelante. El foco de esta historia es el panel UI (FormEngine/DataView CRUD).

## Done Criteria
- Puedo abrir un panel `DataView` mostrando `Config_Workspace`.
- Un administrador puede crear un registro con la estructura del Workspace.
