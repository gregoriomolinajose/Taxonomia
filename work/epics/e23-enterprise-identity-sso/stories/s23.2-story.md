# Story 23.2: Multi-Workspace Configurations Panel UI

## Hypothesis
Si estructuramos los dominios de Workspaces mediante una tabla de metadatos gestionada por el FormEngine (usando `APP_SCHEMAS.Config_Workspace`), podremos permitir conexiones de Identidad Dinámicas sin hardcodear tokens ni domains en código, posibilitando a un Admnistrador dar de alta múltiples inquilinos.

## Description
Crear el modelo `Config_Workspace` en el hub de esquemas base. Configurar permisos de lectura global para Identity Resolvers y permisos CRUD solo para Administradores, utilizando el ecosistema nativo de Taxonomía.

## Acceptance Criteria
- Creada Entidad `Config_Workspace` en `Schema_Engine.gs` respetando la integridad de esquemas.
- DataView renderiza la tabla sin errores.
- Limitación: Solo los SYS_ADMIN o permisos superiores pueden ver el nodo de Workspaces en el dashboard de Settings.
