# Story 23.1: Admin SDK Connector & Avatar Hydration Cache

## Hypothesis
If we leverage the Google Workspace Admin Directory API, we can programmatically resolve visual details (profile photos / avatars) for any given email address associated with the domain, replacing generic initials with real avatar hydration and thus establishing proper identity flow.

## Description
This story introduces the backend wrapper needed to query the `AdminDirectory`. It also implements an in-memory client-side Map to cache responses, ensuring that the system scales without exceeding the Admin SDK quotas.

## Acceptance Criteria
- Given an email address, the backend adapter securely fetches the photoUrl via `AdminDirectory.Users.photos.get` (or equivalent fallback).
- Given the client application fetching users, it caches the resolved photoURLs and does not trigger duplicate network fetches across re-renders.
