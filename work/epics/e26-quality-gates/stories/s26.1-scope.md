# Scope: Lint Purification 

## In Scope
- Refactorización de todos los colores quemados en forma raw dentro de `CSS_App.html` hacia variables CSS paramétricas.
- Refactorización de colores quemados en `src/assets/css/data-view.css` hacia las dependencias estáticas de variables de color de Ionic o Taxonomía (`var(--dv-...)`).
- Substitución de `window.DataAPI.call` duro en `UI_FormSubmitter.client.js:175` por un Handler/Servicio inyectable y provisto al componente en tiempo de registro o a través del `AppContext`.

## Out of Scope
- Implementar nuevas reglas de Lint limitantes de CSS.
- Arreglar o mover las variables semánticas generales que ya existen o crear nuevas de alto nivel (Solo se limitará a apegar el sistema a lo que ya hay).
- Cambiar la arquitectura base de API del backend.

## Done Criteria
1. Se levanta la instancia completa y la UI se pinta de forma completamente consistente pre/post corrección del Lint CSS en Light y Dark Mode.
2. `npm run lint:arch` y `npm run lint:css` corren con EXIT CODE 0 sin excepciones.
3. Se puede hacer sumisión de validadores y el formulario es capaz exitosamente de guardar o realizar transacciones, validando que el cambio en arquitectura al `UI_FormSubmitter` funciona igual que cuando llamaba a `DataAPI` directamente.
