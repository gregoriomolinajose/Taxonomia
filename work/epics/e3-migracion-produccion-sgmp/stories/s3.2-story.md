# S3.2: Pathing Visual e Hidratación de UI en Edición

**As a** System Architect
**I want** to auto-generate the `path_completo_es` string dynamically and hydrate relational dropdowns correctly during edit state initialization
**So that** the Tree Lock mechanism visualizes the parent hierarchy properly without causing latency, and users get real-time feedback of the generated UI path.

### Acceptance Criteria
- `path_completo_es` is read-only and uses `getCalculatedPathEs` logic (e.g. `PATH: Padre > Hijo`).
- Typing into the domain name input (`n0_es`) automatically triggers recalculation for the path in real-time.
- `openEditForm` pre-fills `<ion-select>` options dynamically using local resolvers *before* clamping with `disabled = true`, preventing blank fields in Edit Mode.
- Operates at zero latency without hitting Server-Side Google Apps Script endpoints.
- Version bumped to `1.0.86`.
