# S3.2 — Fix GAS Entry Point & Deployment Routing

## User Story (Connextra)

**Como** desarrollador del proyecto Taxonomia,
**quiero** que la URL pública del Web App de GAS sirva siempre el `Index.html` más reciente con los módulos CSS/JS inyectados,
**para que** los cambios del Epic E3 sean visibles en producción inmediatamente después de cada deploy.

## Gherkin Acceptance Criteria

```gherkin
Feature: GAS Web App Entry Point Routing

  Scenario: El Web App URL sirve el Index.html actualizado
    Given que se ejecutó `npm run deploy:dev` exitosamente
    When el usuario abre la URL del Web App en el navegador
    Then la interfaz muestra el Build Tracker con la versión `v1.0.39`
    And el sidebar carga con los estilos de CSS_DesignSystem y CSS_App

  Scenario: Los includes GAS se resuelven correctamente
    Given que Index.html contiene `<?!= include('CSS_DesignSystem'); ?>`
    When GAS ejecuta `template.evaluate()`
    Then el HTML renderizado incluye el bloque <style> de CSS_DesignSystem
    And no hay errores de "include not found" en los logs de GAS
```

## Root Cause Diagnosis

`doGet()` ya apunta correctamente a `'Index'`. El problema es que `clasp push`
actualiza el código fuente en GAS pero **NO actualiza el URL estático del Web App**.
Para que el URL público sirva la nueva versión, se requiere `clasp deploy --deploymentId {id}`
o crear un nuevo deployment desde la consola de GAS.

## Done Criteria

- [ ] `clasp deploy` ejecutado y nuevo deployment ID registrado.
- [ ] URL del Web App actualizada con v1.0.39.
- [ ] Build Tracker visible con versión correcta.
- [ ] No errores en consola del navegador relacionados con include().
