# Story Design: S11.1 Factory Component Blueprint

## 1. Frame What & Why
**Problem:** `FormEngine_UI.html` depends on 1700 lines of procedural DOM injection (`createElement`) which slows Chromium "First Contentful Paint" limit and mixes UI structure with reactive behavioral logic.
**Value:** By abstracting rendering to string-templates, we achieve O(1) insertion latency and achieve true "Declarative UI", setting the foundation for the next MDM phase while making code readable again.

## 2. Approach
Erradicar procedural iteration en `global.renderForm`.
1. Migrar el bloque `fields.forEach(...)` a un `map().join('')` invocando `window.UI_Factory.buildFieldTemplate(field)`.
2. Las estructuras "complejas" (`searchable_multi`, `chip_input`) se pintarán como contenedores HTML crudos (`<div id="...">`) esperando ser hidratados.
3. Post-Process (`window.UI_Factory.bindEngineEvents()`): se enrutará ciegamente hacia referencias vía `querySelector` asimilando el DOM estático e incrustando detectores asíncronos (`ionChange`, validaciones MDM Subgrid).

*Missing Entity Strategy:* Todos los valores defaults o variables caídas a `undefined` serán saneados a un `String` vacío o interpolaciones resilientes para prevenir fallas `null` silenciadas en el parser (PAT-E-523).

## 3. Examples

### Generación Cruda Vanilla JS (Antes)
```javascript
const ionItem = document.createElement('ion-item');
const ionLabel = document.createElement('ion-label');
// ... setAttribute, style ...
ionItem.appendChild(ionLabel);
```

### Generación Factory Template Declarativo (Ahora)
```javascript
const valstr = data[field.name] || '';
const templateHtml = `
<ion-col size="12" size-md="${field.width || 12}">
   <ion-input id="field_${field.name}" name="${field.name}" label="${field.label}" label-placement="stacked" fill="outline" value="${valstr}"></ion-input>
</ion-col>`;
// targetRow.innerHTML += templateHtml; // Inyectado una sola vez por Row al final
```

## 4. Acceptance Criteria
- **MUST** Mapear campos simples y colecciones sin bucles `createElement()`.
- **MUST** Conservar el comportamiento funcional Topológico, aislando eventos puros en la rutina Post-Proceso.
- **MUST NO** Introducir abstracciones Framework de 3ros (React/Vue). Quedamos crudos (Vanilla + Ionic).
- **SHOULD** Reducir el core principal del render loop en un ~40%.
