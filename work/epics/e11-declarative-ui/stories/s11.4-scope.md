# Story S11.4: Automated Tooling Ecosystem

## Objective
Purgar el repositorio de producción de ruido de QA (Test Functions) y establecer un flujo de minificado CSS automatizado que aligere la descarga inicial de la PWA Taxonómica, alcanzando el estado de madurez Final (Done Criteria) para el Epic 11.

## In Scope
- Creación de `TEST_Suite_UI.html` puro para albergar todas las utilidades E2E y Profilers de memoria, abstrayéndolas del FormEngine.
- Inyección condicional / controlada del Módulo QA en el `Index.html` SPA.
- Instalación de un paso asíncrono en `deploy.js` que utilice una carpeta virtual (`.build/`) para minificar la vasta hoja de estilos (`CSS_App.html` y `CSS_DesignSystem.html`) antes del *clasp push*, protegiendo el código fuente legible original.

## Out of Scope
- Migración a Webpack o Rollup reales (usaremos AST RegExp ligero para Minificar dado que es un proyecto Apps Script).
- Implementación de pruebas e2e automatizadas con Cypress / Puppeteer.

## Testing Strategy
1. Ejecutar `node deploy.js dev`.
2. Verificar en la consola y visor del proyecto remoto que los archivos estáticos CSS aterrizaron sin espacios ni saltos de línea (Minificados).
3. Abrir la URL de Desarrollo, abrir la Consola F12 y llamar `window.__runMemoryProfile_E2E()` comprobando que funciona idéntico a pesar de estar aislado en su propio módulo.
