# Story S11.3: Decoupled Event Controllers

## Objective
Actualmente, interacciones complejas de cambio de esquema como el evento `levelChanged` (que deshabilita/habilita selectores Padre según el nivel taxonómico) están atados monolíticamente al ciclo de vida del propio FormEngine (`FormEngine_UI.html`). El objetivo es segregar este despachador hacia una capa superior de "Event Bus" o "Control Delegativo" para mantener el Formulario 100% puro y declarativo.

## In Scope
- Creación de un Controlador de Eventos Pub/Sub (`UI_Bus` o `EventController`) en `JS_Core.html` para centralizar _Custom Events_ inter-componentes.
- Remover el despachador local y _listener_ (`CustomEvent('levelChanged')`) embebidos en el constructor procedural del Formulario.

## Out of Scope
- Migrar validadores backend de MDM.
- Construir nuevos componentes visuales de Web Components (Shadow DOM).

## Testing Strategy
1. Renderizar formulario `Dominio` o `Categoria`.
2. Alterar el campo "Nivel".
3. Verificar que el campo "Padre" se bloquee o repinte las opciones válidas a través de la comunicación asíncrona por el bus, sin errores de referencia cruzada O(N).
