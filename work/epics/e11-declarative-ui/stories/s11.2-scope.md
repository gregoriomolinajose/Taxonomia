# Story S11.2: Topological & Parsing Resilience

## Context
Tras culminar el Refactor Declarativo en `S11.1`, el siguiente paso crítico en la estabilidad de Frontend escalable es fortificar el consumo de la metadata que el motor visual traga para renderizar. Específicamente, dos debilidades identificadas:
1. **DAG Graph Edges:** Los selectores topológicos traen toda la metadata. Necesitamos un filtro robusto `currentActiveEdges` para ignorar nodos en desuso.
2. **Theming Parse:** Aislar el aplanador de Theming para que `index.css` (o su generador) opere ciegamente (`.default`) sin incondicionales de negocio anidados.

## Scope
### In Scope
- Refactorización del filtro local de aristas topológicas en `AppController` o `FormEngine_UI` (Implementación de `currentActiveEdges` filtering a nivel de memoria).
- Creación de un script limpio para parsear colores base e inyectarlos localmente de manera resiliente aislando un objeto `.default`.

### Out of Scope
- Migrar el Schema global entero.
- Cambiar la base de DB SCD-2 (Solo Frontend).

## Testing Strategy
- **Verificación Topológica:** Confirmar que los items del modal de árbol ya no renderizan IDs marcados como inactivos temporalmente.
- **Verificación CSS:** Confirmar lectura correcta de UI theme tokens primarios.
