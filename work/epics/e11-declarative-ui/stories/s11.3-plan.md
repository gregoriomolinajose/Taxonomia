# S11.3 Implementation Plan
**Story:** S11.3 (Decoupled Event Controllers)
**Size:** M
**Date:** 2026-03-30

## Overview
Reemplazar eventos de manipulación DOM procedural (`CustomEvent`) por un Patrón Pub/Sub puro in-memory embebido estratégicamente dentro del Closure del FormEngine (Protección anti Memory-Leaks).

## Task Execution Order

### T/11.3.1: Instantiate Local Closure Bus [FormEngine_UI.html]
- **Size:** S
- **Desc:** Declarar el modelo lógico `const LocalEventBus = { listeners: {}, subscribe: ..., publish: ... }` dentro de `renderForm` y transmitirlo a `UI_Factory.bindEngineEvents`.

### T/11.3.2: Reactor Migration (Subscribers) [FormEngine_UI.html]
- **Size:** M
- **Desc:** Modernizar el listener interno `bindLevelChangeRepaint` dentro de la factoría. Cambiar `.addEventListener('levelChanged')` a `eventBus.subscribe('TAXONOMY_LEVEL_CHANGED')`.

### T/11.3.3: Emitter Purification [FormEngine_UI.html]
- **Size:** S
- **Desc:** Erradicar el bloque repetitivo `querySelectorAll` dentro de `renderForm` línea ~1121 y sustituirlo por el simple cañonazo `LocalEventBus.publish(...)`.

### T/11.3.4: Manual E2E Validation
- **Size:** XS
- **Desc:** Abrir un Formulario Taxonómico y alternar el "Nivel", comprobando las cascadas de bloqueo visuales del Padre.

---
## Risks & Mitigations
- **Riesgo:** Zombis en la memoria si el usuario abre 20 formularios.
- **Mitigación:** Despejado por Arquitectura de Alcance Léxico local. Al sobreescribirse `app-container`, todo el Bus recolecta basura instintivamente.
