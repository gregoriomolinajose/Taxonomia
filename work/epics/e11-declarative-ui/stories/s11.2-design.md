# Story Design: S11.2 Topological & Parsing Resilience

## 1. What & Why
- **Problem**: Actualmente, la lógica que discrimina qué nodos o ramas del grafo están activos (`es_version_actual !== false`) está hardcodeada y dispersa en el Frontend visual (`FormEngine_UI.html` línea 252, etc.). Adicionalmente, el aplanador JSON de Theming (`CSS_DesignSystem.html`) posee un if _quemado_ para procesar la clave `default`.
- **Value**: Extraer ambos comportamientos hacia proxies de utilidades (`TopologyGuard` y un aplanador Theme Agnóstico) prevendrá roturas en masa si el DDL de la Base de Datos cambia en el futuro, y dejará el Motor Visual 100% abstraído de las reglas de negocio, permitiendo ingestas masivas.

## 2. Approach
Sustituiremos el filtrado local iterativo en `FormEngine_UI` inyectando una función estática de resolución `window.TopologyGuard.getActiveEdges(graph)`. En el Gestor CSS, reemplazaremos el chequeo manual `key === "default"` por una heurística de aplanamiento segura que no requiera reasignación dura de la raíz (`--`).

- **[MODIFY] FormEngine_UI.html**: Reemplazar `.filter(e => e.es_version_actual !== false)` por una delegación limpia.
- **[MODIFY] CSS_DesignSystem.html**: Refactorizar la función anónima `flattenObject` aislando las claves "default" sistemáticamente sin ensuciar la pila C++.
- **[MODIFY] JS_Core.html / Topology_Strategies.js**: Alojar el Namespace `TopologyGuard`.

## 3. Data Flow & Resilience (PAT-E-523)
- **Zero-Trust**: Si el caché de topología está vacío o llega `undefined`, `TopologyGuard` retornará `[]` en vez de explotar con TypeErrors (evitando crashes catastróficos silenciosos).
- **Graceful Rendering**: El aplanador CSS saltará suavemente cualquier token JSON mal engidado en lugar de truncar la iteración de la hoja de estilos.

## 4. Acceptance Criteria
1. **AC1 [Topología]**: Dado un campo _Searchable Multi_ asincrónico con caché poblado, la interfaz lo resuelve filtrando nodos "eliminados" o "históricos" sin iteraciones `.filter(...)` directas en la GUI.
2. **AC2 [Theming]**: Dada la carga en frío del SPA de Taxonomía, los sub-nodos y nodos `.default` de la base de tema se aplican como Propiedades Custom CSS (`--color-primary`, `--text-default`) exitosamente en el `:root`.
3. **AC3 [RaiSE Quality]**: El linter no emite warnings de funciones `filter()` duras relacionadas con base de grafos dentro de HTMLs visuales puros.
