# Design S6.5: Polymorphic Handlers (Strategy Pattern)

## Objetivo
Cumplimentar la recomendación AR H1. `Engine_Graph.js` actualmente confía en una sola instrucción condicional rígida (`maxActiveParents`). Debemos refactorizar el motor para que aplique un Patrón Strategy mediante inyección de funciones Handler, lo cual permite topologías de comportamiento variable que combinen métricas SCD-2 y constraints de negocio sin inflar la complejidad ciclomática de `patchSCD2Edges`.

## Target Solution
Convertir `TOPOLOGY_DICTIONARY` en un set de constructores/objetos con métodos delegados.
```javascript
const Topology_Strategies = {
    JERARQUICA_LINEAL: {
        validate: function(activeEdges) {
            if (activeEdges.length > 1) throw new Error("Violación de Topología...");
        }
    },
    ...
}
```
`Engine_Graph` solo hará:
```javascript
if (Strategy.validate) Strategy.validate(activeEdges, orphanEdges);
```
