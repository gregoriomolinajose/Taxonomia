# Story S6.2: Diccionario de Topologías

## Objective
Proveer a `Engine_Graph.js` de las reglas de validación topológicas para evitar la corrupción de cardinalidad cruzando estructuras organizativas (Funcional vs Matricial). Cuando un Front-End envíe un grafo subgrid con múltiples padres para una entidad cuya configuración (via Schema_Engine) asigne una topología 1:N estricta, el backend debe rechazar la transacción para respetar la Single Source of Truth del negocio.

## Definition of Ready
- S6.1 completada (Configurador `isTemporalGraph` y ruteador instanciados).
- ADR 002 Aprobado.

## Success Metrics
- El Diccionario de Estructuras (8 tipos) existe en código.
- Insersión múltiple falla determinísticamente (Throw Error) si `topology === FUNCIONAL`.
