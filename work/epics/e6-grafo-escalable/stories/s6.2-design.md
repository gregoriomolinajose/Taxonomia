# Design S6.2: Diccionario de Topologías

## Components Target
1. **Engine_Graph.js**:
   - Agregar el mapa de constantes:
   ```javascript
   const TOPOLOGY_DICTIONARY = {
       FUNCIONAL: { maxActiveParents: 1 },
       DIVISIONAL: { maxActiveParents: 1 },
       PLANA_HORIZONTAL: { maxActiveParents: null }, 
       EQUIPOS: { maxActiveParents: null },
       JERARQUICA_LINEAL: { maxActiveParents: 1 },
       LINEA_STAFF: { maxActiveParents: null }, 
       PROYECTOS: { maxActiveParents: null },
       HIBRIDA_MATRICIAL: { maxActiveParents: null }
   };
   ```
   - En `patchSCD2Edges`, antes de abrir la validez a las nuevas transacciones, se contabiliza la sumatoria y lanza la excepción:
   ```javascript
   const rules = TOPOLOGY_DICTIONARY[topology];
   if (rules && rules.maxActiveParents !== null) {
       if (activeEdges.length > rules.maxActiveParents) {
           throw new Error(`Violación de Topología: La estructura [${topology}] solo admite un máximo de ${rules.maxActiveParents} relaciones activas.`);
       }
   }
   ```

## Key Contracts
- `Engine_DB.js` captura los "throw Error" y API\_Universal\_Router\_Wrapper delega el `{status: "error", error: "..."}` al Frontend, que muestra una alerta nativa amigable al usuario sin congelar la app UI. Nada más se requiere.
