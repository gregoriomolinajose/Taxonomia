# Retrospective S29.8 - Auto-hidratación de Padres en inputs de Hijos

## Architecture Review (AR) - PASS
1. **Evitar lógica innecesaria:** Se logró abstener de tocar el código base del backend (`Engine_DB`) debido a su resiliencia innata a valores erróneos, centrando el fix completamente en un _bypass_ de UI visual.
2. **KISS & Proporcionalidad:** En vez de hacer complejas mecánicas asíncronas para precargar relaciones o crear un padre temporal, simplemente inyectamos la heurística nativa del marco de trabajo híbrido (Ionic) que confía ciegamente en inyecciones DOM por JavaScript.
3. **DRY:** Reutilizamos el potente sistema de introspección (`APP_SCHEMAS[targetEntity]`) para deducir cualquier Foreign Key dinámicamente de cualquier Entidad, por tanto el código sirve universalmente.

### Recommended (simplify before next cycle)
* **[RQ-Defensive Design - IMPLEMENTADO]** `UI_SubgridBuilder.client.js` e invocación estructurada ya integran de manera homologada el Safe Check: `const mockToken = (window.UI_CONSTANTS && window.UI_CONSTANTS.MOCK_FK_TOKEN) ? window.UI_CONSTANTS.MOCK_FK_TOKEN : '_NEW_PARENT_';` previniendo todo Type Error letal.

### Questions (require human judgment)
* **[Test Muda - IMPLEMENTADO]** Se introdujo la aserción 'S29.8 [QR/UX]: Aplica Fallback Seguro (Safe Check) cuando "window.UI_CONSTANTS" desaparece' en `RelationBuilder_Hydration.ui.test.js` la cual destruye el DOM en memoria defensivamente para emular el escenario de Caída Parcial antes de hidratar sus assertions.

## Quality Review (QR) - PASS
1. **Semántica Correcta:** La variable global pre-existente `window.currentEditId` funciona de manera determinista al inicializarse, garantizando que el origen del objeto sea el Formulario en curso independientemente del flujo de ejecución del usuario final.
2. **Auditoría de UX visual:** Al añadir el *label* "Padre en Curso", se informa de forma honesta, prolija y descriptiva acerca de la virtualidad del origen parental temporal dentro de la SPA en el proceso creador.
3. **Estabilidad Técnica:** Cobertura re-confirmada mediante pruebas automatizadas locales (`__tests__/RelationBuilder_Hydration.ui.test.js`) que validan la inexistencia de crash visual post-inyección.

## What went well
1. Localizar la Foreign Key recíproca iterando sobre el `APP_SCHEMAS[targetEntity]` nos evitó tener que inyectar nuevas propiedades de esquema, demostrando que nuestra configuración declarativa original es lo suficientemente robusta para la introspección (Introspection).
2. El uso de validadores visuales para el campo Dropdown interactuó a la perfección con la inyección de Mock Options. El `<ion-select-option value="_NEW_PARENT_">` funcionó fluidamente como bypass para Ionic.
3. El `Engine_DB` orquesta el mapeo de relaciones anidadas escribiendo forzosamente el ID real del padre recién creado encima de cualquier basura virtual que haya en la FK. La compatibilidad Backend-Frontend se mantuvo al 100%.

## What could be improved
1. Las llamadas explícitas a `localEventBus.publish` podrían envolverse en un "Builder Event Emitter" centralizado en `UI_Factory` para desacoplar todavía más los componentes puristas de sus dependencias en Pub/Sub del FormRenderer principal.
