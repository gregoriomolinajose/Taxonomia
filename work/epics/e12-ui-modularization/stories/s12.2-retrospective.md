# Retrospectiva S12.2: DataGrid & Routing Abstraction

- **Epic:** E12 UI Modularization
- **Fechas:** 31 Mar 2026
- **Estimado vs Real:** 85m estimados / 60m reales

## Logros y Puntos a Mejorar
- **What went well:** La abstracción de las factorías visuales (`UI_Router`, `UI_DataGrid`) limpió de manera masiva el archivo `DataView_UI.html` remarcando la necesidad de delegación de responsabilidades y *Single Responsibility Principle*.
- **What to improve:** Al purgar referencias de paginación se omitió validar retrospectivamente qué métodos las consumían (`_onPage` usando `_totalPages()`). El uso de referencias cruzadas *stringificadas* provocó una vulnerabilidad de ReferenceError. Es crítico siempre validar AST references tras extracciones masivas.

## Heutagogical Checkpoint
1. **What did you learn?** Que la implementación original de las tablas inyectadas en JS estaba severamente acoplada (Inversión de Control fallida, H14). 
2. **What would you change?** Al abstraer HTML in-line, debo parametrizar preventivamente todos los handlers a través de `this.cfg` en lugar de asumir llamadas al *Global Window*.
3. **Framework improvements?** Las heurísticas de Beck (`/rai-architecture-review`) detectaron de inmediato la falla "H14 Coupling Direction". Debería volverse un paso mandatorio en cualquier refactor de componentes SPA.
4. **More capable of?** Empaquetar Componentes Pasivos ("Dumb HTML Factories") que únicamente reaccionan a datos inyectados sin side-effects.

## Resoluciones Aplicadas
- ✅ Se integró la matemática Pura in-line y se mitigó la abstracción prematura.
- ✅ Cero acoplamiento: `UI_DataGrid` depende ahora 100% de `cfg`.
