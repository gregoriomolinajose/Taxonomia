# Scope: S12.3 GC Safety & Tooling Boundaries

## Boundary
**In Scope:**
- Inserción de `window.onerror` y pre-renderizado de Skeleton en `Index.html` (o `app-root`) antes de cargar los scripts de backend para evitar el White Screen de muerte prematura.
- Inyección de oyentes `ionModalDidDismiss` (u homólogos Ionic/Custom) en `UI_ModalManager.html` y purgas dependientes.
- Eliminación de suites asíncronas no utilizadas en Jest (`*.test.js`).

**Out of Scope:**
- Minificación CSS y refactoring Regex de `deploy.js` (diferido a épica externa).
- Re-diseñar el motor Modal Visual. Solo optimizamos su desmonte en memoria.
- Componentización global (S12.1/S12.2).

## Definition of Done (DoD)
1. Si un script inyectado desde GAS tiene un error sintáctico severo, la UI dibuja un mensaje amistoso de error rojo en vez de quedar eternamente en blanco.
2. Tras cerrar un modal, su nodo HTML en el DOM es borrado (`.remove()`) EXACTAMENTE al culminar su respectiva transición web Ionic, no con temporizadores arbitrarios de ±300ms.
3. El comando de test principal (`npm run test`) reporta 0 tests skippeados / mudos (y los reportados son suprimidos si no aportan mutability safety).
