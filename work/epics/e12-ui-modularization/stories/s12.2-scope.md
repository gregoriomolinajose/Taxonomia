# Scope: S12.2 DataGrid & Routing Abstraction

## Boundaries

### In Scope
- Extraer el motor lógico de navegación (URL sync, popstate) desde el mastodonte `JS_Core.html` hacia `UI_Router.html`.
- Mudar la lógica procedimental de ensamblaje de tablas HTML que recubre `DataView_UI.html` a una abstracción modular (`UI_DataGrid.html` o similar).
- Injectar como dependencias en `Index.html` hacia el runtime universal de GAS.

### Out of Scope
- Migrar el _Schema_ Engine a JSON puro sin compilador GS.
- Alterar la estructura o tokens del `CSS_DesignSystem.html` (mantener total paridad visual).
- Cualquier cambio mayor en Node JS Deployment `deploy.js` ajeno a minificación CSS.

## Done Criteria
1. `JS_Core.html` está despojado del `window.history.pushState` y funciona exclusivamente como Service Worker pasivo del API (Agnostic).
2. Los componentes tabulares de listados (DataTables) actúan por delegación y polimorfismo hacia `DataView_UI`.
3. Pasa pruebas en Sandbox sin quebrar Deep Linking o el botón lateral 'Atrás' del móvil.
