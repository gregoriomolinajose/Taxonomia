# S12.3: GC Safety & Tooling Boundaries

## Historia de Usuario
**Como** Arquitecto de Taxonomia
**Quiero** establecer barreras robustas contra fallas críticas (White Screen of Death) y optimizar el Garbage Collection nativo de los componentes Ionic
**Para** garantizar que la SPA no colapse por errores sintácticos de fragmentos asíncronos y para eliminar las fugas de memoria o flashes provocados por cierres pesimistas (`setTimeout`) de los modales.

## Criterios de Aceptación
1. **API Fail-Fast Scoping:** El `Index.html` principal debe poseer un manejador de excepciones global (`window.onerror`) que provea retroalimentación visual amigable en caso de SyntaxError, evitando el WSOD nativo.
2. **Optimistic Timers Cured:** El método de desmontaje de `UI_ModalManager.html` y otros gestores de overlays deban suscribirse al evento nativo `ionModalDidDismiss` (o similar de Ionic) en vez de requerir timers agnósticos como `setTimeout(() => topModal.remove(), 300)`.
3. **Muda Clean-up:** Eliminar del repositorio los fragmentos inútiles identificados, como tests ignorados (`describe.skip`) de suites migradas, o variables muertas.
