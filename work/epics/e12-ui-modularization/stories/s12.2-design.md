# Architecture Design: S12.2 DataGrid & Routing Abstraction

## What & Why
- **Problem**: El Orquestador Lógico de la SPA (`JS_Core.html`) y de las Vistas (`DataView_UI.html`) contienen miles de líneas combinadas de funciones procedimentales y utilidades espuria entrelazadas (DOM iterators, Sidebar renders, Table headers iterators). 
- **Value**: Extraer la carga de navegación (`UI_Router`) y ensamblajes de UI en tablas (`UI_DataGrid`) incrementa drásticamente la testeabilidad de la aplicación, agilizando los despliegues de GAS y facilitando futuros _Hot-Reloads_.

## Components Affected
- `JS_Core.html` (MODIFIED): Purgando métodos `navigateTo`, `showFormSidebar`, `showHomeSidebar`, etc.
- `DataView_UI.html` (MODIFIED): Purgando subrutinas estáticas `_renderTableView`, `_renderGridView`, `_renderEmpty`, `_renderSkeleton`, y su *Drag n Drop* HTML in-line.
- `UI_Router.html` (NEW): Factory que recibe eventos NAV::CHANGE para montar vistas. Reaccionará usando History API en un futuro.
- `UI_DataGrid.html` (NEW): Factory delegada encargada de renderizar 100% de los polimorfismos del `app-container` relativos a grillas de entidades. Retorna strings/nodos HTML puros.
- `Index.html` (MODIFIED): Configurar el `<?!= include() ?>` de las nuevas factorías lógicas inmutablemente.

## Examples
### Rendering a Grid via DataView_UI orchestrating
```js
// En DataView_UI.html (_rerender() method)
// ANTES (Fat Controller):
const html = _state.view === 'table' ? _renderTableView(_pageData) : _renderGridView(_pageData);
appContainer.innerHTML = html;

// AHORA (Agnostic Controller):
const html = global.UI_DataGrid.buildLayout(_state.view, _pageData, _state.columns, {
    sortCol: _state.sortCol,
    sortDir: _state.sortDir,
    pagination: _state.pageObject, // derived
    entityName: _state.entityName,
    onSortCallback: 'window.DataViewEngine._onSort', // Dependency Inversion
    onDragStart: 'window.DataViewEngine._onDragStart',
    // ...
});
appContainer.innerHTML = html;
```

## Acceptance Criteria
*(Ref S12.2-Story)*
- **MUST**: Aislar `UI_Router.navigate` y removerlo íntegramente de `JS_Core`.
- **MUST**: Aislar `UI_DataGrid.buildLayout` y remover la iteración HTML de `DataView_UI`.
- **MUST NOT**: Romper el arrastre de cabeceras (Drag n Drop) de T10.2 ni los estilos subyacentes.
