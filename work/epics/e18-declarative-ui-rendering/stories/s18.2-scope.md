# S18.2: CSS Declarative Visibility

## Objetivo de la Historia
Eliminar la dependencia nociva de mutar propiedades de estilo imperativas a nivel semántico (`element.style.display = 'none'`) en toda la base de JavaScript. Transicionar a la habilitación/deshabilitación de estados CSS globales gestionadas por el framework Ionic (`ion-hide`) o clases propias del `ThemeManager`.

## Alcance (In Scope)
- Reemplazar todas las ocurrencias directas de `.style.display = 'none'` y `.style.display = 'block' | 'flex'` a través de los componentes UI (`UI_FormStepper`, `UI_Router`, `FormBuilder_Inputs`, `Dashboard_UI`, `DataView_UI`, `Auth_UI`).
- Utilizar el patrón nativo nativo: `element.classList.add('ion-hide')`, `element.classList.remove('ion-hide')`, o preferiblemente el atajo sintáctico `element.classList.toggle('ion-hide', booleanCond)`.

## Fuera de Alcance (Out of Scope)
- Modularizar o refactorizar el HTML interior de dichos componentes (S18.3 y S18.4).
- Extracción de propiedades que no sean `.display`.

## Criterios de Éxito
- La búsqueda global de `style.display` en `src/` no debe arrojar incidencias imperativas.
- Las animaciones o *reflows* de Ionic no deben degradarse, ya que `ion-hide` es interpretado semánticamente por la cascada M3 que definimos preventivamente.
