# Story Review: S18.3 Static Template Isolation

## Metrics
- **Objective:** Aislar Layouts pasivos hacia Nodos Inmutables (Templates HTML5).
- **Status:** COMPLETED.
- **Architectural Debt Incurred:** 0. 
- **Architectural Debt Payed:** 3 (Acoplamiento de Strings a DOM, Strings Hardcodeados en factoría, Memory leaks en `DataView`).

## Retrospective

### What went well
- **Decisión de Arquitectura Conjunta:** El rechazo inicial del plan nos salvó de un Anti-Patrón V4 (La Sobre-ingeniería Estática). Al dividir S18.3 (Estático) de S18.4 (Dinámico), mantuvimos el scope atómico.
- **Apertura Defensiva:** Usar `tmpl ? template.content : emptyFragment` nos brindó "Graceful Degradation" inmediata sin costo extra.
- **DOM Nativo:** `.replaceChildren(fragment)` resultó ser la llave maestra para conectar la cadena asimétrica de HTML hacia DOM seguro.

### What needs improvement
- El puente transaccional temporal `_htmlToFragment()` en `UI_DataGrid` funciona excepcionalmente bien para detener el vector principal de `innerHTML`, pero oculta las mutaciones on-the-fly (`onclick=""` en HTML) dentro del transpilador. Resolver esto estructuralmente es el propósito explícito y aislado de la inminente S18.4.
