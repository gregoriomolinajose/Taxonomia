# Retrospective: S28.3 Modernización & Componentización de Dashboard Card

**Story:** S28.3
**Dates:** 08-04-2026
**Estimated:** 20m
**Actual:** 35m

## Lo que salió bien
- Se migró el diseño rígido acoplado a una plantilla del index HTML a un plugin nativo abstraído (`UI_Component_DashboardCardCont.client.js`).
- El componente acepta variables CSS de temas dinámicos (`var(--ion-color-danger)`) y mapeos estúpidos locales para mayor reutilizabilidad.
- Refactorizamos `Dashboard_UI.html` sin quebrar el motor asincrónico original.

## Lo que se puede mejorar
- Se nos escapó que la aplicación App Script no iba a cargar el módulo en el frontend mágicamente de la base de código. Olvidamos agregar el include macro (`<?!= include(...) ?>`) lo que ocultó silenciosamente nuestra UI en Producción/Dev. Detectar el acoplamiento de Inyección de HTML tempranamente evita bugs molestosos.

## Heutagogical Checkpoint

**¿Qué aprendí?**
Encapsular elementos del DOM en funciones puras vainilla para GAS y registrarlas en `UI_Factory` resulta infinitamente más sostenible para un diseño SPA moderno que mantener monolitos llenos de etiquetas invisibles como pseudo-Shadow DOMs.

**¿Qué cambiaríamos del proceso?**
Comprobar siempre qué dependencias estáticas maneja la plataforma objetivo al crear un nuevo módulo (en este caso el *include* manual de Google Apps Script), antes de asumir que el *bundler* en el local solucionará la orquestación.

**¿Mejoras al framework (RaiSE)?**
Se debería crear a futuro un `rai generate component <nombre>` que automatice no solo la creación de este archivo patrón `IIFE > UI_Factory`, sino que alerte o inyecte el `include` automáticamente dentro del `Index.html`.

**¿De qué soy capaz ahora?**
Soy muchísimo más resiliente encapsulando cualquier widget legacy que está ensuciando un layout global hacia constructores puros para Reactividad Modular sin requerir un VDOM de Framework.

## Mejoras Aplicadas
- Agregamos la regla de control (PAT-G-031) sobre validación del macro `include` al extraer plantillas, promoviendo el conocimiento.
- Archivos limpios en la vista arquitectónica.

## Patterns Added
- **PAT-G-031** (Architecture): Separar plantillas HTML en scripts UI_Factory independientes. Siempre revisar incluirlos explicitamente en Google Apps Script Index.html con `<?!= include(...) ?>`.
