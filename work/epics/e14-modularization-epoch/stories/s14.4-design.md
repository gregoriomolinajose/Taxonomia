# S14.4 Design Document: ES6 Syntactic Purity

## Objetivo
Erradicar los remanentes de sintaxis "Legacy" (ES5) presentes en el renderizador principal de tablas (`UI_DataGrid.html`) y estabilizar el flujo asíncrono de destrucción de ventanas modales (`UI_ModalManager`) usando ES6 Promises / async-await.

## Alcance de la Refactorización

### 1. Template Literals en `UI_DataGrid.html`
Actualmente, `UI_DataGrid.html` genera estructuras complejas de DOM concatenando strings con `+` y variables `var`. Esto no solo es propenso a errores de comillas anidadas, sino ineficiente a nivel de lectura e inferencia.
**Solución:** 
- Reemplazar toda la instanciación de strings de HTML por **Template Literals** (Backticks `` `...` ``).
- Actualizar `var` por `const` y `let`.
- Actualizar iteradores `map(function(item) { ... })` por Arrow Functions (`map(item => { ... })`).
- *Restricción:* Cuidar la interpolación condicional (e.g. `isSorted ? 'sorted' : ''`) incrustándola limpiamente en `${...}`.

### 2. Async/Await en `UI_ModalManager.html`
Actualmente `closeTop()` remueve el modal usando un event listener optimista (`ionModalDidDismiss`) y dispara `topModal.dismiss()` de forma asíncrona sin bloquear el flujo ni esperar la promesa. Esto puede generar "Race Conditions" en la mutación de estado topológico si el componente padre lee el estado global o la URL se actualiza prematuramente.
**Solución:** 
- Refactorizar `closeTop: function()` a `closeTop: async function()`.
- Reemplazar el engorroso listener por un bloque estricto `try { await topModal.dismiss(); } finally { topModal.remove(); }`.
- Garantizar la destrucción y la reasignación de los contadores internos estrictamente al final de la transición visual del DOM.
