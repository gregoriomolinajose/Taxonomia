# S14.3 Design Document: PubSub Topológico

## Objetivo
Invertir la dependencia circular y arquitectónica entre `UI_SubgridBuilder` y `FormBuilder` (`window.renderForm`). El "Subgrid" es un Componente Visual Anidado, no un Enrutador Maestro. Actualmente, `UI_SubgridBuilder` invoca estáticamente a `window.renderForm` (su padre) para instanciar sub-formularios, rompiendo el Principio de Responsabilidad Única (SRP) y acoplándose al ciclo de vida global.

## Problemas de Acoplamiento Detectados
1. **Llamada Global Inversa:** `UI_SubgridBuilder.html` (L224) ejecuta `window.renderForm(field.targetEntity, null, callback)`. 
2. **Dependencia de Búsqueda de DOM Sucia:** `UI_SubgridBuilder.html` (L124) busca dependencias en el DOM raíz a través de `window.currentFormModal` para extraer el valor de `nivel_tipo` dinámicos.

## Solución Arquitectónica: Semantic Pub/Sub (EventBus)

Aprovechando que `FormRenderer_UI.html` y la macro-función `renderForm` ya instancian un `LocalEventBus` encapsulado a nivel léxico (Lexical Scope) para el ciclo de vida del modal:

### 1. Inyección de Dependencias
Modificar la firma de invocación del Subgrid en `FormRenderer_UI`:
```javascript
// Antes
await window.UI_SubgridBuilder.build(field, subCol, data, entityName);

// Después
await window.UI_SubgridBuilder.build(field, subCol, data, entityName, LocalEventBus, modal);
```

### 2. Inversión del Callback por Emisión de Eventos
El Subgrid dejará de ejecutar funciones ajenas. Emitirá un Intent:
```javascript
// UI_SubgridBuilder.html
localEventBus.publish('UI::REQUEST_SUBFORM_OPEN', {
    targetEntity: field.targetEntity,
    onSuccess: (newRecordResp) => {
        // Resolver hidratación anidada
    }
});
```
Y el Macro-Controlador (`FormRenderer_UI.html`) será quien decida cómo atender esa orden:
```javascript
// FormRenderer_UI.html (adentro de renderForm)
LocalEventBus.subscribe('UI::REQUEST_SUBFORM_OPEN', (ev) => {
    const { targetEntity, onSuccess } = ev.detail;
    window.renderForm(targetEntity, null, onSuccess);
});
```

### 3. DOM Isolation Fix
Para erradicar `window.currentFormModal` del `UI_SubgridBuilder`:
Al inyectar directamente el `modal` (que funge como context-root) desde FormRenderer, la búsqueda del input `nivel_tipo` pasará a ser una directiva contextual pura:
```javascript
const liveNivelInput = modalContext ? modalContext.querySelector('ion-input[name="nivel_tipo"]...') : null;
```

## Resultados Esperados
- Cero dependencias globales en el `UI_SubgridBuilder`.
- Integridad Topológica donde la Factoría sólo produce Nodos DOM (Vistas) y emite notificaciones a su Presentador/EventBus.
