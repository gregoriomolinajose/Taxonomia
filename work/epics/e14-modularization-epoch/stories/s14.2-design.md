# Design Specification: S14.2 Native CSS Bundler

## 1. Problem Statement
Actualmente, los estilos de Taxonomía residen en archivos híbridos (`CSS_App.html`, `CSS_DesignSystem.html`) envueltos en etiquetas `<style>`. Esto destruye la "Developer Experience" (DX) al deshabilitar el IntelliSense nativo de CSS, linting, y formateadores (Prettier) en los editores de código (VS Code), ya que se tratan como archivos HTML incrustados.

## 2. Proposed Architecture
Adoptaremos una arquitectura de **Native Assets** donde los archivos fuente serán estricta y puramente `.css`, y nuestro pipeline de Node.js (`deploy.js`) actuará como un **Bundler "Just-In-Time"** (JIT) durante el despliegue.

### El Pipeline JIT:
1. **Source of Truth:** Los archivos se moverán a `src/assets/css/app.css` y `src/assets/css/design-system.css`.
2. **Build Transformation:** 
   * `deploy.js` leerá los `.css` nativos de `src/assets/css/`.
   * Ejecutará `CleanCSS()` para minificar el AST del CSS.
   * Envolverá el string resultante en `<style>` ... `</style>`.
   * Emitirá `CSS_App.html` y `CSS_DesignSystem.html` dentro de la carpeta temporal `.build/`.
3. **Zero-Touch Backend:** `Index.html` en App Script no sufrirá **ningún** cambio. Seguirá llamando `<?!= include('CSS_App'); ?>`, manteniendo la integración con el backend intacta.

## 3. Topología de Directorios
```text
src/
 ├─ assets/
 │   └─ css/
 │       ├─ design-system.css  (Nuevo: Pure CSS)
 │       └─ app.css            (Nuevo: Pure CSS)
 └─ Index.html (Sin cambios)

.build/
 ├─ CSS_DesignSystem.html      (Generado por deploy.js)
 └─ CSS_App.html               (Generado por deploy.js)
```

## 4. Dependencias y Riesgos
- **Riesgo 1 (Clasp Ignore):** `.claspignore` podría estar bloqueando carpetas anidadas o permitiendo `.css`, pero como enviamos desde `.build/`, solo los `.html` compilados serán enviados.
- **Riesgo 2 (Pathing):** Al mover el código, borrar los archivos originales `src/CSS_App.html` y `src/CSS_DesignSystem.html` para no tener fuentes duplicadas de la verdad.

## 5. Criterios de Éxito
- [ ] Múltiples archivos `.html` con `<style>` desaparecen de `src/`.
- [ ] VS Code reconoce el código como CSS puro (Syntax Highlighting completo).
- [ ] El comando `npm run deploy:dev` compila el CSS exitosamente al formato GAS HTML sin intervención humana.
