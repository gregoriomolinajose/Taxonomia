# S14.1 Scope: FormEngine Splitting

## Goal
Descuartizar el monolítico componente procedural `FormRenderer_UI.html` (~1300 líneas), implementando el patrón de Clases (ES6 Classes) o Factory Pattern para lograr Encapsulamiento, Single Responsibility Principle (SRP) y reducir la complejidad cognitiva. 

## Done Criteria
- [ ] Partir `FormRenderer_UI.html` al menos en 3 submódulos independientes (Ej: `UI_Stepper`, `UI_FormState`, `UI_FormSubmitter`).
- [ ] Garantizar que no haya regresiones en los flujos de creación/edición Modal de entidades (Polimorfismo).
- [ ] Implementar Inyección de Dependencias para proveer a los sub-componentes acceso a la configuración de Entidades y a la metadata de `window.APP_SCHEMAS`.
