# S21.3: Soft-Delete Topological Integrity Implementation

**As a** Governance System Administrator
**I want** to perform soft-deletes (logical deletions) on nodes in the database instead of physically removing them
**So that** parent-child relationships and dependent analytics do not cascade into corruption and historical data remains queryable.

## Acceptance Criteria

**Scenario:** Soft-deleting a node
*Given* an existing record in the database
*When* I invoke `removeData` or its equivalent for that record
*Then* the record is not physically deleted from the `SpreadsheetApp` backend
*And* its passive flag `_isDeleted` (or similar metadata) is set to `true`
*And* subsequent `listData` or search operations omit this record by default

**Scenario:** Soft-deleted node validation
*Given* a soft-deleted record
*When* attempting to assign it as a parent in a subgrid relation of another entity
*Then* the system rejects it as an invalid/archived node
