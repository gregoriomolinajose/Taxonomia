# S21.2 Scope: Asynchronous Bus Debouncing & Typeahead Filters

## Boundaries

**In Scope:**
- Implementation of a global `debounce` utility within `JS_Core` or the `AppEventBus`.
- Refactoring `UI_DataGrid` search inputs to use the debounced logic to prevent DOM-freeze during aggressive local row-filtering.
- Addressing the global Omnibar behavior to prevent over-firing global state updates (`NAV::CHANGE` or search queries).

**Out of Scope:**
- Infinite scrolling logic (belongs to a different paradigm).
- Web Worker thread offloading (we remain in the main thread using standard async timeouts).
- Real-time WebSockets synchronization.

## Done Criteria
- A standard `.debounce(fn, delay)` mechanism exists and is attached to the AppEventBus or available globally.
- `UI_DataGrid` text inputs visually react fluidly without blocking the main browser thread while the user types.
- The `Index.html` or equivalent component consuming the filter emits the filter event only after the keystroke pause.
- No regression in search results accuracy.
