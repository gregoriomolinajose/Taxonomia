# S21.2: Asynchronous Bus Debouncing & Typeahead Filters

## User Story
**As a** frequent system operator in the Taxonomia SPA
**I want** my keystrokes in search bars and DataGrids to be debounced asynchronously
**So that** the application does not freeze (thread-freeze) or trigger excessive RPC calls while I am still typing, improving the overall responsiveness and UX.

## Acceptance Criteria
```gherkin
Feature: Asynchronous Debouncing in UI

  Scenario: User types rapidly into the Omnibar or DataGrid search
    Given the user is focused on a search input field
    When the user types multiple characters quickly (e.g. within 300ms)
    Then the system should NOT trigger the filtering or RPC search logic for every keystroke
    And the system should only execute the search once the user pauses typing for a predefined debounce threshold (e.g. 250-400ms).

  Scenario: Typeahead UI reflects waiting state
    Given the user is typing rapidly
    When the debounce timer is active
    Then the UI should optionally show a subtle typing indicator without blocking interactions
    And the EventBus should swallow intermediate `FILTER::CHANGE` events to prevent over-rendering.
```
