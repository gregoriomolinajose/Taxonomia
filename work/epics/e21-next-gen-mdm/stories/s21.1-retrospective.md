# Story Retrospective: S21.1 Optimistic Locking Pipeline

## 1. What Went Well
- Logramos anclar la validación matemática de versionamiento usando la variable plana `_version` directo en el `Adapter_Sheets.upsert` de $O(1)$. No requirió una Query extra, manteniendo latencia puramente lineal.
- Se reutilizó el ecosistema JSON Response global modificando sólo 2 strings (`errorType: 'CONCURRENCY'`).

## 2. What Could Be Improved
- Actualmente el "Soft Error" le advierte al usuario, pero le sugiere un Refresh global para recuperar los datos nuevos. Pudimos haber forzado un *Auto-Fetch*. Se dejará en backlog si UX detecta fricción manual.

## 3. Learnings
- Google Apps Script soporta la destrucción y mapeo numérico impecable desde el `Array` tabular, sin lidiar con los cast a string, demostrando una excelente resiliencia en V8 para cálculos auto-incrementales.
- El Submitter no tuvo que reingeniarse gracias a que `payload[fieldName]` captura inputs ocultos si y solo si poseen el tag `name=""`.

## 4. Action Items
- Para `S21.3 Soft Delete` (que también mapea a la manipulación del `Estado` interno de las hojas), es indispensable asegurar que omitimos también la inserción forzada.
