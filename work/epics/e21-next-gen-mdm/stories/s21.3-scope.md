# S21.3 Scope Boundaries

## In Scope
- Modify `Engine_DB.gs` (specifically `removeData`, `deleteData`, `listData`) to evaluate and toggle an `_isDeleted` metadata flag instead of physical row deletion.
- Enhance `Adapter_Sheets.js` to handle `_isDeleted` persistence seamlessly seamlessly as part of O(1) mutations.
- Ensure the Frontend (`UI_DataGrid` / Global Search) gracefully ignores nodes with the `_isDeleted` flag enabled.
- Protect topological parents: child nodes inheriting topological metadata should evaluate parent's soft-deleted state.

## Out of Scope
- A dedicated UI for administrators to perform "Hard" deletions or empty a "Recycle Bin".
- Refactoring `API_Universal` beyond passing the flag.
- Modifying E22's Omnibar component or interfering with another agent's work.

## Done When
- [ ] Backend `removeData` transforms into a topological Soft-Delete.
- [ ] Database `listData` and queries correctly filter out soft-deleted nodes.
- [ ] Attempting to bind a deleted entity results in a clear error.
- [ ] Verified via manual testing without corrupting the Spreadsheet.
