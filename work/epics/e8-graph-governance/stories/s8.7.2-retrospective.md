# Retrospective: S8.7.2 - Event Listener de Repaint Binding

## Summary
- **Story ID:** S8.7.2
- **Dates:** 2026-03-30
- **Size:** XS
- **Estimated / Actual Time:** 20m / 20m

## What Went Well / What to Improve
- **Well:** El oyente (`eventListener`) asíncrono reaccionó perfectamente a los cambios reactivos dentro del DOM (evento `levelChanged`). Esto impidió roturas frontales durante la mutación del árbol, eliminando posibles referencias fantasma a "Padres" obsoletos.
- **Improve:** Sufrimos una repetición literal de la lógica de inyección de `<ion-select-option>` en la fase original y en el callback de Repaint. Esta vulneración silenciosa a las normas DRY (*Don't Repeat Yourself*) se toleró como hotfix, pero ha sido anotada para expurgación obligatoria mediante la modularización planeada en E9.

## Heutagogical Checkpoint
1. **What did you learn?**
   El recálculo de jerarquías sobre matrices UI pre-filtradas exige la eliminación absoluta (Flush) de la data previa en la memoria del componente sub-virtual del navegador (e.g. `freshFiltered = []`) para garantizar un borrón atómico.
2. **What would you change about the process?**
   Instalar reglas de validación que impidan declarar *Custom Events* complejos de manera "inline" en el script integrador de la vista `HTML`. Esto asegura el Single Responsibility Principle y agiliza lecturas.
3. **Are there improvements for the framework?**
   El *Parking Lot* debe ser el estándar a partir de ahora cada vez que emitimos código repetido intencionalmente como Hotfix. (Estrategia de Deuda Consentida explícita).
4. **What are you more capable of now?**
   Puedo crear bucles de interacción isomórfica cerrados desde un mismo archivo de renderizado, asegurando que los eventos propagados por una mutación de un campo UI actualicen de forma inteligente todos sus Dropdowns subordinados *ad-hoc*.

## Improvements Applied
- Reactividad directa para *Strict Level Jumps* garantizada sin necesidad de *Page Reloads*.
- El elemento se inhabilita automáticamente si se percibe la selección explícita del Nodo Principal (`cLevel` = 1).

## Patterns Added/Reinforced
1. **ADDED [architecture]:** "DOM Repaint Reactivity. Todo repintado asíncrono en interfaces jerárquicas ocasionado por mutación temporal de dependencias debe contemplar un flush defensivo (vaciado de estado local) previo a la re-inyección visual para prevenir la selección de relaciones huérfanas en el DOM." (From S8.7.2)
