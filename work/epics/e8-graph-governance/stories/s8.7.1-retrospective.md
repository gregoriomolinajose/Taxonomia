# Retrospective: S8.7.1 - In-Line UX & LIFO Modal Stack

## Summary
- **Story ID:** S8.7.1
- **Dates:** 2026-03-30
- **Size:** M
- **Estimated / Actual Time:** 90m / 120m

## What Went Well / What to Improve
- **Well:** La incrustación iterativa del `FormEngine` sobre sí mismo permitió a los *Data Stewards* de la aplicación cargar linajes completos (ej. crear una Familia huérfana mientras se llena el formulario del Módulo) sin tener que abandonar o recargar la página. Un avance radical para el *Data Entry*.
- **Improve:** En la refriega térmica frente a Ionic, acoplamos el índice de Control a una variable explícita nativa global (`window.formModalStack`). Las dependencias de variables universales de Runtime rompen el paradigma Reactivo e Isomórfico y han sido anotadas para refactorización (Deuda Técnica, Parking Lot E9).

## Heutagogical Checkpoint
1. **What did you learn?**
   El anidamiento UI profundo es poderoso pero engañoso; crear superposiciones (*Z-index layers*) requiere de un Garbage Collector manual. Descubrimos empíricamente incrustando métodos *Pop()* que cerrar vistas superpuestas lógicamente no destruye el nodo visual instantáneamente si hay referencias de memoria (*Closures*) sosteniéndolo en RAM.
2. **What would you change about the process?**
   Establecer un límite de profundidad máximo (*Max Depth UI Guard*) algorítmico desde el Inicio. No limitar un esquema de *Modals* invoca problemas de saturación de dispositivo móvil que es mejor truncar lógicamente en el Controller.
3. **Are there improvements for the framework?**
   Todos los atributos Reactivos (*Theme Managers, Event Listeners*) deben residir en Controllers de SPA (*SubgridState.js*) en vez del DOM (*HTML*). Hemos dejado constancia rigurosa en el estacionamiento (Parking Lot).
4. **What are you more capable of now?**
   Tengo una habilidad inquebrantable para orquestar motores `Data-Driven` y `Config-Driven` (Schema_Engine) renderizando componentes de interfaz abstracta que superan fronteras lógicas, posibilitando a los usuarios dibujar Grafos Temporales Multidominio al vuelo.

## Improvements Applied
- Apilado (*Push*) y Cierre manual (*Pop*) de Interfaces superpuestas.
- Creación paramétrica abstracta del Dropdown de Entidades forzando su lectura desde el Component Factory.
- Soporte visual de *Truthiness* mitigando dependencias asimétricas.

## Patterns Added/Reinforced
1. **ADDED [architecture]:** "Modal Stack Recursion (MDM). Permitir la instanciación in-line de formularios superpuestos (Pila LIFO) agudiza la eficiencia del Operador de Datos al prevenir cambios de pantalla (Context Flushing), con la obligación estricta de desapilar explícitamente tras operaciones I/O exitosas para prevenir Memory Leaks." (From S8.7.1)
