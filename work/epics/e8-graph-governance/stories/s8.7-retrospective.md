# Retrospective: S8.7 (In-line UX & Branch Shifting)

## Summary
- **Story ID**: S8.7
- **Dates**: Mar 30, 2026
- **Estimated Time**: 45 minutes
- **Actual Time**: 45 minutes
- **Status**: Completed

## What Went Well
- La conceptualización del `Modal Stack` en lugar del reemplazo de DOM nos permitió escalar la UX del FormEngine sin tener que rehacer módulos pesados ni ensuciar `DataViewEngine`.
- Enlazar la resolución del Callback en `__onSaveSuccessFallback` evitó la necesidad de implementar un Dispatcher de Eventos centralizado excesivo, manteniendo el acoplamiento ligero y resolviendo instantáneamente el problema del Contexto.
- El interceptor `isTemporalGraph` en el selector Single se ejecutó suave y a prueba de errores gracias a comparar `newVal !== originalVal` antes del Rollback.

## What Needs Improvement
- La variable global primitiva `window.currentFormModal` tuvo que coexistir a pesar del stack para no re-escribir docenas de llamadas al DOM en rutinas auxiliares o externas. Debemos vigilar que futuras refactorizaciones a componentes custom siempre usen `window._closeTopModal()` en lugar del viejo método `window.currentFormModal.dismiss()`.

## Heutagogical Checkpoint

**What did you learn?**
Implementar apilamiento LIFO (Last-In-First-Out) en una Single Page Application de Vanilla JS (con WebComponents de Ionic) exige responsabilidad estricta en la destrucción de elementos. `.remove()` fue crítico para evitar que los Modales anidados se conviertan en Memory Leaks al cerrarlos.

**What would you change about the process?**
Diseñar "Branch Shifting" se debatió mucho desde backend, pero la solución real provino de agregar `ion-alert` nativo en UX. Hubiera sido mejor conceptualizar este Guard de Confirmación durante la historia S8.4 (Deletion Strategies Exprimentation).

**Are there improvements for the framework?**
Sí, considerar la creación de un `ModalManager.js` independiente si las alertas e interceptores de la UI se vuelven más ubicuos, retirando tanta responsabilidad al `FormEngine_UI.html` bruto.

**What are you more capable of now?**
Concebir Sistemas de Componentes Anidados Completamente Autónomos (Renderizador Recursivo). Una UI modular MDM es verdaderamente invencible si cada Formulario ignora en absoluto que hay otros apilados, garantizando un State encapsulado por invocación LIFO.

## Patterns Added/Reinforced
1. **[Added] PAT-G-009**: Form Modal Stacking (LIFO) via `formModalStack` tracking to allow recursive FormEngine inline entity creation instead of overriding the active DOM instance (`-c FormEngine, Modal Stack, Recursion, UI`).
2. **[Reinforced] PAT-G-006**: *Graceful UI Degradation (Zero-Touch Interceptors)* al aislar Validaciones de Topología (Branch Shifting Guards).

## Applied Fixes
- Fix estructural a Memory-Leaks de Modales nativos al implementar inyección forzada con destrucción `.remove()`.
