# User Story S8.3: Re-parenting & Temporal SCD-2

**Como** usuario del motor gráfico
**Quiero** poder asignar un nodo hijo existente a un nuevo padre (Re-parenting o Robo de Nodos)
**Para** corregir estructuras sin violar la base de datos temporal ni alterar la historia pasada de asignaciones (SCD-2).

## Acceptance Criteria
- C1: Si un nodo es "robado" por otro padre, la arista antigua debe cerrar su `valido_hasta` con la fecha actual, marcando `es_version_actual = false`.
- C2: Debe instanciarse una nueva arista SCD-2 entre el nuevo padre y el hijo con `valido_desde` actual.
- C3: Si la `topologyRules.allowOrphanStealing` es `false`, la UI o DB Enforcer debe denegar el "robo" e informar exclusividad.
