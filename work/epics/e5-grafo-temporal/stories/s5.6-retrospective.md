# Retrospective: S5.6

## Learnings
1. **User Over System:** Al inicio la historia buscaba refactorizar el cálculo de `orden_path` en `Math_Engine` usando búsquedas sobre el puente DAG. El feedback oportuno redireccionó el esfuerzo hacia eliminar dichas variables por completo: "Ya no vamos a necesitar el path orden, ni el path completo". Esto nos ahorró complejidad innecesaria.
2. **Framework Leverage:** El motor `orchestrateNestedSave` del `Engine_DB.js` fue lo suficientemente genérico para absorber el modelo recursivo SCD-2 (una vez inyectadas las fechas `valido_hasta`) lo cual demuestra que los "Golden Patterns" de backend reducen drásticamente la creación de código nuevo en cada iteración.

## Action Items
- Para futuras migraciones a DAGs, auditar proactivamente qué esquemas dependen visualmente de sus campos calculados y pre-aprobar su eliminación.
