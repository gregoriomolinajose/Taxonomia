# Retrospective: S5.2 - Script ETL de Migración Estructural Transaccional

## Overview
- **Story:** S5.2
- **Epic:** E5 - Grafo Temporal Multi-Estructural
- **Date:** 2026-03-28
- **Estimated Size:** M (45 mins)
- **Actual Size:** S (30 mins)

## What Went Well
- **Rendimiento O(1):** Reducir todas las mutaciones a memoria RAM y limitarnos a exactamente 2 envíos transaccionales masivos con `setValues()` permitió acatar las estrictas cuotas de velocidad del V8 Engine.
- **Detección Defensiva:** Prever el *fallback* de fechas `new Date().toISOString()` para filas donde el `created_at` faltase evadió fallos catastróficos. Interceptar strings "NULL" erróneos fue clave para limpiar el rastro `id_padre`.

## What to Improve
- En proyectos GAS de este grosor, trabajar scripts *one-shot* en seco (sin TypeScript, sin unit tests integrados que puedan mockear el `SpreadsheetApp`) se siente como código a ciegas. Tuvimos que inyectar múltiples `Logger.log` por seguridad. A futuro, se podrían aislar las funciones pura-lógicas pasando Arrays crudos como argumentos.

## Heutagogical Checkpoint
1. **What did you learn?**
   Refresqué de forma práctica cómo los Arrays bidimensionales operan en JS: modificar `row[index]` muta la Data original de inmediato porque Array de Arrays sostiene referencias puras en memoria. Cero necesidad de reconstrucción pesada.
2. **What would you change about the process?**
   Exigir un test unitario nativo Jest local. `SpreadsheetApp` no existe en Node, pero se pudo haber hecho un Mock de la interfaz `Sheet.getDataRange().getValues()` para correr el pipeline S5.2 en Terminal instantáneamente antes del deploy.
3. **Are there improvements for the framework?**
   El *Adapter_Sheets* debe incorporar una matriz de *Mocking* o un servidor test local con Data Falsa, ya que probar ETL's destructivos empujándolos a DEV puede romper toda la instancia de la DB si algo sale mal.
4. **What are you more capable of now?**
   Programar lógica ETL 100% aislada de Google Apps Script con latencia minúscula para re-arquitecturas pesadas sin depender del interfaz humano.

## Patterns & Calibration
- **Pattern Reinforced:** *Atomic Migrations (No Loops)*. Nunca invocar a Sheets API dentro de un `for`. Todo batch.
- **Pattern Added:** *Defensive Ghost Handling*. `parentId.toString().trim() !== "NULL"` para evadir los vicios inyectados manualmente por humanos en Sheets.
- **Calibration (S5.2):** Size M re-calibrado a S. El diseño del algoritmo previo fue tan claro que la ejecución fluyó muy rápido.
