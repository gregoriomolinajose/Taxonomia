# User Story: S5.2

## Connextra
**Como** Administrador del Sistema
**Quiero** ejecutar un script ETL in-memory sobre la hoja de Dominios
**Para** trasladar automáticamente las relaciones planas Padre-Hijo hacia aristas independientes Temporal-DAG en `Relacion_Dominios` y purgar la columna vieja de manera atómica sin generar downtime en producción.

## Gherkin ACs
```gherkin
Feature: ETL de Migración de Grafos (Flat-to-DAG)

  Scenario: Transformación a Tabla Puente (SCD-2)
    Given la hoja `Dominios` empadronada con la columna `id_dominio_padre`
    When el ETL en Apps Script invoca la función temporal de migración
    Then el script debe instanciar un array en RAM aislando a todos los nodos con padres válidos
    And asentar la relación `id_nodo_padre` e `id_nodo_hijo` en `Relacion_Dominios` con la fecha `valido_desde` como la fecha de creación original del Dominio
    And marcar explícitamente `valido_hasta` como un string vacío `""`
    And erradicar la columna `id_dominio_padre` original sobre-escribiéndola con string vacío `""`
```
