# Story Design: S5.2 - Script ETL de Migración Estructural

## 1. What & Why
**Problem:** Actualmente los árboles de control dependen de una columna estricta anclada dentro de la base de datos `Dominios` (`id_dominio_padre`). La modificación a Grafos Temporales nos dejó con metadatos obsoletos, y no podemos editarlos a mano.
**Value:** Construiremos un script determinista One-Shot de *Google Apps Script (ETL_Runner.gs)* para consumir toda la plana en milisegundos y parir el nuevo libro transversal histórico sin alterar los datos remanentes.

## 2. Approach
- **Script Standalone:** Inyectaremos el archivo `src/ETL_Runner.gs`. No contaminaremos `Adapter_Sheets.js` dado que esta función se borrará en Sprint futuros.
- **Extract (`getValues()`):** Cargamos la hoja "Dominios" ubicando las columnas `id_dominio_padre`, `id_dominio` y `created_at`.
- **Transform (Generador SCD-2):**
  - Si `id_padre` !== `""`, generamos UUID (`RELA-XXXXX`).
  - Llenamos la fila del Grafo: `id_nodo_padre`, `id_nodo_hijo`, "Militar_Directa", "1", `creacion_original`, `""`, `TRUE`.
  - Destruimos: `fila_dominio[indexPadre] = ""`.
- **Load (`setValues()`):** Creamos hoja `Relacion_Dominios` si no hay. Y volcamos tanto Relaciones Nuevas como Dominios Mutados en dos comandos `setValues()`.

*Falla de Integración (PAT-E-523):* ¿Qué pasa si falta el ID de creación inicial (`created_at`) en `Dominio`?
**Strategy:** Fallback a `new Date().toISOString()`. Nadie falla silenciosamente.

## 3. Concrete Examples
### Firma de Función Pseudo-Código:
```javascript
function ETL_Pivot_E5() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheetDom = ss.getSheetByName("Dominios");
  const data = sheetDom.getDataRange().getValues();
  
  // Headers and parsing
  const headers = data[0]; 
  const pIdx = headers.indexOf("id_dominio_padre");
  const cIdx = headers.indexOf("created_at");
  const dIdx = headers.indexOf("id_dominio");
  
  let relaciones = [["id_relacion", "id_nodo_padre", "id_nodo_hijo", "tipo_relacion", "peso_influencia", "valido_desde", "valido_hasta", "es_version_actual", "created_at", "created_by", "updated_at", "updated_by"]];
  
  for(let i=1; i<data.length; i++) {
     let row = data[i];
     if (row[pIdx]) {
         // Create relation
         const rID = "RELA-" + Utilities.getUuid().substring(0,5).toUpperCase();
         const startDate = row[cIdx] ? row[cIdx] : new Date().toISOString();
         relaciones.push([rID, row[pIdx], row[dIdx], "Militar_Directa", 1, startDate, "", true, new Date().toISOString(), "ETL_SYSTEM", "", ""]);
         // Remove legacy parent
         row[pIdx] = "";
     }
  }
  
  // Bulks
  sheetDom.getRange(1, 1, data.length, data[0].length).setValues(data);
  // CREATE Relacion_Dominios and inject...
}
```

## 4. Acceptance Criteria
- **MUST:** Ejecución completa con menos de 2 llamadas a `setValues()`.
- **MUST:** Ignorar filas cuyo `id_padre` ya sea `""` o nulo.
- **MUST:** Forzar la palabra clave `""` (string vacío) en las coordenadas `valido_hasta` de la nueva tabla.
