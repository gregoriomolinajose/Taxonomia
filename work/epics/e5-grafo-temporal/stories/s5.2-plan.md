# Story Plan: S5.2 - Script ETL de Migración Estructural Transaccional

## Overview
- **Story:** S5.2
- **Epic:** E5 - Grafo Temporal Multi-Estructural
- **Size:** M (Medium - Lógica crítica en Servidor)
- **File Targeting:** `src/ETL_Runner.gs` (Nuevo)

## Atomic Tasks

### 1. Inicialización del Motor Standalone y Extracción (TDD: Red -> Green)
- **Descripción:** Crear `src/ETL_Runner.gs` (archivo limpio). Declarar función `ETL_Pivot_E5()`. Instanciar la lógica de conexión `SpreadsheetApp` apuntando al libro ID central. Extraer masivamente `Dominios` en variable constante in-memory.
- **Archivo:** `src/ETL_Runner.gs`
- **AC Reference:** "El script debe instanciar un array en RAM aislando a todos los nodos".
- **Dependencias:** Ninguna.

### 2. Lógica de Paridad SCD-2 y Mutación Cero (Transform)
- **Descripción:** Mapear los índices dinámicos del `Header Row` (id_dominio_padre, id_dominio, created_at). Recorrer la matrix saltando cabeceras. Si existe `id_padre` válido: 
  - Instanciar array de arista `[UUID, Padre, Hijo, Tipo...]` aplicando *fallback* `toISOString()` si falla la fecha de origen.
  - Sobre-escribir in-memory la matriz original insertando un string defensivo `""` en el índice del padre obsoleto.
- **Archivo:** `src/ETL_Runner.gs`
- **AC Reference:** "marcar explícitamente válido_hasta como un string vacío y vaciar celda original".
- **Dependencias:** Tarea 1.

### 3. Operación de Load y Ejecución Bulk O(1)
- **Descripción:** Incorporar bloque condicional `insertSheet` para crear `Relacion_Dominios` si es inexistente e inyectar el Array Header en la primera fila. Ejecutar dos comandos universales `getRange().setValues()`: Uno sobre `Relacion_Dominios` (Aristas nuevas) y otro sobre `Dominios` (Matriz destrozada sobre-escrita).
- **Archivo:** `src/ETL_Runner.gs`
- **AC Reference:** Todas las escrituras puramente en Batch sin loops de I/O.
- **Dependencias:** Tarea 2.

### 4. Integration Check (Run from Apps Script)
- **Descripción:** Ejecución de prueba controlada de `ETL_Pivot_E5()` en el entorno de desarrollo y validación manual auditando el Spreadsheet de pruebas en la hoja destino.
- **Archivo:** GSheet Dev DB.
- **Dependencias:** Tarea 3.

## Execution Order
El archivo al ser *one-shot* se escribirá de forma secuencial descendente (Tareas 1 a 3). 
*Nota de Riesgo (Risk-First):* La estructura generadora de UUID temporal es el mayor riesgo, por lo que la tarea 2 encapsula toda la complejidad.
