# Scope S5.1

## Objective
Registrar la nueva entidad puente `Relacion_Dominios` dentro de `JS_Schemas_Config.html` y deshabilitar el obsoleto padre plano en `Dominio`, preparando el SSOT para inyecciones transaccionales.

## In Scope
- Creación de objecto `Relacion_Dominios` con: `idField: "id_relacion"`, `titleField: "id_relacion"`.
- Definición de "fields" para:
  - `id_nodo_padre` (type: select, lookupSource: getDominioOptions)
  - `id_nodo_hijo` (type: select, lookupSource: getDominioOptions)
  - `tipo_relacion` (type: select, ['Militar_Directa', 'Matricial_Secundaria', 'Colaborativa_Red'])
  - `peso_influencia` (type: number)
  - `valido_desde` (type: ms-date)
  - `valido_hasta` (type: ms-date)
  - `es_version_actual` (type: checkbox)
- Supresión física del campo `id_dominio_padre` del schema `Dominio`.

## Out of Scope
- Interfaz gráfica para manipular directamente este esquema en el DataView_UI (La manipulación es transaccional por el Backend).
- Inserción a hojas de cálculo reales (Esto es la Historia S5.2 - ETL).

## Done Criteria
- [ ] La variable `APP_SCHEMAS.Relacion_Dominios` compila correctamente.
- [ ] La constante `APP_SCHEMAS.Dominio` ha suprimido `id_dominio_padre` previniendo renders.
