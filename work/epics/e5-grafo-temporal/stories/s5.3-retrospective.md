## What Went Well
- **Intercepción Defensiva:** Ubicar el Setter Graph `_updateGraphEdges()` *dentro* del `orchestrateNestedSave` en vez de engrosar un Adapter Sheets mantuvo al Motor DB aislado.
- **Campos Virtuales Transitorios UI:** Agregar el flag conceptual `isTransientGraphEdge` a `id_nodo_padre` del `APP_SCHEMAS.Dominio` permitió que el FormEngine UI siguiese dibujando el Input select sin forzar la escritura de la columna en la BD física plana, quebrando el enlace duro Front-Disk.
- **Invalidación de Caché:** Integrar `_invalidateCache("Relacion_Dominios")` junto a la operación atómica garantiza que la próxima llamada a red de la vista Dataview obligue la reconstrucción de Topologías.

## What to Improve
- Aún lidiamos con un diseño de código sincrónico sobre Google Apps Script (`Utilities.getUuid()`, `new Date()`) que no es localmente testeable fuera del propio motor de V8 Sheets. Sería óptimo migrar estos Hooks topológicos a puros *Pure Functions* enviando el array de Edge en vez de forzar a `_updateGraphEdges` abrir un Google API.

## Heutagogical Checkpoint
1. **What did you learn?**
   El paradigma de *Caja Negra* sobre UI Genérica: El Componente FrontEnd (`FormEngine`) solo debe preocuparse de mapear el objeto `APP_SCHEMAS` visible y lanzar un `postMessage`. El Backend es capaz de secuestrar llaves transitorias (`delete flatPayload.id_nodo_padre;`) antes de que el Adapters_Sheets cometa una inserción física defectuosa.
2. **What would you change about the process?**
   Faltaron *Unit Tests Jest* para la función principal `_updateGraphEdges`. Se deben preparar mockups del objeto JSON para automatizar este punto en la CI/CD sin depender de Deploy a Dev manual por Clasp.
3. **Are there improvements for the framework?**
   El Motor `Schema_Engine.gs` debería estandarizar la variable `transient: true` globalmente en UI Forms, permitiendo definir campos ficticios que no se envíen por API, con propósitos exclusivos de UX (ej. validaciones derivadas de password dobles, grafos...).
4. **What are you more capable of now?**
   Acoplar modelos de datos bidimensionales y jerarquías temporales asimétricas donde la capa de persistencia se fragmenta y difiere drásticamente de la estructura del payload Front-end.
