# User Story: S5.4

## Connextra
**Como** Administrador del Taxonomía Operativa
**Quiero** que al eliminar un Dominio, sus sub-dominios (Hijos) escalen automáticamente al Dominio Padre del nodo borrado (Abuelo)
**Para** preservar la integridad del árbol taxonómico (Grafo Temporal) sin crear nodos huérfanos ni generar cascadas de borrado destructivas (Mutilación de metadatos históricos).

## Gherkin ACs
```gherkin
Feature: API Universal Cascade Flattening (Tree Healing)

  Scenario: Eliminación de Nodo Intermedio
    Given un Grafo donde el DomA es padre de DomB, y DomB es padre de DomC
    When el usuario ordena la eliminación de DomB a través del `Engine_DB.delete()`
    Then el backend intercepta el borrado antes del volcado físico
    And cancela la Arista Histórica `DomA -> DomB` (valido_hasta = hoy, version_actual = false)
    And cancela la Arista Histórica `DomB -> DomC` (valido_hasta = hoy, version_actual = false)
    And forja una nueva Arista Activa `DomA -> DomC` transfiriendo el linaje taxonómico
    And finaliza ejecutando el borrado (o soft-delete) de DomB en la tabla estática `Dominios`
```
