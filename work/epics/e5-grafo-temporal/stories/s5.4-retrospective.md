# Retrospective: S5.4 - Algoritmo de Cascade Flattening 

## Overview
- **Story:** S5.4

- **Actual Size:** M (60 mins)

## What Went Well
- **Rendimiento Ultrarrápido:** Restringir estrictamente la manipulación de la hoja a un ciclo `getDataRange().getValues()` -> `mutación en RAM` -> `getRange().setValues()` evitó el antipatrón mortal de Google Apps Script (`appendRow` iterativo). Al ser O(1) de cara a la API y O(N) puro en JavaScript, el saneamiento temporal es imperceptible para el usuario final.
- **Detección Explicita de Raíz:** El consejo proactivo del Arquitecto previno un error catastrófico de inyección de nulls en la topología temporal (Edge Case donde un Dominio de Nivel 0 es eliminado). Al obviar la asignación de abuelo, los nietos son exitosamente promocionados a Nodos Raíz.

## What to Improve
- Aún se debe prever el caso de Borrados Masivos (Batch Delete). El `_flattenGraphNode` diseñado procesa id's uno a uno, lo cual, de gatillarse masivamente desde el frontend, dispararía el setValues X veces. En S5.5 se deberá estudiar si la API Universal permite selecciones múltiples de borrado o si está capado.

## Heutagogical Checkpoint
1. **What did you learn?**
   Las reglas arquitectónicas impuestas *a priori* (Bulk Ops Imperative) modifican drásticamente cómo una IA aborda el problema. En vez de depender del azúcar sintáctico orientado a arreglos (como `.push()` en la hoja), fuí forzado a lidiar con el vector puro e inyectar *Strings Vacíos* al final del vector dinámico para mantener la integridad de `data[0].length`.
2. **What would you change about the process?**
   El Edge Case Raíz documenta la falta de formalismo al tratar la Raíz como un tipo de nodo; para el sistema general, un nodo sin padre es de Nivel 0. Debemos formalizar `id_padre == ""` como un constructor estándar para todos los Niveles 0, evitando iteraciones fantasma.
3. **Are there improvements for the framework?**
   El orquestador `Engine_DB` debería inyectar de oficio un booleano `.isRoot` o un Flag Semántico a las entidades para prevenir bucles condicionales extraños.
4. **What are you more capable of now?**
   Programar transacciones tipo cascada puristas, que resuelven amputaciones de árboles topológicos interceptando un motor genérico y mutándolo a su estructura en memoria subyacente de altísima latencia.

## Patterns & Calibration
- **Pattern Reinforced:** *RAM Bulk Mutator Hook*. Todo Hook pre-transactional en Google Sheets *debe* ejecutarse 100% in-memory y retornar la matriz íntegra reescrita.
- **Pattern Added/Discovered:** *Topological Promotion*. Ante la ausencia de Abuelo, los hijos se auto-promueven al nivel jerárquico supremo automáticamente en memoria, protegiendo al árbol.
- **Calibration (S5.4):** Size L re-calibrada a M, puesto que la guía precisa del bulk-imperative aceleró de facto la redacción del código de saneamiento.
