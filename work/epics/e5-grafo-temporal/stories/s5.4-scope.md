# Scope S5.4

## Objective
Preservar la integridad referencial y prevenir grafos desconectados (islas) cuando un nodo intermedio es eliminado. El algoritmo debe redirigir dinámicamente todos los punteros hijos hacia el abuelo activo en un entorno de SCD Type 2 (Dimensiones Lentamente Cambiantes).

## In Scope
- Creación de un Hook o Interceptor de Borrados para `Engine_DB.js` o `Adapter_Sheets.js`.
- Algoritmo O(N) para buscar Abuelos (`id_nodo_padre` de la arista donde el nodo a borrar es el hijo).
- Algoritmo O(M) para buscar Nietos (`id_nodo_hijo` de la arista donde el nodo a borrar es el padre).
- Modificación masiva en `Relacion_Dominios` (Mutación y Anexión).
- Soft Delete o borrado físico del registro `Dominio` plano.

## Out of Scope
- Operaciones en la Interfaz Gráfica (La UI seguirá emitiendo un `API.delete("Dominio", id)` idéntico).
- Manejo de "Ciclos" (Se asume por diseño de UI que esto es un Árbol / DAG estricto Cíclico Inexistente).
- Borrados masivos de más de un dominio a la vez por iteración simple.

## Done Criteria
- [ ] Eliminar un nodo con un padre y dos hijos transfiere a ambos hijos al abuelo.
- [ ] Todas las aristas previas hacia el nodo eliminado quedan registradas históricamente (soft-delete histórico) cambiando su status a `es_version_actual = FALSE`.
