# User Story: S5.3

## Connextra
**Como** Analista del Formulario (DataView/FormEngine)
**Quiero** modificar el `Dominio Padre` de un registro usando la UI genérica
**Para** que el backend intercepte el cambio, invalide el vínculo anterior (SCD-2) y genere una nueva arista temporal, manteniendo la base intacta sin corromper la planitud estructural original.

## Gherkin ACs
```gherkin
Feature: API Universal DAG Setter

  Scenario: Inserción de Nueva Arista (Relación Nueva)
    Given un payload de actualización en `Engine_DB.js` que contiene `id_nodo_padre`
    When `upsertRecord()` procesa la entidad `Dominio`
    Then el backend separa `id_nodo_padre` del payload
    And inserta el payload limpio en la hoja `Dominios`
    And inserta una nueva fila en `Relacion_Dominios` vinculando al padre y al hijo con `es_version_actual = true`

  Scenario: Caducidad SCD-2 (Re-parenting)
    Given que el objeto Dominio ya poseía un `id_nodo_padre` previo
    When se envía un payload de actualización con un nuevo padre
    Then el backend intercepta el cambio
    And localiza la arista anterior en `Relacion_Dominios` donde `es_version_actual = true`
    And muta el campo `valido_hasta` inyectando la fecha actual y `es_version_actual = false`
    And crea la nueva arista activa en la tabla puente
```
