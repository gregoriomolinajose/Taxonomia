# Story Plan: S5.3 - Backend Setter DAG e Inserciones SCD-2

## Overview
- **Story:** S5.3
- **Epic:** E5 - Grafo Temporal Multi-Estructural
- **Size:** L (Large - Core System Refactor)
- **Targets:** `src/Engine_DB.js`, `src/Schema_Engine.gs`

## Atomic Tasks

### 1. Inyección de Campo Virtual Transitivo (Schema_Engine.gs)
- **Descripción:** Las interfaces renderizan desde `APP_SCHEMAS`. Al haber mutilado `id_dominio_padre` físicamente en S5.1, debemos re-introducir un campo funcional **exclusivo para la UI** bautizado `id_nodo_padre` en `APP_SCHEMAS.Dominio`. El FrontEnd lo enviará al Backend, y el Backend lo ignorará para `Dominios` pero lo usará para `Relacion_Dominios`.
- **Target:** `src/Schema_Engine.gs`

### 2. Implementación de Subrutina de Orquestación (`_updateGraphEdges`)
- **Descripción:** Crear una función auxiliar pura en `Engine_DB.js` (`_updateGraphEdges(sheetNameObj, primaryKeyVal, newParentId)`). La función abrirá `Relacion_Dominios`, buscará Aristas vigentes donde `id_nodo_hijo == primaryKeyVal` y `es_version_actual == true`. Si detecta cambio de Padre, mutará la arista histórica (`valido_hasta = sysDate`, `es_version_actual = false`) y *pusheará* la nueva arista activa. Volcará con `Adapter_Sheets.upsertBatch()`.
- **Target:** `src/Engine_DB.js`

### 3. Interceptor Transaccional en `orchestrateNestedSave`
- **Descripción:** Modificar el router de inserciones `orchestrateNestedSave` de `Engine_DB.js`. Si `entityName === "Dominio"` y el Payload contiene `id_nodo_padre`, aislar la variable dinámicamente (`const d_padre = flatPayload.id_nodo_padre; delete flatPayload.id_nodo_padre`). Posterior a la escritura del maestro (Paso B), invocar la subrutina topológica pasando la PK.
- **Target:** `src/Engine_DB.js`

### 4. Caché Invalidation y Prueba
- **Descripción:** Añadir purgado a `_invalidateCache("Relacion_Dominios")`. Correr `npm run deploy:dev` para empujar el backend y probar desde la UI insertando/modificando el dominio padre.
- **Target:** `src/Engine_DB.js`

## Execution Order
El TDD exigirá modificar el esquema primero (Task 1) para recuperar la UI. Luego, diseñar la sub-rutina de Grafos (Task 2) aisladamente antes de conectarla al nervio central (Task 3).
