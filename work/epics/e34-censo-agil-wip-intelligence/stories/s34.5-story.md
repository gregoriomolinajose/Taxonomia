# Story: S34.5 - Mapa de Capacidad E2E

## User Story
**COMO** Líder de Transformación o RTE
**QUIERO** visualizar el despliegue as-is completo del Portafolio organizado en swimlanes por Grupo de Productos y Equipos
**PARA** entender instantáneamente las capacidades dispuestas y reasignar esfuerzo operacional sin abrir múltiples detalles de nodo.

## Acceptance Criteria (Gherkin)

**Escenario 1: Representación Completa del DataStore en Swimlanes**
*DADO* que el usuario accede a la vista de "Mapa de Capacidad"
*CUANDO* el renderizador procesa los datos Cacheados del Store (`AppDataStore`)
*ENTONCES* la vista no realizará overfetching de la Base de Datos
*Y* proyectará una jerarquía visual `Grupo_Productos` -> `Equipo` -> `Persona`
*Y* clasificará a las Personas con base en su rol temporal dentro de la vista.

**Escenario 2: Adaptabilidad Móvil**
*DADO* que el usuario abre la vista "Mapa de Capacidad" en un dispositivo móvil
*CUANDO* se pintan las jerarquías
*ENTONCES* los swimlanes adoptan propiedades responsivas usando stack cards (CSS flex o grilla CSS) para permitir scrollear verticalmente o paneo horizontal sin romper la UI (viewport width <= 768px).
