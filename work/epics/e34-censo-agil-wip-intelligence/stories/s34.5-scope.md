# Scope: S34.5 - Mapa de Capacidad E2E

## In Scope
- Creación o habilitación de la vista `DataView` tipo "Visual UI" o Componente Custom (e.g. `UI_View_CapacityMap.html`) si no existe, o re-utilización de algún scaffold de Censo.
- Consumo estricto de `window.AppDataStore` (o su homólogo global de estado) para no lanzar llamadas `google.script.run`. Toda la data ya fue extraída proactivamente por el Bootstrapper del motor temporal.
- Representación en grid CSS bidimensional (Grupo -> Equipos -> Personas).
- Diseño Web Responsivo prioritario a la vista vertical de "cartas apiladas" para móvil.

## Out of Scope
- Interacción Drag-and-Drop (cambiar capacidad entre equipos de forma visual queda para el próximo release de re-asignación táctica activa).
- Edición In-Line dentro del Mapa. El mapa es Read-Only (redirigirá al detalle de nodo si se requiere editar).

## Done Criteria
- [ ] La UI pinta exitosamente el árbol de Portafolio `Grupos de Prod -> Equipos -> Miembros`.
- [ ] La renderización no dispara llamadas repetitivas/N+1 a Apps Script (`console` limpio de redundancia fetching).
- [ ] Renderizado responsivo para ancho <= 768px probado.
- [ ] Pruebas Playwright (E2E) aseguran que el componente del Mapa asume la carga y está visible bajo un ID específico.
