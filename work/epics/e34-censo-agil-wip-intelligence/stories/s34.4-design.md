# S34.4 Design: UI Grupos de Productos & Capacidad y Dashboard Fix

## Objetivo
Visualizar los Equipos asignados a un Grupo de Productos (entidad `Grupo_Productos`) mediado por una relación jerárquica temporal 1:N, y remediar el GAP-3 del Dashboard, el cual impide renderizar correctamente el gráfico Donut de capacidades de equipo debido a una lectura de datos obsoleta.

## Propuesta Arquitectónica (Schema & UI)

### 1. UI Reactiva de Grupo de Productos a Equipo (`Schema_Engine.gs`)
En `Grupo_Productos`, se inyectará una relación de subgrid transaccional que herede del mismo motor `uiBehavior: 'subgrid'`.
```javascript
{ 
    width: 12, 
    name: "equipos_asignados", 
    type: "relation", 
    relationType: "hijo", 
    targetEntity: "Equipo", 
    virtual: true,
    uiBehavior: "subgrid", 
    label: "Equipos Ágiles del Grupo (Opcional)", 
    isTemporalGraph: true, 
    graphEdgeType: "GRUPO_PRODUCTO_EQUIPO", 
    topologyCardinality: "1:N" 
}
```
Esto creará un panel inferior en la forma del Grupo de Producto, permitiendo vincular/desvincular equipos sin navegar al detalle de cada Equipo, favoreciendo una UX _Top-Down_.

### 2. Remediación de Dashboard UI (GAP-3) (`Dashboard_UI.html`)
Actualmente, el script de Dashboard intenta mapear `f.section === 'Capacidad'` en campos iterativos de `Equipo`, lo cual arroja arreglos vacíos (`dynamicFields` = 0) e interrumpe la inicialización de ApexCharts para esa sección.
- **Data Pivot:** Reordenaremos la visualización para evaluar `eq.metodologia`.
- **Estructura Data:** `{ Scrum: N, Kanban: Y, Híbrido: Z }`.
- **Inyección UI:** En `chartTopology` (ApexCharts) alimentaremos las `labels` con los Keynames y la `series` con el Count.
