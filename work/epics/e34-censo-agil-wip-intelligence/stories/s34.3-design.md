# S34.3 Design: Subgrid Limitado (Capacidad Operativa) & Roles 1:1

## Objetivo
Implementar la recomendación operativa Ágil: El Subgrid de "Personas Asignadas" representará exclusivamente la *Capacidad Operativa (Development Team)*. Los roles de liderazgo y orquestación se gobernarán mediante relaciones 1:1 (Singletons). Esto simplifica el modelo topológico y evita corrupciones lógicas (ej. un equipo con 3 Product Owners).

## Propuesta UI y Modelo de Datos (Schema_Engine.js)

### 1. Migración 1:1 para Liderazgo Ágil
En `Equipo.fields`, los campos huérfanos actuales de texto se convertirán a relaciones oficiales de grafo 1:1 (como `uiComponent: 'select_single'`):
```javascript
{ name: "scrum_master_id", type: "relation", relationType: "hijo", targetEntity: "Persona", label: "Scrum Master / Team Coach", isTemporalGraph: true, graphEdgeType: "EQUIPO_SM", uiComponent: "select_single", valueField: "email", labelField: "nombre", required: false, width: 6 },
{ name: "product_owner_id", type: "relation", relationType: "hijo", targetEntity: "Persona", label: "Product Owner", isTemporalGraph: true, graphEdgeType: "EQUIPO_PO", uiComponent: "select_single", valueField: "email", labelField: "nombre", required: false, width: 6 }
```

### 2. Filtro Duro en el Subgrid (Capacidad Base)
En `Equipo.fields` para `personas_asignadas`, inyectaremos propiedas declarativas (Zero Code):
```javascript
{ name: "personas_asignadas", type: "relation", relationType: "hijo", childSchema: "Persona", virtual: true, uiBehavior: "subgrid", allowedRoles: ["Developer", "Tester", "Tech Lead"], label: "Célula Base (Developers/Testers)" ... }
```
**Modificación Frontend** (`UI_SubgridBuilder.client.js`):
- Al popular el `lookupSource` (Opciones para vincular), leeremos si el campo contiene `field.allowedRoles`. 
- Si existe, el modal filtrerá las opciones para arrojar **solo** Personas cuyo `rol_agil` coincida con la lista permiitda. (Requerirá que `FormRenderer` hidrate el string del `rol_agil` en las tuplas lookup o lo consultemos del `DataStore.get('Persona')` local directo en `UI_SubgridBuilder`).
