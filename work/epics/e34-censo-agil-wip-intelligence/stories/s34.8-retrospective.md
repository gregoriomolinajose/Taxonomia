# S34.8: Estabilización de Capacidad, Grafo y Desacoplamiento Schema-Driven

## Resumen
- **Fecha:** 2026-04-13
- **Épica:** E34
- **Objetivo:** Resolver carreras al hidratar dependencias de subgrilla recién creadas, ajustar detalles de layout del Mapa de Capacidad y avanzar en arquitectura Schema-Driven separando el acoplamiento a Formularios.

## What went well
1. Se diagnosticó con rapidez el defecto en `JS_Core` relativo a `inGraph` vs `inFallback` donde registros optimistas no aparecían debido a latencias asíncronas de base de datos. Se resolvió elegantemente hibridando ambas mecánicas reactivas mediante validación estricta de PKs dinámicas.
2. Logramos sobrellevar las pesadas restricciones de maquetación inyectadas por los componentes heredados de Ionic (`ion-card-header`) en el layout descendiendo a bloques `div` crudos permitiendo modelado Flex asimétrico puro.
3. Se superó formalmente con grado "A" la auditoría de Arquitectura Desacoplada (ShuHaRi) moviendo lógicas troncales fuera de la jurisdicción local de `UI_FormUtils`, instaurando centralmente un `Schema_Utils`.
4. Todos los ajustes culminaron empujados a DEV y PROD sin incidentes en el CI/CD (`v1.2.8`).

## What to improve
- Algunos comandos nativos de RaiCLI fallan temporalmente si Powershell es re-codificado incorrectamente con codificaciones puras ASCII (Character Map `\u2713` bug en despliegues automatizados) que abortan prints visuales. No afecta los flujos lógicos, pero requiere mitigación local de Terminal.
- Múltiples tests locales con *Playwright* colisionan ruidosamente cuando los corre *Jest* nativo por una falta de segregación de folders. Debemos alinear `.jest.config.js` para ignorar `/e2e/`.

## Heutagogical Checkpoint
- **¿Qué aprendimos?** El Shadow DOM de Ionic V6+ requiere un wrapper sin clases estandarizadas si queremos flexbox verdaderamente nativo controlando `align-items`. Y, el tipado defensivo ante integraciones cruzadas es una salvación silente.
- **¿Qué cambiar en el proceso?** Adoptar `typeof entityName === 'string'` como cláusula estándar en firmas topológicas para amortiguar datos crudos desde Grid DataSets.
- **¿Capacidad actual?** Ahora el ecosistema Reactivo de `DataStore` puede hidratar nodos dinámicos sin obligar un refresh global, al unísono con el validador Schema-Driven, marcando un avance estructural masivo sobre las iteraciones de la E29.

## Patterns Added
- **Technical - UI/Ionic:** (ADDED) Bypass Ionic Shadow DOM block constraints in complex UI widgets by using raw div flexbox containers instead of `ion-card-header`.
- **Architecture - Schema-Driven:** Dependencias de validación de modelos deben estar aisladas (`Schema_Utils`) totalmente agnósticas de vistas o controles UI (como Modales/Dominios/Grids).
