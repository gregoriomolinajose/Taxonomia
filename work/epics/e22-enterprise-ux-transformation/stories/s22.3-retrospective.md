# Retrospectiva S22.2 y S22.3: Omnibar y Limpieza UI

**Fechas:** Abril 2026
**Estimado vs Real:** ~2hrs / ~1.5hrs (Alta eficiencia técnica gracias a E19/S19.1)

## Resumen Ejecutivo
Construimos un motor de búsqueda global de 0 latencias (Omnibar) reciclando la precarga del grafo temporal y eliminamos ruido visual, fortaleciendo el estándar Enterprise SaaS de la aplicación.

## Heutagogical Checkpoint (Reflexiones)

**1. What did you learn? (¿Qué aprendimos?)**
Aprendimos que el patrón de delegación total hacia la metadata del backend (`APP_SCHEMAS.metadata.showInMenu`) simplifica dramáticamente el desarrollo del Frontend. Lograr que la lógica comercial provenga del diccionario central evita tener condicionales quemados (`if name == 'sys_roles'`) en componentes web como el Omnibar.

**2. What would you change about the process? (¿Qué cambiaríamos del proceso?)**
Antes de sugerir refactorizaciones locales dentro de Componentes UI, el escrutinio de diseño (`rai-architecture-review`) siempre debe inspeccionar la *"Fuente Única de Verdad"* (SSoT) para confirmar si el "hack" no es síntoma de un metadato erróneo remoto.

**3. Are there improvements for the framework? (Mejoras al marco de trabajo)**
Añadir a la matriz de Quality Review una heurística que escanee literales en duro de entidades (ej. strings tipo `"Sys_Permissions"`) dentro de los archivos de vista (`.html`).

**4. What are you more capable of now? (Nuevas capacidades adquiridas)**
Ahora somos capaces de construir UI Components ultrarrápidos (0 milisegundos de espera de red) apoyándonos exclusivamente en el `__APP_CACHE__` inyectado en el Bootstrap asíncrono. La plataforma está arquitectónicamente lista para consumir este modo local en otras vistas como el Dashboard Contextual (S22.4).

## Patrones Incorporados
- **Zero-Latency Search:** (Architecture) Invocación de arreglos en memoria temporal del cliente evadiendo latencias RPC hacia el Backend GAS.
- **SSoT Flags:** (Codebase) Mutar las reglas de dominio centralizadas (`Schema_Engine.gs`) es preferible al código imperativo parcheado en la vista (Design Purity).
