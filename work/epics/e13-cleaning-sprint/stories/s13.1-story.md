# Story: S13.1 - Subgrid Extractor

## User Story
**Como** Arquitecto del Sistema Frontend
**Quiero** extraer la lógica de renderizado de Subgrids (Relaciones M:N) fuera del FormRenderer_UI
**Para** reducir la complejidad ciclomática del inyector principal y prevenir regresiones cuando se modifiquen los formularios simples.

## Acceptance Criteria (Gherkin)

**Scenario 1:** Extracción limpia sin pérdida de eventos
*Given* que tengo un formulario con un Subgrid activo (ej. Células de un Sector)
*When* renderizo el formulario usando el nuevo `UI_SubgridBuilder`
*Then* el Subgrid debe dibujar los items anidados con sus tooltips correctos.
*And* las acciones de eliminar/agregar items anidados deben seguir reflejándose en la UI principal.

**Scenario 2:** Independencia del FormRenderer
*Given* el código fuente de `FormRenderer_UI.html`
*When* busco la función original `renderSubgrid`
*Then* no debe existir dentro del archivo.
*And* la clase debe rondar las < 800 líneas.
