# Design S7.1: UI Dynamic Component Mapping

## The Problem
Actualmente el formulario renderiza los campos en base a tipos como `text` o `number`. Las relaciones tipo Grafo transaccional SCD-2 no tienen un renderizador UI de primer nivel, dejándolas invisibles en el formulario normal (requiriendo Subgrids complejos o vistas forzadas).

## The Solution
Modificaremos el parser en `FormEngine_UI.html` para interceptar `f.type === 'relation'` y leer el tag `f.uiComponent`. 
- Si el tag es `select_single`: se empuja un combo de jerarquía clásica.
- Si el tag es `searchable_multi`: se empuja un `<ion-select multiple="true">` rellenado al instante mediante la hidratación sincrónica con caché (`window.__APP_CACHE__[f.targetEntity]`).

Este enfoque "Metadata-Driven UI" permite que cualquier entidad futura pueda adquirir selects transaccionales multi-parent con solo retocar el `Schema_Engine.gs`.
