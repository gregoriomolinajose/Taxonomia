# Design: S15.3 Component Architecture Decoupling (M)

## Problem Statement
`FormRenderer_UI.html` actualmente viola el principio de Única Responsabilidad (SRP) de S.O.L.I.D. manteniendo un bloque de control masivo de 160 líneas (`switch(field.type)`) en su iteración principal de campos. Aunque en la Epic 14 logramos encapsular los constructores de cada tipo de input en el namespace `UI_Factory`, el orquestador principal aún contiene toda la lógica de qué constructor usar, cómo manejar casos de grafos temporales, aserciones `branch-shifting` y pre-populado dinámico de relaciones M:N e inyecciones de dependencias. 

Adicionalmente, hay referencias obsoletas en otros lugares que intentan usar `window.renderForm` como fallback de Subgrids.

## Objective
Convertir el `FormRenderer_UI.html` en un motor puramente orquestador de layout (Grid layout) y metadatos. Toda la delegación y la elección de qué UI Node crear será transferida a la factoría oficial `UI_Factory` a través de un único contrato canónico: `UI_Factory.buildFieldNode(...)`.

## Architecture Paradigm
Alineado con el Blueprint V4 (Declarative Components):
1. **Delegación Total (Zero FormRenderer Knowledge):** El `FormRenderer_UI` simplemente pedirá: "Dame el Nodo DOM para el field X".
2. **Encapsulamiento de Lógica de Negocio (Graph interceptors):** Toda validación de cambio jerárquico (`Branch Shift Interceptor`), evaluación de `Temporal Graph Parents`, etc. pertenece al dominio del `UI_Factory` al momento de inicializar un elemento `relation`. 

## Implementation Details

### 1. Extracción de `buildFieldNode` en `FormBuilder_Inputs.html`
Agregaremos una nueva función estática a la clase de factoría:
```javascript
global.UI_Factory.buildFieldNode = function(field, entityName, data, localEventBus) {
    // switch (field.type) { ... }
    // Entire mapping logic block extracted from FormRenderer_UI.html
    return domElement;
};
```

### 2. Reducción en `FormRenderer_UI.html`
Las líneas `352-512` de `FormRenderer_UI` se reducirán a:
```javascript
const inputEl = global.UI_Factory.buildFieldNode(field, entityName, data, LocalEventBus);
```
El objeto `inputEl` se añadirá limpiamente al `ionCol`.

### 3. Eliminar fallbacks de Subgrid dependientes
Existen referencias *hardcoded* residuales como `window.renderForm` en Subgrids o validaciones anacrónicas que serán sustituidas por Eventos Pub/Sub o simplemente borradas si corresponden a código inalcanzable (Dead Code).

## Integration Considerations
Esta refactorización toca el núcleo central del renderizado de formularios. Debemos asegurar que todos los parámetros requeridos por las lógicas complejas de relación (como `global.currentEditId` o la hidratación de datos `data`) sean pasados limpiamente hacia el scope de la factoría.
