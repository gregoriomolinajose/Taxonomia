# Design: S15.1 Pipeline Validation & Bootstrapping Stability

## Approach
This story resolves three structural stability risks: Deploy pipeline silences, WSOD lack of mitigation, and False-Positive Truthiness on empty strings.
We won't overhaul the entire build step to a bundler (Rollup/Esbuild) per K.I.S.S., but we will integrate `acorn` merely as an AST validation gate in `deploy.js`.
In the frontend, we add a passive `window.onerror` in `Index.html` to catch exceptions during template rendering, removing the invisible native DOM and showing a styled panic box. Lastly, we apply `.trim()` on `UI_FormSubmitter.html` payloads.

### Affected Components
- `deploy.js` (Modify: Add acorn AST parsing).
- `src/Index.html` (Modify: Add `<template id="wsod-mitigator">` and `window.onerror` script).
- `src/UI_FormSubmitter.html` (Modify: Add `.trim()` inside `validateRequiredFields`).

## Examples
### 1. AST Validation Fail
```javascript
// A syntax error inside a script
const x = "broken string
console.log(x);
```
**Expected Output on `node deploy.js`:**
```
Pipeline Blocked: SyntaxError: Unterminated string constant (1:10) in FormRenderer_UI.html
```
*Build fails, clasp push is never reached.*

### 2. WSOD Mitigation
When `window.renderForm()` throws an unhandled exception before the components mount.
**Expected UI State:**
A red box indicating: *"Error Severo SPA: Unexpected token. Refresca la ventana o contacta a soporte."*

### 3. Truthiness Trap
```javascript
let input = "   ";
// Older behavior: input != null is TRUE. Submits spaces to Backend.
// New behavior:
if (input.trim() === "") throw new Error("Field is required");
```

## Acceptance Criteria
- AC1: `deploy.js` fails deterministically when ANY processed HTML file contains invalid JS.
- AC2: The WSOD fallback UI correctly escapes error details to prevent self-XSS when rendering.
- AC3: Emitting `"  "` (blank spaces) string values is reliably rejected by the Submitter module before hitting GAS backend.
