# Story: S15.1 - Pipeline Validation & Bootstrapping Stability

**As a** System Architect,
**I want to** validate JS syntax strictly before deployment and mitigate frontend crashes gracefully,
**So that** invalid code is never sent to Apps Script and users see a controlled error screen instead of a raw White Screen of Death (WSOD).

## Setup
- `deploy.js` script used to deploy to clasp.
- `Index.html` serves as the entry point.
- `UI_FormSubmitter.html` manages data validation to backend.

## Acceptance Criteria
- AC1: `deploy.js` parses the AST of script tags using `acorn.parse`.
- AC2: If `deploy.js` detects a syntax error, it throws and stops execution prior to deployment.
- AC3: `Index.html` contains `window.onerror` that catches runtime failures and replaces DOM with an Error Block UI.
- AC4: `UI_FormSubmitter.html` performs `String(val).trim()` before checking for truthiness.
