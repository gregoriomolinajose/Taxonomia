# Story S20.3: ES6 ThemeManager Evolution

## User Story

**As a** frontend architect  
**I want to** encapsulate the Theme CSS hydrator inside an orchestrated Module Pattern (`window.ThemeManager`)  
**So that** the application boots predictably without rogue procedural scripts executing randomly outside of the `DOMContentLoaded` lifecycle hook.

## Acceptance Criteria

### Theme Orchestration
- **Given** the file `UI_ThemeManager.html`
- **When** the code is evaluated
- **Then** it should not execute any logic immediately (no procedural `hydrateThemeConfig(...)` at the root).
- **And** it must export a `window.ThemeManager` object with an `init()` payload.

### Lifecycle Hook Connection
- **Given** the global bootstrapper in `JS_Core.html`
- **When** `DOMContentLoaded` fires
- **Then** it cleanly invokes `window.ThemeManager.init()` which in turn executes the internal hydrator.
