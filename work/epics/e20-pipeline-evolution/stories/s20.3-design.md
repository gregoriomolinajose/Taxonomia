# Story Design: S20.3 - ES6 ThemeManager Evolution

## 1. What & Why
- **Problem**: En S11 sentamos la regla UI §14, cargando y aplanando en O(1) los CSS Vars. Sin embargo, en `UI_ThemeManager.html` escribimos código ejecutable "al vuelo" fuera de todo hook, es decir procedural, lo que interfiere con los eventos y expone variables temporales. 
- **Value**: Estructurarlo siguiendo el Patrón de Módulo (Module Pattern) blinda su espacio de memoria. En su lugar exportaremos `window.ThemeManager = { init: ... }`, uniendo este módulo a la llamada bootstrapper asíncrona de `JS_Core.html` (línea 171: `if (window.ThemeManager) window.ThemeManager.init();`).

## 2. Approach
- **Refactor `UI_ThemeManager.html`** [MODIFY]:
  - Quitar el prefijo `window.__THEME_TOKENS__` global y encapsularlo dentro de `ThemeManager.tokens`.
  - Crear el objeto orquestador `window.ThemeManager = { ... }`.
  - Incorporar la función `flattenObject` u `hydrateThemeConfig` como un atributo privado o método `_hydrate()`.
  - Implementar un método `init()` que mandará llamar internamente a la auto-hidratación de los tokens.
  - Eliminar por completo la invocación a nivel raíz `window.hydrateThemeConfig(window.__THEME_TOKENS__);`.

## 3. Concrete Examples

### Bootstrapper Synchronization
```javascript
window.ThemeManager = {
  tokens: { color: { ... } },
  
  _hydrate: function(tokens) {
     // Aplanamiento O(1)
  },

  cycleTheme: function() {
     // placeholder (si no reside aqui)
  },

  init: function() {
    this._hydrate(this.tokens);
    console.log('[ThemeManager] Initialized successfully');
  }
};
// Notice: no "this.init()" or "window.ThemeManager.init()" floating around. 
// It waits strictly for the DOMContentLoaded.
```

## 4. Acceptance Criteria
- **MUST**: Extirpar variables sueltas como `window.__THEME_TOKENS__`.
- **MUST**: `JS_Core.html` inicia silenciosamente `ThemeManager` una vez el DOM termine su Parse Tree.
- **MUST**: La UI mantendrá sus colores originales sin romperse.
