# Story Plan: S20.3 - ES6 ThemeManager Evolution

## Overview
- **Story ID:** S20.3
- **Size:** S (2 SP)
- **Goal:** Aislar y transicionar la lógica de Theming hacia un Módulo Orquestado (Module Pattern), anulando scripts interpretados bloqueantes globales.

## Tasks

### T1. Refactorización de ThemeManager (`UI_ThemeManager.html`)
- **Descripción:** Transformar el archivo procedural en una clase estática / módulo `window.ThemeManager`.
- **Archivos:** `src/UI_ThemeManager.html`
- **Cambios a aplicar:**
  1. Absorber `__THEME_TOKENS__` hacia `ThemeManager.tokens`.
  2. Traducir global `hydrateThemeConfig(tokens)` a `ThemeManager._hydrate(tokens)`.
  3. Establecer `ThemeManager.init()` que invoque `_hydrate`.
  4. Suprimir completamente las invocaciones procedurales libres.
- **Verificación:** Revisar el HTML renderizado (o la sintaxis JS) para certificar que el archivo solo declara la constante estructural sin llamar ejecutores hasta que Bootstrapper lo pide.
- **Dependencias:** Ninguna cruzada (JS_Core ya lo llama orgánicamente en la linea 171).

## Execution Order
1. **T1** (Reescritura de `UI_ThemeManager.html`).

## Risks & Mitigations
- **Riesgo:** Si el Bootstrapper no invoca `init()`, la plataforma cargará sin estilos CSS prefigurados. Sin embargo, validamos anteriormente que el motor *sí invoca* explícitamente `window.ThemeManager.init()`. Eliminaremos los restos huella global.
