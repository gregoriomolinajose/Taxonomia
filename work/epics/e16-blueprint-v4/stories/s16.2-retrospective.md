# S16.2 Retrospective: Action Router Purification

**Dates:** 01-Apr-2026
**Story ID:** S16.2
**Epic ID:** E16 Blueprint V4

## What Went Well
- **Seamless Segregation:** The extraction of `API_Universal.gs` into `Controller_Action.gs` and `Controller_Lookups.gs` was flawlessly retro-compatible because of Google Apps Script's Global Namespace concatenation feature. The SPA required zero refactoring for these routing endpoints.
- **Rule of 3 Application:** During architecture review, we successfully identified and collapsed five duplicate `get[Entity]Options` generators into a single, highly robust `getGenericOptions()` abstractor.

## What to Improve
- We caught the architectural duplication *after* initial implementation during the `rai-architecture-review`. Identifying repetitive boilerplate mapping functions during the `rai-story-design` phase saves execution cycles.

## Heutagogical Checkpoint
1. **What did you learn?**  
   We learned that relying on dynamic `undefined` truthiness evaluation (`opt.value && ...`) for filtering can create silent semantic bugs downstream if IDs are `0` or numeric. The use of strict null/undefined checks is mandatory for relational Primary Keys.
2. **What would you change about the process?**  
   When performing extraction of a God-class, automatically extract duplicate chunks into generic handlers on the fly rather than verbatim copying them.
3. **Are there improvements for the framework?**  
   `APP_SCHEMAS.metadata` should proactively dictate the target `value` and `label` fields explicitly so `getGenericOptions(entity)` doesn't have to statically guess mapping conventions `nombre` vs `nombre_producto` in the backend. 
4. **What are you more capable of now?**  
   The application routing layer is mathematically separated from the data layer. We can test `API_Universal_Router` mocks effortlessly without dragging the heavyweight `Engine_DB` transactions to memory.

## Improvements Applied
- Patched `.filter` logic truthiness trap.
- Wrapped `SpreadsheetApp.openById` against invalid IDs explicitly.
- Centralized Data Bindings using `.map()` with `getGenericOptions`.
