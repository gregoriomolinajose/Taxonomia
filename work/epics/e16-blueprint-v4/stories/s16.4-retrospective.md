# S16.4 Retrospective: Inversion of Control en FormFactory

## Verification
- **Architecture Flow:** FormBuilder_Inputs consolidado con delegación (IoC) al remover el Switch statement macro.
- **Failures Found:** Trasladar la carga cognitiva de un switch duro a un mapa dinámico levantó un Issue transaccional en campos anidados (Relaciones Dependientes de otro Field). La resolución obligó a re-evaluar asíncronamente el Dependency-Injection via `UI_Factory`.

## Learnings
1. **Config Over Code:** Extraer dependencias pesadas hacia el Schema permite iterar la UI muchísimo más rápido. Las validaciones pueden testearse de manera agnóstica sin requerir inyectar DOM.
2. **Open-Closed Principle:** Añadir un Input Tipo "Date-Range" ahora no requiere tocar el Core Engine, basta adherirlo al Registro.

## Process Quality Review
- El framework RaiSE demandó justificar el diseño previo en la etapa de Arquitectura. Resultó beneficioso porque detectamos que aislar los validadores mitigaría fugas de memoria en cada Render (Regresión).
- **Tooling GAP:** El parser AST a veces marca "Undefined Factory" si el orden de los archivos `<?!= include(...) ?>` no garantiza que el core se cargue antes que las dependencias extendidas (Dependencia Cronológica). El Dependency Graph manual falló una vez; se resolvió escalando a Index Loadings.
