# Story: S16.2 Visibility Flags (FormRenderer Schema-Driven)

**As a** system architect
**I want to** control field visibility directly from the Schema Engine
**So that** the FormRenderer and FormBuilder don't require hardcoded business validation lists to determine what inputs to hide.

## Acceptance Criteria
- **Scenario:** Hiding internal fields declaratively
  - **Given** the Schema Engine explicitly marks a property with `uiDisplay: "hidden"` or `type: "hidden"`
  - **When** the `FormRenderer_UI` dynamically constructs the layout 
  - **Then** the component gracefully hides or skips the input node natively without relying on hardcoded arrays (e.g. ignoring `updated_at` manually instead of letting the SSOT govern it).
