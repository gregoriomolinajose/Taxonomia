# S16.1 Retrospective: UI Metadata SSOT Migration

**Dates:** 01-Apr-2026
**Story ID:** S16.1
**Epic ID:** E16 Blueprint V4

## What Went Well
- **Surgical Precision:** The migration from the hardcoded `Index.html` `window.ENTITY_META` array into the `APP_SCHEMAS.metadata` tree in `Schema_Engine.gs` was remarkably clean. No regressions occurred on the downstream SPA routing.
- **Architectural Refinement:** Leveraging `getEntitiesByMetaFlag` centralized the traversal logic away from `Dashboard_UI` and `UI_Router`, drastically reducing cyclomatic complexity.

## What to Improve
- We initially implemented an "Opt-out" filtering mechanism which we actively corrected to an "Opt-in" architecture mid-flight. For future declarative refactoring, **Always prioritize explicit Opt-In flags** for metadata to maintain bounded contexts securely. 
- Overlooked the redundant `.map` branch filtering in the downstream files before the Quality Review step manually caught it.

## Heutagogical Checkpoint
1. **What did you learn?**  
   We learned that relying on dynamic `undefined` evaluation over explicit schema mapping can lead to silent errors or "Muda". By asserting `=== true` strictly at the Iterator, we secured the UI topology against loose flags.

2. **What would you change about the process?**  
   When performing variable/flag extractions, hunt down **every consumer** of that block in a single search pass to prune immediately rather than relying purely on the Linter/Quality Review at the end.

3. **Are there improvements for the framework?**  
   The `Schema_Engine.gs` file is increasingly taking pure infrastructural responsibility for the Client Side. We could consider splitting pure definitions from logic schemas if the file hits 1000 LOC.

4. **What are you more capable of now?**  
   The application's sidebar and dashboard are now 100% configuration-driven. Adding new domain primitives is as easy as an insert in `Schema_Engine.gs`, empowering operations without front-end deployments.

## Improvements Applied
- Patched dynamic `NaN` sort vulnerability using Fall-through defaults `(a[1].order || 999)`.
- Replaced Opt-out with Opt-in visibility filtering explicitly requiring truthiness.

## Patterns
- **Added:** `PAT-G-024` (Architecture) Opt-In over Opt-Out Convention: Assumed-true creates structural technical debt for declarative configuration objects like ui metadata. Assume hidden/false unless explicitly stated to prevent invisible entities leaking into views.
