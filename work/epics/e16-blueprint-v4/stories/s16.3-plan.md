# S16.3 Implementation Plan
**Story:** S16.3 DataGrid Minimalism & Layout Extraction
**Size:** L
**Date:** 01-Apr-2026

## Overview
Decompose `DataView_UI.html` by extracting mathematical search, sort, pagination, and heavy CSV parsing logic into a new pure structural file `DataEngine_UI.html`.

## Tasks

### Task 1: Create `DataEngine_UI.html`
- **Description:** Abstract core array processing and IO operations.
- **Files:** `[NEW] src/DataEngine_UI.html`
- **Implementation:** Move `_applyFilter`, `_applySort`, `_exportCSV`, `_importCSV` out of `DataViewEngine` and inject them into a new singleton `window.DataEngine_UI`.

### Task 2: Refactor `DataView_UI.html`
- **Description:** Clean up `DataView_UI.html` orchestrator to become stateless.
- **Files:** `[MODIFY] src/DataView_UI.html`
- **Implementation:**
  - Remove previously identified mathematical functions.
  - Refactor `_onSearch`, `_onSort`, `_exportCSV` handler clicks to delegate the actual work to `DataEngine_UI.html`.
  - Pass the current state (`_state.data`, `_state.filtered`) by reference or state arguments to the engine.

### Task 3: API_Engine Inject
- **Description:** Expose the file in the GAS Web App pipeline.
- **Files:** `[MODIFY] src/Index.html`
- **Implementation:** Append `<?!= include('DataEngine_UI'); ?>` directly before `DataView_UI` is included.

### Task 4: E2E Integration Verification
- **Description:** Automation verification.
- **Verification:** `node deploy.js dev`

## Execution Order
Sequential: Task 1 -> Task 2 -> Task 3 -> Task 4. 
High Risk Structural change.
