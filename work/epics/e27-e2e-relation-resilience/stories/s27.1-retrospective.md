# Story Review: S27.1 - E2E Relation Resilience (Subgrid Optimization)

## Summary
- **Story IDs**: S27.1 (Taxonomia E2E Testing & Subgrid Resilience)
- **Status**: Completed (Tests Passing: `npx playwright test` = 100% Green)
- **Epic**: E27-e2E-Relation-Resilience

## What Went Well
- Logramos identificar rápidamente que la falla no era un error de red sino una limitación arquitectónica de hidratación en tiempo cero. 
- La arquitectura Publish/Subscribe del `AppEventBus` demostró ser completamente escalable.
- Integrar Playwright con Logs directos del navegador nos dio gran visibilidad sobre errores silenciosos (Truthiness traps y Undefined states).

## What to Improve
- Debemos evitar asignar funciones _callbacks_ de persistencia (como fallbacks estacionales) atadas incondicionalmente a las propiedades DOM (`__onSaveSuccessFallback`) para evadir el "spooky action at a distance".

## Heutagogical Checkpoint
1. **What did you learn?** 
   Aprendí que para desacoplar de verdad las vistas jerárquicas como Master-Detail (Subgrids), las actualizaciones optimistas no deben asumir que el backend retornará todo el Record re-hidratado.
2. **What would you change about the process?** 
   Nunca inyectar propiedades arbitrarías mutantes en el DOM; utilizar los eventos estándar estandarizados. 
3. **Are there improvements for the framework?** 
   A nivel framework de E2E, documentar en las variables `.env` las rutas quemadas como `.auth/chrome-profile` ayudará a la portabilidad inter-desarrolladores.

## Patterns Added/Reinforced
- **Pattern**: "Use Native CustomEvents over monkey-patching DOM properties for cross-component lifecycle communication"
- **Pattern**: "When executing optimistic UI updates, merge the local Request Payload with the Response ID instead of relying on the backend to return full objects."
