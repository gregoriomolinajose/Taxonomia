# S27.4 Story Design: Hybrid Topological UX

## What & Why
**Problem:** Actualmente la transición al modelo "Scroll Vertical" mató permanentemente a `UI_FormStepper.client.js` provocando una deuda topológica si se quieren visualizar formularios súper masivos paginados.
**Value:** Al instaurar una lógica dual (híbrida), las capas de la empresa tienen flexibilidad total usando `section` y `uiBehavior: "linear"` para definir si sus usuarios disfrutarán de paginación o scroll continuo, todo desde un único motor robusto que previene fatiga visual.

## Approach
Reinsertar bloque de instanciación del `window.UI_FormStepper` únicamente bajo la compuerta `steps.length > 1`.
Además, introducir un interruptor principal a nivel `APP_SCHEMAS[entity].metadata.uiBehavior === 'linear'`, de esta forma si el esquema exige no usar Wizard (Stepper), las n-secciones mapean directamente a Grid-Rows lineales cayendo en el comportamiento Scroll nativo, coexistiendo pacíficamente con elementos divisores custom.

## Examples
### Ejemplo 1: Múltiples Secciones (Stepper Default)
```javascript
// Schema de Entidad
{
  fields: [
    { section: "Persona", type: "text", name: "nombres" },
    { section: "Cargo", type: "text", name: "puesto" }
  ]
}
// Render Output:
// => Drawer dibuja 2 links laterales bajo el Stepper. 
// => "Guardar" se esconde y "Siguiente" permite cambiar.
```

### Ejemplo 2: Múltiples Secciones con Override Linear
```javascript
// Schema de Entidad
{
  metadata: { uiBehavior: "linear" },
  fields: [
    { section: "Persona", type: "text", name: "nombres" },
    { section: "Cargo", type: "text", name: "puesto" }
  ]
}
// Render Output:
// => Drawer dibuja TODO a la vez. No hay Sidebar de Stepper.
// => Se inyecta auto-visor con "Persona", input "nombres".
// => Inyecta auto-divisor con "Cargo", input "puesto".
// => Sticky footer posee [ Guardar ] de golpe.
```

### Ejemplo 3: Inyección Local de Divisores 
```javascript
// Schema de Entidad
{
  fields: [
    { section: "Persona", type: "text", name: "nombres" },
    { section: "Persona", type: "divider", label: "Finanzas" },
    { section: "Persona", type: "text", name: "cuenta" }
  ]
}
// Render Output:
// => Solo existe 1 sección (Persona). Por tanto modo LINEAR DEFAULT.
// => "nombres" aparece. Una barra horizontal nativa de ion-grid con H2 "Finanzas" divide.
// => Finalmente el input "cuenta". 
// => Si hubiese existido Stepper activo en otras zonas, esta sección "Persona" habría contenido a estos 3 de todos modos sin quebrar la paginación.
```
