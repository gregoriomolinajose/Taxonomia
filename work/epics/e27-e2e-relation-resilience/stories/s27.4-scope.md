# S27.4 Scope Boundaries

## In Scope
- Refactorización de `FormRenderer_UI.client.js` para alternancia contextual.
- Integración funcional de divisores topológicos (`type: "divider"`) en configuraciones multi-paso y pasos lineales.
- Modificación del algoritmo para contar numéricamente la métrica topológica (omitiendo indefinidos o nulos) para decidir el Branch del Renderer (`>= 2` is Stepper, `< 2` is Linear).
- Actualización de `__tests__/e2e/hierarchy-stress.spec.js` para asegurar que las rutinas Playwright se adapten transparentemente si hay Stepper o Scroll Linear al momento de registrar componentes complejos o de múltiples "vistas" requeridos.

## Out of Scope
- Refactorización del propio motor de la UI legacy en `UI_FormStepper.client.js` (se reutilizará el componente actual sin agregarle nuevas responsabilidades formales extra).
- Alteraciones en DataViews UI o Tableros; estamos encapsulados únicamente en el dominio de Form Rendering.

## Done Criteria
- `FormRenderer_UI` dibuja la entidad Unidad_Negocio como una UI totalmente Linear Scroll asumiéndose 1 sola sección válida por default, con un simple "Guardar".
- `FormRenderer_UI` dibuja satisfactoriamente el Wizard UI Stepper de un Form asociado a 2 secciones reales o más de su respectiva topología, introduciendo Sidebar interactivo y la secuencia de navegación ("Atrás", "Siguiente").
- `__tests__/e2e/hierarchy-stress.spec.js` finalizan en OK sin dependencias fragiles ni colisiones a la hora de buscar modales y submits dinámicos.
