# Retrospectiva: S37.5 (Módulo Bulk ETL)

## Resumen Ejecutivo
Se implementó de manera exitosa el **Motor Extract, Transform, Load (ETL)** para ingerir archivos CSV y convertirlos en carga transaccional hacia la Base de Datos subyacente. Esto evita la inserción manual 1 a 1 de cientos de registros en catálogos y directorios.

## Decisiones Específicas
1. **Lotes Resilientes (Client-Side Chunking):** Rompimos el paradigma clásico "Sube todo al Server" delegando la división de memoria al navegador (V8 Engine). Esto destruyó de raíz la muralla de limitación de Tiempo de Cómputo de 6 Minutos que padecía Infraestructura. Al mandar de 50 en 50, logramos simular una cascada asíncrona robusta.
2. **Defensa de Parseo CSV (Lookahead Regex):** Se fabricó un Parser desde cero usando expresiones regulares inteligentes para extraer Strings entre comillas (Ej: `"Apple, Inc"`), ignorando las comas internas que usualmente rompen otros scripts de importación de bajo presupuesto.
3. **Graceful UI Reactivity:** Integramos un Hack-Fix microscópico sobre los WebComponents StencilJS de Ionic (añadiendo retardos de redibujo `setTimeout(()=>{}, 50)`) garantizando que los mensajes de progreso del ETL (`Procesando Lote 8 de 14`) resuenen sin detonar "Race Conditions".
4. **Idempotencia Transaccional:** El Backend Router delega todo a `upsertBatch`. Si hay IDs que ya existen, se actualizan. Si un registro carece de ID, generamos un UUID pre-impacto.

## Criterios de Aceptación Logrados
- [x] Interfaz de Carga CSV (File Input Oculto asíncrono)
- [x] Parser Seguro Cero-Dependencias (DataEngine_ETL.client.js)
- [x] Ruteo Backend (BulkInsert proxy -> upsertBatch)
- [x] QA: Trampas Truthiness Sorteadas, Fugas de Memoria Nullificadas, Re-Renders Suprimidos.

## Reflexión Final
El uso de SRP (*Single-Responsibility Principle*) al extraer la pesadilla del parser hacia su propio módulo salvando a la pobre Toolbar de colapsar, pasará a la historia como una gran victoria Arquitectónica iterativa en RaiSE.
