# S37.6 Design: Desacoplamiento del Drawer Header

## 1. Patrón Arquitectónico
La creación de la cabecera interactiva de los menús laterales (Drawer / FormRenderer) ha ido acumulando heurísticas y lógica estructural hardcodeada (líneas 90 a 240 en `src/FormRenderer_UI.client.js`). Propongo extraer esta carga DOM hacia el Patrón Factory, creando un componente presentacional e inyectable, eliminando dependencias locales e instancias implícitas.

## 2. API Contract: `UI_Factory.buildDrawerHeader(config)`
El componente tomará un objeto config puro sin dependencias de estado ocultas:
```javascript
{
    entityName: 'string', // Nombre de entidad en esquema
    data: 'object', // Data base para el título y Avatar
    localEditId: 'string', // Label o Badge local en UI
    onClose: 'function', // Promesa iterativa
    actions: 'array' // [{ icon, title, callback }]
}
```

## 3. Manejo de Abstracciones (Singletons vs Módulos)
En los archivos de cliente del framework `.client.js`, el estado de la UI suele anidarse en Namespace Globales (e.g. `window.UI_FormUtils`, `window.UI_DataGrid`).
Continuaremos con este modelo de gobernanza para el cliente por medio de la asignación `window.UI_Factory = {}`.

## 4. Reemplazo del Código
En `FormRenderer_UI.client.js`, unas 100 líneas serán sustituidas por:
```javascript
const headerInfo = window.UI_Factory.buildDrawerHeader({
    entityName: entityName,
    data: data,
    localEditId: localEditId,
    onClose: () => {
        if (window.AppEventBus) window.AppEventBus.publish('MODAL::CLOSE_REQUEST');
        else if (window._closeTopModal) window._closeTopModal();
    }
});
```
