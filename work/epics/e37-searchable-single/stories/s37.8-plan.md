# Story Plan S37.8: Spike Triage (Pruebas Legadas)

## Resumen
**Tamaño:** S (3 SP)
**Fecha:** 2026-04-15
**Objetivo:** Destruir los 11 *Failures* arrojados por Vitest y estabilizar los Mocks.

## Tareas Atómicas

### Tarea 1: API Router (Test Mock Fix)
- **Modificar:** `__tests__/API_Universal_Router.test.js`
- **Descripción:** El test delega la llamada de borrar a un `Catalogo` de juguete enviándole el ID directamente. Debido al E29 Strict Primary Key, se asume 'id' por defecto, rompiendo la inyección. Se proveerá el esquema mock con su PK.

### Tarea 2: Gobernanza Topológica (Test Mock Fix)
- **Modificar:** `__tests__/Engine_DB_Orchestration.test.js` y `__tests__/MasterDetail_Regression.test.js`
- **Descripción:** Ambas pruebas operan sobre entidades imaginarias (ej. `PadreLevel`, `Grupo_Productos`). Inyectar `primaryKey: 'id'` explícitamente en la simulación de `getAppSchema` para que el Backend Analyzer no levante las Hard-Errors de Gobernanza.

### Tarea 3: UI Renderer (Test JS-DOM Fix)
- **Modificar:** `__tests__/FormEngine_UI.ui.test.js`
- **Descripción:** Replicar a nivel JS-DOM la provisión global estandarizada del generador semántico de Títulos (`window.Schema_Utils.getSemanticTitle`).

### Tarea 4: Adapter Sheets (Test Setup Fix)
- **Modificar:** `__tests__/Adapter_Sheets.list.test.js` y dependencias (podría ser el script global de pruebas si `SpreadsheetApp` se instanció ahí).
- **Descripción:** Prevenir el `Cannot read properties of undefined (reading 'setValue')` de `_ensureSheetExists`. Mockear `insertSheet()` para que el `getRange(1,X).setValue(...)` sea un *Dummy Callback* sin efectos.

## Criterios Término
Validado y corriendo con `npx vitest run` y obteniendo "PASS" general.
