# Story Plan S37.2: DataGrid Reactivity & UX

## Resumen
**Tamaño:** M (5 SP)
**Fecha:** 2026-04-15
**Dependencias:** Arquitectura S37.1 consolidada en UI.

## Tareas Atómicas

### Tarea 1: Racionalización de Columnas Grid
- **Modificar:** `src/DataView_UI.client.js`
- **Descripción:** Refactorizar la función constructora de cabeceras para que el ID Léxico (`Lexical_ID`) y el campo `nombre` ocupen forzosamente el lugar número 1 y 2 en el Grid. Remover duplicaciones de campos estéticos que estorban visualmente.
- **Verificación:** Navegar al Dataview de Producto y comprobar el orden.

### Tarea 2: Redibujado Encolado (DataGrid Reactivity)
- **Modificar:** `src/DataView_UI.client.js`
- **Descripción:** Añadir una variable global / local en `_isRedrawQueued`. Conectar un bus `AppEventBus.subscribe('DATA::UPDATED')` y en su callback implementar el bloqueo por `requestAnimationFrame` que consume `_rerenderData()`.
- **Verificación:** Trigger un guardado desde un modal hijo; el Grid exterior debe verse modificado sin apretar F5.

### Tarea 3: Transmutación Semántica (`_resolveLogicalValue`)
- **Modificar:** `src/UI_DataGrid.client.js`
- **Descripción:** Implementar una heurística limpia. Cuando reciba un campo de base de datos cuyo UUID pertenece a una llave foránea de relación (Ej. el portafolio asignado de un Dominio), no imprimir el UUID, sino consumir internamente `DataStore.getTargetTitle()` devolviendo un string presentable.
- **Verificación:** UI debe mostrar "Portafolio Retail" en vez de ID alfanuméricos en la grilla.

## Orden de Ejecución y Justificación
1. Primero Tarea 3 (Transmutación) antes que nada. Ocultar UUIDs protege al usuario visualmente. 
2. Segundo Tarea 1, asegurándonos que luego de traducir todo, las cabeceras estén ordenadas por Importancia (ID y Nombre delante).
3. Por último Tarea 2 para darle vida al DataGrid inyectando la reactividad.

## Riesgos y Mitigaciones
- **Riesgo:** El `requestAnimationFrame` podría enmascarar un bug infinito (ciclo de update circular) si hay dependencias re-lanzando guardados.
- **Mitigación:** Asegurar que los componentes de capa de formulario solo lancen evento *después* y explícitamente tras consolidación HTTP OK desde el backend, y el Grid solo consuma sin mutar nada más.
