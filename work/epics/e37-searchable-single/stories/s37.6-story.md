# Story: S37.6 Desacoplamiento del Drawer Header como Componente Puro Reutilizable `UI_Factory` o Módulo Independiente

## 1. User Story
**As a** UI Developer (Taxonomia SPA)
**I want** to extract the repeated Drawer Header logic (title, badges, buttons, close logic) into a pure reusable component
**So that** all drawers (Forms, Subgrids, Previews) have a consistent look and feel without code duplication (H9).

## 2. Acceptance Criteria (Gherkin)
```gherkin
Feature: Centralized Drawer Header 

  Scenario: Renderizado de Layout Estándar
    Given que el sistema necesita renderizar un contexto dentro de un ion-drawer
    When se solicita `UI_Factory.buildDrawerHeader(config)`
    Then debe retornar un DOM node con el título alineado, el subtítulo si aplica, y el botón de cierre [X].

  Scenario: Inyección de Botones Accionables
    Given que el contexto requiere acciones adicionales
    When se pasan `actions` en la configuración del header
    Then deben renderizarse botones Atlassian-style en el extremo derecho previo al botón de cierre.

  Scenario: Auto-Theme y Dark Mode Compliance
    Given que el usuario está utilizando el tema oscuro
    When se invoca `buildDrawerHeader()`
    Then los íconos (ion-icon) y el subtítulo deben respetar la herencia de las variables CSS de ThemeManager.
```

## 3. Heutagogical Goals
- Master the abstraction of DOM node composition using pure function signatures.
- Reinforce DRY patterns across Vanilla JS Custom Elements.
