# Scope: E19 - Core Framework Resilience & Strictness

## Objective
Enhance the Taxonomia SPA architecture by injecting defensive programming patterns, enforcing strict evaluation throughout the core modules, and introducing safe boundaries.

## In Scope
- **Validator Truthiness**: Fixing silent bugs generated by `JSON.parse` swallows and falsy type coercions.
- **Ionic Promises**: Catching and appropriately awaiting Ionic Modal/Toast/Alert components.
- **AppEventBus Telemetry**: Building a lightweight publish-subscribe error and analytics logger mechanism.
- **Error Boundaries**: Refactoring global `try/catch` and `window.onerror` fallback UI.

## Out of Scope
- Major UI/UX redesigns.
- Implementing an external monitoring SaaS inside the Telemetry system (logs remain localized/console-first).

## Planned Stories
- **S19.1**: Truthiness & Strict Types Refactoring.
- **S19.2**: Defusing Floating Promises in UI components.
- **S19.3**: Implementing `AppEventBus` Telemetry.
- **S19.4**: Global Error Boundaries & Fallback Graceful Degradation.

## Done Criteria
- [x] No warnings about unhandled promises when interacting with UI.
- [x] Validations strictly test for undefined/null rather than generic truthiness.
- [x] Architecture passes the `rai-quality-review` successfully.
