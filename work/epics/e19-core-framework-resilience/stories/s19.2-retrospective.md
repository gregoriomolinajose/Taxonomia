# Story Retrospective: S19.2 (Defusing Floating Promises)

## 1. Context
- **Story:** S19.2
- **Objective:** Evitar que promesas caídas en métodos pre-construidos de Ionic se transformaran en UnhandledRejections letales para nuestra frontera defensiva (S19.4).

## 2. Review
- **Architecture Validation:** Passed. Creación exitosa de `window.PresentSafe(el)`.
- **Code Quality:** Uso pragmático de un Wrapper Factory en lugar de duplicar código en las capas visuales. Resolvimos el problema de compatibilidad preservando el `return` en la Promesa.
- **Risks Adverted:** Las recargas tardías de scripts en Modal y WebComponents ahora lanzan un WARN en Telemetría en lugar de tirar toda la estructura DOM mediante el mitigador WSOD (Fatal Fallback).

## 3. Learnings
- **Actionable Insight:** Cuando un framework de terceros (como Ionic) impone interfaces frágiles (ej. arrojar excepciones sin un default catch), siempre es más económico encapsular la API en un Gateway unificado en vez de educar a todas las capas superiores a usar `try-catch`.
