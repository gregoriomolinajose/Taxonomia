# S19.3: AppEventBus Telemetry Foundation

## User Story
Como Arquitecto de Sistema,
Quiero tener un bus de eventos publicador/suscriptor a nivel global (Telemetry)
Para poder registrar, canalizar y consultar trazas y flujos asíncronos y errores no bloqueantes (Silent Failures) sin degradar el rendimiento principal del navegador.

## Acceptance Criteria
- [ ] Existe un objeto global `window.Telemetry`.
- [ ] Posee el método de inyección de traza: `Telemetry.track(action_name, payload, level)`.
- [ ] No interrumpe la experiencia del usuario (async by default).
- [ ] Formatea todos los outputs hacia objetos JSON uniformes en la Consola `(console.info, console.warn, console.error)`
- [ ] El patrón arquitectónico debe permitir (en un futuro opcional) enviar estos logs por batch a `google.script.run` (Google Analytics / Sheets Logging).
