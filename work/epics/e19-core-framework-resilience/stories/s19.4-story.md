# S19.4: Global Error Boundaries & Graceful Degradation

## User Story
Como Arquitecto de Software,
Quiero interceptar la estrategia de mitigación "Fail-Fast WSOD" preexistente en `Index.html` y bifurcar sus datos,
Para que además de mostrar una pantalla limpia y amigable al usuario (previniendo pantalla en blanco), el sistema inyecte los traces de `window.onerror` asíncronamente en el nuevo `AppEventBus Telemetry` mitigando vulnerabilidades de ceguera técnica.

## Acceptance Criteria
- [ ] Reutilizar el template de mitigación pre-existente en `Index.html`.
- [ ] La función `triggerWSOD(errDetail)` debe inyectar de forma segura la telemetría evaluando primero `if (window.Telemetry) { Telemetry.track('FATAL_ERROR', errDetail, 'FATAL'); }`.
- [ ] No alterar las clases base de la UI preexistente para evitar roturas estilísticas.
