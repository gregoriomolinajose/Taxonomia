# S19.4: Global Error Boundaries & Graceful Degradation

## User Story
Como Usuario Final,
Quiero que si ocurre un error catastrófico de renderizado o una excepción asíncrona no manejada, el sistema me muestre un aviso amigable y se recupere (o bloquee de forma limpia),
Para no enfrentarme a la "Pantalla Blanca de la Muerte" y perder el contexto de mi sesión, mientras que internamente los administradores reciben el log del error.

## Acceptance Criteria
- [ ] Implementar interceptor `window.onerror` y `window.onunhandledrejection`.
- [ ] Las fallas deben rutar hacia `window.Telemetry.track('FATAL_ERROR', err, 'FATAL')`.
- [ ] Al detectar un error `FATAL`, la UI debe inyectar temporalmente un Modal/View de Fallback amigable (Ej. "Algo salió mal").
