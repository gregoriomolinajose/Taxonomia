# Scope: S19.4 (Global Error Boundaries - Architecture Reuse)

## Boundaries

### In Scope
- Hooking the existing `triggerWSOD` inside `Index.html` to logically report data to `window.Telemetry` (if instantiated).
- Validation of Error interceptor mapping.

### Out of Scope
- Building a new Error Boundary from scratch. (Violation of DRY, flagged by Architecture Review).
- Altering the CSS or HTML structure of the `wsod-mitigator` template.

## Done Criteria
- [ ] A synthetic error successfully fires the `Index.html` fallback UI while registering a `FATAL` tag inside the standard Dev Console Telemetry output route.
