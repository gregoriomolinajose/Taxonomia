# Scope: S19.4 (Global Error Boundaries)

## Boundaries

### In Scope
- Injectors in `JS_Core.html` for `window.onerror` and `window.addEventListener('unhandledrejection')`.
- Hooking those listeners securely to `Telemetry.track`.
- Creating a Fallback UI that takes over `div#app-container` upon Fatal Error rendering to show a warning icon and simple message.

### Out of Scope
- Granular React-style `<ErrorBoundary>` components per module. The fallback will be absolute full-page (App-level only).
- Advanced automated bug-reporting to Jira/Git.

## Done Criteria
- [ ] A synthetic error thrown intentionally from the developer console (e.g. `setTimeout(() => { throw new Error('Test') }, 100)`) successfully transforms the page into the Fallback UI instead of leaving corrupted DOM state.
