# Story Plan: S19.3 (AppEventBus Telemetry Foundation)

## Overview
- **Story ID:** S19.3
- **Size:** XS (1 SP)
- **Objective:** Generar el wrapper global `window.Telemetry` en el motor base que encripte la comunicación entre logs de consola y el EventBus.

## Atomic Tasks

| ID | Description | Targeted Files | Size | Deps |
|---|---|---|---|---|
| **T1** | Inyectar `window.Telemetry` Object | `src/JS_Core.html` | XS | None |
| **T2** | Vincular `AppEventBus.publish` | `src/JS_Core.html` | XS | T1 |
| **T3** | E2E Manual Test | Navegador | XS | T2 |

### Task 1: Inyectar `window.Telemetry`
Insertar bloque conceptual debajo de `window.AppEventBus` (aprox. línea 30).
**Acceptance Criteria/Verification:** `typeof window.Telemetry` devuelve `object`.

### Task 2: Implementar Log Routing (`track(action, payload, level)`)
Integrar el formateo de fecha (ISO), selección de severidad `(console.warn, console.error...)` e inyección de paquete hacia `AppEventBus.publish('TELEMETRY::TRACK', packet)`.  Asegurar que sea `async` (fire and forget) envolviéndolo en `setTimeout` o ejecutándolo sin bloqueos.
**Acceptance Criteria/Verification:** `Telemetry.track` no rompe el flujo actual de JS.

### Task 3: Manual Integration (Gemba)
Levantar SPA localmente o recargar Google Apps Script.
Detonar `Telemetry.track('TEST_DRIVE', { ok: true }, 'INFO')` en consola.
Validar impresión.

## Execution Order
Ejecución lineal T1 -> T2 -> T3 en un solo commit modular (ya que están en el mismo archivo).

## Risks & Mitigations
Si `window.AppEventBus` no existe (porque se mueve al final del archivo en una refactorización), el componente arrojará `undefined error`. Se mitigará añadiendo un chequeo seguro `if (window.AppEventBus)`.
