# S19.1: Truthiness Refactoring

## User Story
Como Arquitecto de Resiliencia,
Quiero reemplazar la validación y parsing numérico "falsy" en el sistema
Para garantizar que variables con valores significativos como `0` no sean falsamente bloqueadas ni omitidas por el intérprete.

## Acceptance Criteria
- [ ] Formularios de Alta / Modificación no son bloqueados por inputs con valores numéricos que resuelven falsy (como `"0"` o `0`).
- [ ] Expresiones matemáticas previenen usar `|| 0` cuando un `NaN` estricto es lo analizado.
