# S24.1: DataView UI MVP Refactor (Split State/Toolbar/Grid)

## Goal
Fractionalize the `DataView_UI` God Object (currently 763 LOC) into isolated components following standard Separation of Concerns: Model (State), View (Toolbar, Grid Builders), Presenter (DataViewEngine Core Controller).

## Expected Outcomes
- Creation of isolated files (e.g. `DataView_State.html`, `DataView_Toolbar.html`, `DataView_Grid.html`) while preserving the main `DataViewEngine` as an orchestrator.
- Maintain existing Zero-XSS purity.
- Ensure regression tests show identical behaviour for sorting, pagination, lazy-loading, and drag & drop.
