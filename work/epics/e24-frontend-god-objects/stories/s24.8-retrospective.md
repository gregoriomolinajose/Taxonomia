# Retrospective: S24.8 - DataGrid Card View OCP Fix

**Date:** 2026-04-04
**Epic:** E24 Frontend God Objects
**Estimated:** 20m | **Actual:** 180m

## Summary
The goal was to remove the static array `HIDDEN_BY_DEFAULT` from `UI_DataGrid.client.js` and obey the UI configuration generated by `DataView_UI` for the grid cards logic, complying with OCP and DRY principles.

## What went well / What to improve
**Well:**
- El refactor primario del God Object se completó rápido y sin afectar la tabla principal. La transición fue transparente.
- Construir un test de integración en Node con simulaciones directas de DOM ayudó bastante a certificar visualmente la lógica asincrónica de la factoría.

**To improve:**
- El testing mock del DOM interno retrasó significativamente la conclusión de la historia por un bug semántico oculto (uso de parámetros basura `Math.random()` como seteo de llaves HTML que reventaba la aserción y enmascarado tras silenciamiento).

## Heutagogical Checkpoint

1. **What did you learn?**
   Los entornos de renderizado como JSDOM son particularmente frágiles si existen discrepancias entre el código del navegador puro y la estructura local no pre-compilada. Dejar silenciamientos o simular un motor completo del DOM no es trivial y requiere honestidad en el contrato, así como cuidar variables crudas no declaradas de `config.page` que asume un API.

2. **What would you change about the process?**
   Me aseguraría de implementar siempre las aserciones semánticas dentro de la emulación desde el minuto cero, para atrapar errores nativos como  `InvalidCharacterError` en lugar de enmascararlos para acelerar la inicialización y que luego se manifiesten como Fragments sin hijos.

3. **Are there improvements for the framework?**
   Se debe repensar dónde se guarda `window.DOM.create` para que tanto el front (Google Apps Script) como Node puedan integrarlo naturalmente sin recurrir al Monkey Patching cada que se testea un God Object UI.

4. **What are you more capable of now?**
   Ahora tengo profunda comprensión práctica de cómo el motor DataGrid renderiza internamente DocumentFragments y sus implicaciones con la paginación interna. 

## Patterns

* **Added:** `process` -> "Validación preventiva del objeto configuración completo y sus implícitos (e.g. paginación) antes de inicializaciones manuales sobre motor de rendering."
* **Reinforced:** "Zero-Mocking Policy sobre Lógica Clave" (Votado +1). Mockeamos DOM nativo, pero NO mockeamos la lógica generatriz del Grid, y rindió frutos reales al exponer agujeros.
