# S24.7: UI Components Hardening

## What went well
1. Se dotó a los selectores híbridos múltiples (`SearchableMulti`) de una armadura de parceo escalonado (String -> JSON -> Array/Fallback). El renderizado ahora es pragmático; un valor anómalo del caché ya no detendrá el Main Thread del Frontend (*White Screen of Death*).
2. Tuvimos la agudeza estructural (durante la fase de Quality/Architecture Review) de reconocer un gravísimo anticuado "Shotgun Surgery Smell" donde `buildSearchableMulti` estaba inyectado dos veces en el codebase.

## What went wrong
1. El código del Framework Ionic se nos escapó de las manos y fue duplicado negligentemente en versiones anteriores por temor a romper dependencias asíncronas en `Index.html`. Fue más valioso corregir este diseño ahora (a pesar de estar técnicamente "fuera del scope" previsto de la historia) que seguir propagando el parche sintáctico doble.

## Learnings
* **Type Dishonesty en Cargas Lentas**: Cuando tratamos con entornos donde Apps Script parsea celdas a JSON indiscriminadamente, el Frontend debe operar bajo el principio de Zero-Trust a la hora de hidratar datos relacionales. No se puede confiar en un simple `Array.isArray()`.
* **DRY by Subtraction**: El mejor código para el cliente es el que no se envía. Eliminar 200 líneas de un `FormBuilder` saturado recorta instantáneamente un porcentaje sustancial de los tiempos de carga en dispositivos de gama baja.
