# S24.3: FormBuilder_Inputs Factory Atomization

## Goal
Desarmar el tercer God Object del Frontend: `FormBuilder_Inputs.html` (680 LOC). Este archivo fue diseñado como una "Factoría Pura", pero aloja constructores incrustados para web components excesivamente complejos como listas dinámicas (M:N primitivas) e interfaces de selección modal (Searchable Multi-Select).

## Expected Outcomes
- Separar componentes atómicos primitivos de los componentes complejos (Compuestos).
- Extraer `buildSearchableMulti` (generador de pop-ups de catálogo M:N) a su propio archivo/namespace.
- Extraer `buildDynamicList` a un componente de Layout específico.
- Extraer el `Relation Builder` (Reglas estáticas de Nivel y Topología para Aristas).
- El archivo `FormBuilder_Inputs.html` debe quedar como un mero router o "Builder Registry" para primitives (Texto, Número, Combo), bajando por debajo de las 200 líneas de código.
