# S24.2: FormRenderer Dependency & Node Extraction

## Goal
Fractionalize the `FormRenderer_UI` God Object (786 LOC) into modular utilities. The FormRenderer currently holds dense logic for:
1. Resolving Dropdown Lookups via GAS (`_resolveLookupSources`).
2. Attaching DOM Dependency Iterators (Zero-Touch UI `ionChange` listeners).
3. Overriding and Disabling UI blocks via ABAC.

## Expected Outcomes
- Extract Dependency Event Handlers to a pure `UI_FormDependencies` module.
- Move Data resolvers to an isolated fetching mechanism.
- Extract ABAC rendering lockouts to a generic `UI_FormUtils.applyReadOnly` pattern.
- Resulting `FormRenderer_UI` should sit comfortably below 400 LOC, focused solely on its DOM layout mechanics (`stepper`, `rows`, and `columns`).
