# S31.8 · Wiring de ensureProvisioned() en Adapter_Sheets

## Contexto

Durante el Architecture Review de E31 (heurística H1), se identificó que la función
`ensureProvisioned(entityName, ss)` en `Adapter_Sheets_Provisioner.gs` está definida,
documentada y testeada como punto de lazy-init para `Adapter_Sheets`, pero **ningún
call site real la invoca** en el código de producción.

La función fue diseñada para ser llamada antes del primer write de cualquier entidad,
garantizando que el schema físico (la hoja de Sheets) existe y está alineado con
`APP_SCHEMAS` antes de cualquier operación de escritura.

## Story Statement

**Como** desarrollador de plataforma,  
**quiero** que `Adapter_Sheets.upsert()` llame automáticamente a `ensureProvisioned()`
antes de ejecutar el primer write de cualquier entidad,  
**para que** el schema físico en Sheets se valide y repare automáticamente sin intervención
manual, completando el contrato de la función lazy-init.

## Criterios de Aceptación

- [ ] `Adapter_Sheets.upsert()` invoca `ensureProvisioned(entityName)` en la primera
  escritura por entidad (lazy: cache para no re-invocar en cada write).
- [ ] El resultado de `ensureProvisioned()` se registra en Logger para trazabilidad.
- [ ] Si `ensureProvisioned()` falla, el error es capturado y logueado pero **no bloquea**
  el write (fail-soft para no romper operaciones existentes).
- [ ] Existe al menos un test unitario que verifica que `ensureProvisioned` es llamada
  antes de la primera escritura.
- [ ] Los tests pre-existentes de `Adapter_Sheets` pasan sin modificación.

## Notas de Diseño

### Cache de provisioning
Para evitar llamar a `ensureProvisioned()` en cada write, implementar un Set de entidades
ya provisionadas en el módulo:

```js
const _provisionedEntities = new Set();

function _ensureEntityReady(entityName) {
  if (_provisionedEntities.has(entityName)) return;
  const result = ensureProvisioned(entityName);
  Logger.log('[Adapter_Sheets] ensureProvisioned: ' + JSON.stringify(result));
  _provisionedEntities.add(entityName);
}
```

### Call site en upsert()
```js
function upsert(entityName, record) {
  _ensureEntityReady(entityName);  // ← new
  // ... existing upsert logic
}
```

### Fail-soft
El error de provisioning NO debe propagarse al caller de `upsert()`. La escritura
debe continuar aunque la reparación de esquema falle (best-effort).
