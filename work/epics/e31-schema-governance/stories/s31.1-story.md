# S31.1 — Auditoría de Duplicaciones en Schema_Engine

> **Epic:** E31 — Schema Engine Governance Layer
> **Branch:** `story/s31.1/auditoria-duplicaciones-schema`
> **Size:** S
> **Status:** In Progress

## User Story (Connextra)

**Como** miembro del equipo de desarrollo de Taxonomia,
**quiero** un inventario completo y cuantificado de todos los patrones duplicados en `Schema_Engine.gs`,
**para** tener el mapa de duplicaciones que sustente y guíe las extracciones de S31.2 y S31.3 sin especulación.

## Acceptance Criteria (Gherkin)

```gherkin
Feature: Auditoría de Duplicaciones en Schema_Engine

  Scenario: Inventario de topologyRules duplicados
    Given el archivo Schema_Engine.gs existe en src/
    When se analiza el bloque topologyRules de cada entidad
    Then se produce un reporte con:
      | Campo | Valor |
      | Número de entidades con topologyRules | N |
      | Número de presets únicos identificados | K |
      | Líneas potenciales a eliminar | L |
    And cada preset está nombrado con una clave candidata

  Scenario: Inventario de campos comunes duplicados
    Given el archivo Schema_Engine.gs existe en src/
    When se analiza el arreglo fields de cada entidad
    Then el reporte incluye:
      | Campos que aparecen en >1 entidad (nombre, declaración) |
      | Número de entidades que los comparten |
    And se proponen nombres de FIELD_TEMPLATES candidatos

  Scenario: Reporte verificable
    Given el reporte de auditoría está escrito
    When el desarrollador lo revisa
    Then puede confirmar que todos los patrones duplicados están capturados
    And el reporte incluye decisiones de diseño preliminares para S31.2 y S31.3
```

## Notes

- El output de esta story es **documental** — un reporte de auditoría en `stories/s31.1-audit-report.md`
- No se modifica código fuente en esta story
- El reporte alimenta directamente las decisiones de S31.2 (TOPOLOGY_PRESETS) y S31.3 (FIELD_TEMPLATES)
