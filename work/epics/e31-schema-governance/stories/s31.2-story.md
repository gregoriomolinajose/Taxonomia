# S31.2 — Catálogo TOPOLOGY_PRESETS

> **Epic:** E31 | **Branch:** `story/s31.2/topology-presets-catalog` | **Size:** M | **Status:** In Progress

## User Story

**Como** desarrollador de Taxonomia,
**quiero** que las reglas topológicas de las entidades estén definidas en un catálogo `TOPOLOGY_PRESETS` centralizado y nombrado,
**para** que agregar una nueva entidad solo requiera referenciar un preset por clave en lugar de redeclarar 9 líneas idénticas.

## Acceptance Criteria

```gherkin
Feature: TOPOLOGY_PRESETS catalog

  Scenario: Preset existente resuelve identico a inline
    Given TOPOLOGY_PRESETS.JERARQUICA_ESTRICTA_GRAPH_STD existe
    When se accede a APP_SCHEMAS.Portafolio.topologyRules
    Then el objeto resultante es profundamente igual al bloque inline original

  Scenario: Presets son inmutables
    Given TOPOLOGY_PRESETS esta congelado con Object.freeze
    When se intenta mutar una propiedad (preset.maxDepth = 99)
    Then la mutacion falla silenciosamente (strict mode) o lanza error
    And el valor original permanece intacto

  Scenario: Consumidores sin cambio de contrato
    Given Schema_Engine es cargado con TOPOLOGY_PRESETS
    When Engine_DB llama a getEntityTopologyRules("Portafolio")
    Then devuelve el mismo objeto que antes (retrocompatibilidad total)

  Scenario: Nueva entidad usa referencia
    Given TOPOLOGY_PRESETS.JERARQUICA_ESTRICTA_GRAPH_STD existe
    When se declara una nueva entidad con topologyRules: TOPOLOGY_PRESETS.JERARQUICA_ESTRICTA_GRAPH_STD
    Then el schema de la nueva entidad tiene topologyRules completo sin redeclarar propiedades
```
