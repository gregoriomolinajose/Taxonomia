# Scope: s2.1 UX Evolution - Sidebar de 3 Estados

## In Scope
- Creación de ciclo de estado en JavaScript (Full -> Mini -> Oculto) para el menú lateral en Desktop.
- Inyección de clases CSS personalizadas para manejar el ancho `72px` y ocultamiento de labels en el Estado Mini.
- Deshabilitación/manejo dinámico del componente `<ion-split-pane>` y `<ion-menu>` para el Estado Oculto.
- Emisión de un evento 'resize' sintético en JavaScript o lógica equivalente para forzar el redimensionamiento automático de ApexCharts.
- Garantizar animaciones CSS suaves durante las transiciones.
- Asegurarse de que el comportamiento en móvil (< 992px) permanezca intacto.

## Out of Scope
- Rediseño de los iconos del menú.
- Cambios en el diseño interno (colores, fondos) del dashboard más allá de lo necesario para el reflow.
- Cambios en el comportamiento del menú de formulario (wizard) a menos que comparta el mismo DOM y se beneficie gratis.

## Done Criteria
- [ ] Botón en el header alterna correctamente entre los 3 estados (Full, Mini, Hidden).
- [ ] En estado Mini, ancho es de ~72px, iconos centrados, labels ocultos.
- [ ] Transiciones son suaves mediante CSS.
- [ ] ApexCharts detectan el nuevo viewport y redibujan gracias al trigger de `resize`.
- [ ] En vista móvil todo sigue funcionando como un menú overlay nativo.
