# Design S30.2: Robustez en Muteo Estructural de Relacionales

## Contexto & Flujo (What & Why)
- **Problema**: El modo "Sólo Lectura" para registros vinculados (Subgrids) se lograba inyectando CSS destructivo (`pointer-events: none; opacity: 0.6`) sobre clases comodín (e.g., `ion-button[fill="clear"]`) usando el utilitario de seguridad `UI_FormUtils.applyReadOnlyLock()`. Aunque esto prevendría los clics, es una medida débil (bypassable desde las DevTools) y genera nodos DOM basura que no serán usados.
- **Valor**: La "Programación Estructural" determina que la UI no debería tan siquiera inyectar y bindear (event-listeners) controles mutables sí el usuario/entorno no tiene permiso. Removiendo los botones programáticamente cerramos vectores de manipulación y ahorramos memoria (V-DOM overhead).

## Componentes y Cambios
1. **`FormRenderer_UI.client.js`**
   - Adelantar la evaluación `ABAC.can('update', ...)` a la fase de pre-dibujado en `openEditForm` para inyectarlo como *prop-drilling* (`readonly: !canEdit`) a la sub-función `renderForm()`.
   - Modificar la firma de `renderForm(..., config={})` para propagar este flag paramétrico hacia el Builder del Subgrid.
2. **`UI_SubgridBuilder.client.js`**
   - Recibe la prop en `config` en su firma `build(..., config={})`.
   - Si `config.readonly === true`, *no inyectar* el nodo modal interactivo en memoria, *no hacer append* del `addBtn`, y en la construcción del loop the childRecords, *omitir* el `delBtn` por cada tupla.
3. **`UI_FormUtils.client.js`** *(Opcional)*
   - Verificar que el `applyReadOnlyLock` siga funcional para Text-Inputs regulares (esto es correcto) pero validando no sobrescribir estilos de los botones si de por sí ya no existen (seguro).

## Ejemplos & Contratos
**Firma Propuesta**:
```javascript
// FormRenderer_UI.client.js
const canEdit = !window.ABAC || window.ABAC.can('update', entityName, id);
await global.renderForm(entityName, record, null, { readonly: !canEdit });

// UI_SubgridBuilder.client.js
build: async function(field, container, data, entityName, localEventBus, modalContext, config = {}) {
    const isReadonly = config.readonly === true || field.readonly === true;
    ...
    if (!isReadonly) {
        header.appendChild(addBtn);
    }
    ...
}
```

## Criterios de Aceptación (AC)
- **MUST**: Extracción del chequeo de ABAC previo a la hidratación para condicionar la llamada de generación.
- **MUST**: Los botones para Borrar y Agregar Sub-records NO DEBEN EXISTIR en el DOM cuando la vista esté en Reader Mode.
