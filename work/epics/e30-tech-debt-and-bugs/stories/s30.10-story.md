# Story S30.10: Auditoría Universal IPC Deserialization

> **Epic:** E30 (Tech Debt & Bugs)
> **Status:** DONE
> **Assignee:** Antigravity

## Problem Statement
La validación previa de la recaída S30.9 descubrió un cuello de botella sistémico: el motor V8 de Google Apps Script falla silenciosamente ("dropping postMessage") al serializar objetos y colecciones (Arrays de Objetos) anidadas cuando se envía a través de `google.script.run`.

## Objective
Auditar y estandarizar sistemáticamente las respuestas HTTP/RPC de toda la suite de Controladores (como `Controller_Lookups.gs` y `Controller_Action.gs`) para que cualquier objeto complejo sea siempre pasado como un `String` nativo vía `JSON.stringify()`, mientras que el `DataAPI.client.js` captura asíncronamente y lo des-serializa.

## Technical Design
1. Auditar en `Controller_Lookups.gs` métodos devuelven arrays raw (`getDominioOptions`, `getPersonasOptions`, `getGenericOptions`).
2. Envolver el return de esos endpoints en `JSON.stringify(options)`.
3. Ajustar `DataAPI.client.js` para autorizar a Arrays (`[`).
4. Ajustar Caché Interna para parsear seguridades.
