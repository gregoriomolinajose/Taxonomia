# S30.3 Scope: DOM Pasivo a Estado Local en Componentes

## Objective
Erradicar la anti-patronal manipulación transitoria del DOM mediante *Hidden Inputs* que almacenan JSON *stringified* para mantener el estado de sub-estructuras (ej. `__subgrid_data_X`, `__relations_X`). Migrar la tenencia del estado efímero a estructuras de JavaScript estables (Local State via Closures o State Managers livianos) ancladas asíncronamente al ciclo de Submit.

## In Scope
- Auditar y remover `document.createElement('input')` con `type = "hidden"` que transporten objetos JSON en `UI_FormSubmitter.client.js`, `UI_SubgridBuilder.client.js` y `UI_Component_DynamicList.client.js`.
- Introducir o consolidar `SubgridState.client.js` como gestor de memoria base sin fuga de variables.
- Modificar el flujo de `UI_FormSubmitter` para extraer la hidratación cruzada desde variables o closures en vez de recolectar valores de nodos ocultos.

## Out of Scope
- Migración al framework React/Vue. Todo sigue siendo Vainilla JS mediante abstracciones puras.
- Refactorización de Formularios que NO utilicen campos dinámicos anidados (subgrids/dynamic lists/relations).

## Done Criteria
- Ningún _hidden input_ serializado se inyecta en el DOM de Taxonomia para guardar _state_ temporal de componentes complejos.
- Los subgrids, relation builders y dynamic lists conservan las modificaciones temporalmente en Local State Javascript.
- Todo se recolecta uniformemente por `UI_FormSubmitter` en el `buildPayload` antes de enviar al backend.
