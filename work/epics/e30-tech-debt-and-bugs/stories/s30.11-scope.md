# Scope S30.11: Protocolo de Extracción Nodal Frontend

## Boundaries

**In scope:**
- Modificar componentes clave desarrollados en S30.2 y S30.3 (`UI_Component_DynamicList`, `UI_Component_RelationBuilder`, `UI_Component_SearchableMulti`, `UI_SubgridBuilder`) para exponer uniformemente un bloque exportador `getValidatedValue()`.
- Refactorizar las rutinas de compilación final de carga útil en `UI_FormSubmitter.client.js`. Todo el casquete de IF-ELSE ad-hoc que revisa el Local State directamente debe reducirse un 70%.

**Out of scope:**
- Alterar la persistencia de bases de datos. Los WebComponents exportarán exclusivamente lo que el FormSubmitter inyecta a la tubería IPC hacia la DB. La DB en sí (Google Sheets) no sufre iteraciones aquí.
- Agregar nuevos WebComponents ni modificar la UI de los pre-existentes de cara al usuario final. Este refactor es un "Under the hood code-contract API" para el renderizador.

## Done Criteria
1. El archivo `UI_FormSubmitter.client.js` no cuenta con comprobaciones hardcodeadas de Clases DOM `(ej. if(el.dataset.type === 'multi_link'))`, y en cambio delega la responsabilidad ciegamente invocando algo genérico como `try { payload[name] = _element.getValidatedValue(); } catch...`.
2. Todos los componentes pesados (Subgrid, Searchables, DynamicLists) responden fielmente al Protocolo implementando en su objeto JavaScript nativo dicha subrutina de auto-parseo.
3. El despliegue a QA pasa exitosamente guardando formularios pesados sin alteraciones a su JSON payload nativo.
