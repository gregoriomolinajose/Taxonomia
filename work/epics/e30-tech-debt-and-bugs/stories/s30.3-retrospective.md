# S30.3 Retrospective

**Story:** S30.3 - DOM Pasivo a Estado Local en Componentes
**Epic:** E30
**Size:** M
**Dates:** 04-10-2026 / 04-10-2026

## Summary
La historia se propuso detener el anti-patrón de guardar complejos objetos JSON stringificados dentro de inputs hidden creados al aire. Logramos desterrar esa práctica migrando todos los datos anidados de Subgrids y Selectores M:N hacia un estado persistente temporal anclado al contenedor del DOM (`activeForm._LocalState` y `_LocalStateSelection`).

## What Went Well / What to Improve
**Well:**
- Logramos reemplazar todo el viejo flujo O(n) de JSON Parse en runtime hacia una inyección brutalmente veloz mediante `Object.assign`.
- Vitest respondió estupendamente a los Mock-Updates confirmando que el desacoplamiento FormSubmitter <-> Component Builders ha mejorado radicalmente.

**To Improve:**
- Identificamos un problema estructural (H10 Architecture Asymmetry): La forma en que `DynamicList` interactúa con `UI_FormSubmitter` para parsear sus colecciones anidadas es diferente al LocalState puro que usa `SubgridBuilder`. Por eso hemos creado la Historia **S30.11**.
- Aprendimos que inyectar referencias inmutables nativas presenta un riesgo menor de Memory Leaks; esto fue remediado usando limpiezas pre-asignadas en la propiedad del nodo `activeForm._LocalState = {}`.

## Heutagogical Checkpoint
1. **What did you learn?**
Aprendí que el Garbage Collector y el V8 des-optimizan los host objects en Chrome al usar `delete`; y que migrar de parseo a memoria de clausura requiere una defensa "Circular Reference" (`WeakSet` stringifier) en el payload builder.
2. **What would you change about the process?**
Evitar depender de parseo asíncrono para matrices grandes; deberíamos estandarizar la persistencia "Nodal" de forma nativa desde el inicio.
3. **Are there improvements for the framework?**
Vitest Mocks sufrieron contención por usar un nodo fantasma sin limpiar `innerHTML` en iteraciones sucesivas (`app-container`). Deberíamos asegurar que los reset hooks se limpien incondicionalmente en `afterEach`.
4. **What are you more capable of now?**
El manejo de Arquitectura Nodal de JavaScript está hiper blindado contra fallas cruzadas sin requerir dependencias React/Vue, demostrando escalabilidad Vanilla real.

## Patterns Added / Reinforced
- **[Pattern] Architecture**: Evitar JSON.stringify en inputs hiddenDOM para estructuras complejas; preferir inyectar data en `activeForm._LocalState` para un performance in-memory nativo.
- **[Pattern] Security**: Reforzar el Payload final de RPC con destructores cíclicos en `JSON.stringify (WeakSet Replacer)` para prevenir fallas V8 si la UI inyectara basura de Nodos Reactivos inmutables dentro del State Local.
