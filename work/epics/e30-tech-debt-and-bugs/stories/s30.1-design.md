# Design S30.1: Limpieza Segura de Assets Temporales

## Contexto & Flujo (What & Why)
- **Problema**: El script `deploy.js` elimina forzosamente la carpeta temporal `assets` del `.build` usando el File System (`fs.rmSync`). Manipular el FS pasivamente en un pipeline puede ocasionar colisiones temporales o condiciones de carrera (Race Conditions) con IDEs y watchers si se detiene el bundle antes de completarse, además de agregar lógica imperativa innecesaria en el build-step.
- **Valor**: Menos mutación del File System. Un pipeline de compilación debe idealmente orquestar (Build/Bundle) y delegar el filtrado a las herramientas encargadas del envío (Clasp). 

## Componentes y Cambios
1. **[MODIFY] `deploy.js`**
   - Eliminar el bloque de código que invoca `fs.rmSync(assetsCssDir...` en las líneas finales del script.
   - Dejar intacta la generación y lectura de `.css` que transforma `minified` para ser inyectadas como `<style>` components. Solo se elimina la "Limpieza de limpieza".
2. **[MODIFY] `.claspignore`**
   - Validar que incluya explícitamente rutas seguras para omitir archivos CSS crudos y temporales (ya incluye `**/*.css`, se reforzará con ignorar la carpeta `assets/`).

## Ejemplos & Contratos
**Antes**:
```javascript
// deploy.js cleanup (L148-153)
const assetsCssDir = `${buildDir}/assets/css`;
if (fs.existsSync(assetsCssDir)) {
    fs.rmSync(assetsCssDir, { recursive: true, force: true });
}
```
**Después**:
```javascript
// Eliminado el bloque de código de mutación.
// En su lugar, el .claspignore manejará la exclusión.
```

**.claspignore nuevo**:
```text
node_modules/
package.json
package-lock.json
jest.config.js
__tests__/
*.test.js
.antigravityrules
**/*.css
assets/
```

## Criterios de Aceptación (AC)
- **MUST**: `deploy.js` no ejecute `fs.rmSync` sobre `assets` en ningún momento luego del build.
- **MUST**: Clasp pueda pushear exitosamente a Google Server sin mostrar errores de `Unsupported File Type` por enviar CSS.
- **MUST**: El build conserve de todos modos los scripts encriptados dentro de `<style>`.
