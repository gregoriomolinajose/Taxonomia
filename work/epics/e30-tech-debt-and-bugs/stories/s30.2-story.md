# Story S30.2: Robustez en Muteo Estructural de Relacionales

## User Story
**Como** Arquitecto de Frontend
**Quiero** remover la dependencia de hacks CSS (`pointer-events: none`) para mutear subgrids cuando están en modo lectura
**Para** garantizar una restricción determinística a nivel JavaScript y prevenir brechas lógicas o exploits donde un usuario active interacciones ocultas.

## Acceptance Criteria
- Enlazar la configuración de inicialización del `UI_SubgridBuilder` de manera que exponga un *prop* o parámetro de estado paramétrico `readonly`.
- `FormRenderer_UI` u otros padres, cuando decidan mutear el formulario o el bloque relacional, deben pasar este parámetro determinístico en lugar de inyectar clases o IDs CSS "catch-all" que fuercen opacidad u ocultamiento.
- El SubgridBuilder debe anular programáticamente la inserción de botones de agregar y capacidades de edición cuando `readonly === true`.
