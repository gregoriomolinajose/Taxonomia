# Story Plan: S30.3 - DOM Pasivo a Estado Local en Componentes

**Ref:** S30.3
**Epic:** E30
**Size:** M (3-5 SP)

## Overview
Decompose the implementation to replace `input[type="hidden"]` data transfer with transient local memory (`_LocalState` attached to `activeForm`).

## Execution Order
The tasks will be executed sequentially to ensure core `FormSubmitter` compatibility is prepared before refactoring the sub-components that consume it.

### Task 1: Refactor `UI_FormSubmitter` to Cosechar Local State
- **Description:** Implement `Object.assign(payload, activeForm._LocalState)` logic when building payloads, skipping nodes that use data-parser. Drop fallback exceptions related to recursive parsing.
- **Files:** `src/UI_FormSubmitter.client.js`

### Task 2: Implement Closure Injection in `UI_SubgridBuilder`
- **Description:** Erase `document.createElement('input')` logic. Locate `activeForm` recursively and inject `childRecords` directly into `activeForm._LocalState[field.name]`.
- **Files:** `src/UI_SubgridBuilder.client.js`
- **Verification:** Unit UI testing passing `__tests__/UI_SubgridBuilder.ui.test.js --browser`.

### Task 3: Implement Closure Injection in `DynamicList` y `SearchableMulti`
- **Description:** Extend the same logic over to `UI_Component_DynamicList.client.js` and `UI_Component_SearchableMulti.client.js` to ensure uniform form component architecture. Eliminate all remaining `hidden` inputs.
- **Files:** 
  - `src/UI_Component_DynamicList.client.js`
  - `src/UI_Component_SearchableMulti.client.js`
- **Verification:** Carga de Formularios complejos sin crashes visuales ni fugas de selectores.

### Task 4: Manual E2E Playwright Verify (Integration Test)
- **Description:** Boot the platform locally and submit a form with a complex subgrid and dynamic list to ensure `UI_FormSubmitter` natively digests and posts data without regression.
- **Files:** Deployment + Frontend App
