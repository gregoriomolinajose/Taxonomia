# Scope S30.8: Lexical Identity & ID Fallback

## In Scope
- **Implicit ID Fallback:** Provisión de fallback si un Schema no tiene PK. Deberá autoinyectar una columna lógica `id` con un UUID puro sin prefijos.
- **Lexical Consecutive Identifier:** Crear el atributo arquitectónico virtual `lexical_id` (o el que derive el `name` visible) apoyado por un acrónimo prefijo (ej: `PORT`) concatenado con un contador consecutivo atómico por tabla (Ej: `PORT-1`).
- **UI Badge Identity:** Ampliar el framework UI para que dicho identificador Lexical (`PORT-1`) se dibuje de manera limpia y bloqueada en la cabecera (como Badge flotante) al abrir un registro en el frontend, en vez de un input o de ocultarse del humano.

## Out of Scope
- Migrar datos viejos borrados (los IDs consecutivos se formarán basados en el número máximo existente visible, o en el `getLastRow()`).
- Refactorizar flujos del DAG engine.

## Done Criteria
1. Creación de una entidad omite crash si no define `primaryKey`. 
2. Toda entidad cuenta un prefijo y numeración unívoca humana.
3. FormRenderer eleva la ficha visual en todos los formularios por igual.
