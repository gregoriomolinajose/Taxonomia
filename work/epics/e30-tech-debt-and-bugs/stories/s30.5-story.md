# Story: AST RegExp Modifiers (Deploy Pipeline)

Como Ingeniero DevSecOps de Taxonomia,
Quiero transicionar el pipeline de despliegue (`deploy.js`) de expresiones regulares manuales a *AST Parsers* completos de HTML/CSS/JS,
Para evitar fallas de compilación frágiles, fugas de minificación, o errores de parsing al empaquetar código para Google Apps Script.

## Context
Actualmente `deploy.js` utiliza RegExp (`/<script\b[^>]*>([\s\S]*?)<\/script>/gi`) para ubicar JS para acorn, o hace concatenaciones burdas. Las Expresiones Regulares son insuficientes para analizar jerarquías asíncronas de HTML o CSS variables de Ionic. 

## Acceptance Criteria
- `deploy.js` extirpa las RegExp encargadas de localizar Scripts y `<style>` tags en favor de un parseador AST de HTML.
- El minificado CSS no corrompe Ionic CSS variables ni rompe con pseudo elementos.
- El deploy script sigue empujando íntegramente a `clasp push`.
