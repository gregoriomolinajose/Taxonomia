# S30.4 Design: Evolución de ThemeManager

## 1. What & Why
**Problem:** La lógica del gestor visual (`ThemeManager`) actualmente está rota en dos mitades literales: Un trozo que hidrata variables asíncronamente alojado en `UI_ThemeManager.client.js`, y otro trozo engarzado como métodos huérfanos dentro de `<script>` en `Theme_Engine.html`. Esto previene el testing unitario y ensucia el Global Namespace.
**Value:** Al unificarlo bajo un patrón Singleton de Clases ES6 nativas (`class UI_ThemeManager`), posibilitamos pruebas V8 seguras y garantizamos de una vez por todas que la capa visual de Taxonomia opere bajo principios limpios de POO (O(1)).

## 2. Approach
Vamos a realizar un refactor unificador destructivo y reconstructivo:
1. **[MODIFY] `src/UI_ThemeManager.client.js`**: Convertiremos todo en un verdadero archivo de clases, conteniendo las propiedades `tokens`, `themeIcons`, los getters correspondientes y la lógica `_hydrate`. Al final del arcvhio exportaremos `window.ThemeManager = new UI_ThemeManager();`.
2. **[DELETE] `src/Theme_Engine.html`**: El archivo de inyección HTML ya no tendrá propósito de existencia; será purgado.
3. **[MODIFY] `src/Index.html`**: Se borrará la referencia al motor obsoleto `<?!= include('Theme_Engine'); ?>`.
4. **Resiliencia (Value preservation gate):** Toda la lógica de "ApexCharts dynamically resizing" alojada en el `setTheme` antiguo se conserva inalterada dentro del método de instancia.

## 3. Examples
### Uso Interno Actual VS Esperado:
**Como se llamaba antes**:
```javascript
// Disperso y propenso a ReferenceError
window.ThemeManager.initThemeManager();
window.ThemeManager.cycleTheme();
```
**Como seguirá funcionando**:
La interfaz permanecerá idéntica pero respaldada por un constructor formal:
```javascript
// La invocación y el estado se mantienen Singleton.
window.ThemeManager.initThemeManager();
```

### Constructor y Definición Interna (Sneak Peek)
```javascript
class UI_ThemeManager {
    constructor() {
        this.tokens = { ... };
        this.themeIcons = { 'light': 'moon-outline', 'dark': 'sunny-outline' };
        this.currentTheme = 'light';
    }
    
    get availableThemes() {
        return Object.keys(this.themeIcons);
    }

    initThemeManager() {
        const savedTheme = localStorage.getItem('app_theme') || 'light';
        this.setTheme(savedTheme);
        this.hydrateTokens();
    }
    
    // ... rest of the pure ES6 methods
}

window.ThemeManager = new UI_ThemeManager();
```

## 4. Acceptance Criteria
- **MUST**: Todo el código de `ThemeManager` debe habitar exclusivamente en `UI_ThemeManager.client.js`.
- **MUST**: Eliminar formalmente el template de backend `Theme_Engine.html` asegurando que todos sus métodos migren a la clase nueva de forma impecable.
- **MUST**: Los UI Integration Tests deben pasar fluidamente sin fallas por falta de contexto en window.
