# S30.3 Design: Cierre de Brecha de DOM Pasivo a Estado Local

## 1. What & Why
**Problem:** Actualmente subgrids y listas dinámicas esconden arreglos JSON anidados transformados en strings dentro de tags `<input type="hidden">` del form nativo. Esto requiere parseos frágiles (con riesgo de caídas asíncronas) y expone vectores temporales de side-effects visuales.
**Value:** Mantener las matrices de datos exclusivamente en la memoria `V8` subyacente (Caché asíncrono o State Closure) reduce en un O(n) el renderizado DOM, evita las fugas y robustece el Event Flow para testeo automático (nuestros tests en NodeJS / JSDOM son más felices sin parseos redundantes).

## 2. Approach
Sincronizaremos las colecciones JSON de `UI_SubgridBuilder.client.js`, `UI_Component_DynamicList.client.js` e híbridos, acoplándolas directamente al nodo contenedor transitorio (`activeForm._LocalState = {...}`).
`UI_FormSubmitter`, encargado del Submit final, primero escaneará las tags primitivas, y finalmente aplicará un `Object.assign(payload, activeForm._LocalState)` nativo para compilar el form antes del POST.

**Componentes afectados:**
- `src/UI_SubgridBuilder.client.js` [Modify] (Eliminar inyección subyacente oculta de `<input>`).
- `src/UI_Component_DynamicList.client.js` [Modify] (Eliminar inyección subyacente oculta de `<input>`).
- `src/UI_FormSubmitter.client.js` [Modify] (Cosechador del activeForm._LocalState).

## 3. Examples

### Form Submitter Colección Segura
```javascript
// En vez de:

    Object.assign(payload, activeForm._LocalState);
}
```

## 4. Acceptance Criteria
- **MUST** Subgrids guardan y leen datos nativamente al cerrar el modal primario sin fallos.
- **MUST** `UI_FormSubmitter` cosecha limpiamente los componentes mutados en memoria sin requerir inputs intermedios.
- **MUST NOT** Romper el renderizado E2E de Playwright o romper el pipeline del Builder.
