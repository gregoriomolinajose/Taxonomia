# Scope S30.5: AST RegExp CSS Minifier & Compiler Upgrade

## In Scope
- Sustituir todas las llamadas a Expresiones Regulares (`RegExp`) en `deploy.js` (como la extracción de `<script>` en el validador acorn, y el limpiador mágico del módulo QA) por librerías formales de AST para HTML (eg. `node-html-parser`, `parse5` o similar nativo compatible).
- Mantener compatibilidad absoluta estricta con el sistema `Clasp` y el entorno `.build` efímero.
- Garantizar que CSS sea minificado usando AST (`CleanCSS` o PostCSS) de forma robusta, validando sintaxis sin expresiones burdas.

## Out of Scope
- Migrar el framework base de compilación a `Webpack` o `Vite` (mantenemos el script nativo de node `deploy.js` por velocidad Zero-Config para Google Apps Script).
- Alterar código de negocio (archivos HTML o JS dentro de `src/`). El foco exclusivo es Infraestructura (DevSecOps) en `deploy.js` o `package.json`.

## Done Criteria
- `deploy.js` corre fluidamente sin chocar en regex malformados si un archivo HTML es inyectado.
- El build CSS/JS es seguro, extraído con validadores semánticos y no heurística de strings.
- `npx clasp push` aprueba los artefactos en local.
