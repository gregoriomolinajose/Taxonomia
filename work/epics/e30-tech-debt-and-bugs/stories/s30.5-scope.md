# Scope S30.5: AST RegExp CSS Minifier

## Boundaries

**In scope:**
- Modificar el archivo local NodeJS CLI `deploy.js` (y scripts auxiliares si los hubiera) específicamente para el tramo que empaqueta todo el framework CSS.
- Si decidimos depender de una nueva librería (`esbuild`, `parse5`, `clean-css`), instalarla dev-dependency en package.json.
- Integración perfecta con el formato requerido en Google Apps Script (HTML virtual containers `<style>...</style>`).

**Out of scope:**
- Modificar componentes Javascript o TS. Únicamente tocamos el pipeline CSS del despliegue.
- Rehacer la arquitectura de diseño CSS, S30.5 se centra EXCLUSIVAMENTE en cómo Node recolecta, minifica y concatena ese CSS durante el push a GCP.

## Done Criteria
1. El script `deploy.js` puede minificar el CSS compilado nativamente usando parsers formales y no Regex inestable.
2. Si un developer incluye reglas `@media`, seudoclases o cálculos en el CSS de Dashboard o Components, estas son empacadas fielmente sin pérdida de sintaxis.
3. El pipeline debe testarse localmente creando un *Dev Deploy* local para validar la compilación segura.
