# Scope S30.7: Estandarización Universal de Primary Key

## In Scope
- Auditar y asegurar que todas las entidades en `APP_SCHEMAS` (`Schema_Engine.gs`) posean un campo de identificador único validado o que defina explícitamente `primaryKey: 'id'`.
- Estandarizar la inferencia de PK en la ruta de las bases de datos (si la declaración está ausente, forzar fallback a `id`).
- Asegurar que herramientas de ORM (`DataAPI`, `Adapter_CloudDB` u otros interceptores) reconozcan uniformemente el mapeo de `primaryKey`.

## Out of Scope
- Migrar datos físicos existentes en las bases de datos de Google Sheets o Cloud DB para que se adapten a una nueva conformación de UUID (se mantiene el estándar subyacente).
- Modificar el sistema de control de accesos ABAC sobre esas Keys.

## Done Criteria
- `Schema_Engine.gs` es explícito en todas sus declaraciones acerca de qué columna rige como llave primaria.
- Tests (si aplican) o revisión de calidad afirman que la creación o actualización de entidades jamás cruza cables.
