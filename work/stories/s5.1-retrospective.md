# Story Retrospective: S5.1 Test Suite Coverage Gap Closure

**Story:** S5.1
**Epic:** E5 - Test Coverage & Quality
**Status:** Completed
**Tests Verification:** 89 Passed, 0 Failures

---

## 1. What was delivered

- **Anti-Mockery Compliance (QA-7)**: Eradicated `jest.mock(Adapter_Sheets)` from `Engine_DB.test.js`. Wrote a custom in-memory mock for `SpreadsheetApp` that allows integration tests to run the *real* `Adapter_Sheets.upsert` logic.
- **`Adapter_Sheets.list()` coverage**: 5 new tests. Verified default audit filtering, `includeAudit=true` behavior, empty sheet handling, tuple formatting, and serialization safety.
- **`Adapter_Sheets.upsert()` coverage**: 5 new tests. Verified create-path audit injection (4 fields), update-path immutability (`created_at` preserved), and PK missing errors.
- **`API_Universal` contract tests**: 4 new tests. Ensured all output payloads cleanly survive `JSON.parse(JSON.stringify())` to prevent silent `postMessage` drops caused by native Rhino `Date` objects.
- **Bug Fix**: Discovered and patched a runtime bug in `Adapter_Sheets.list()` where empty data ranges would return unfiltered headers, breaking the audit slimming rule.
- **Pipeline hardening**: Patched `deploy.js` with a robust 3-attempt `stdout` parsing backoff loop to guarantee `npx clasp push` success.

## 2. What went well

- The TDD approach directly exposed the `Adapter_Sheets.list()` early-exit bug. Without writing the empty-sheet test, that bug would have leaked to production.
- Converting `Engine_DB.test.js` to use an in-memory `SpreadsheetApp` mock was highly successful. The Facade now correctly delegates to the real adapter logic without altering the database.

## 3. What could improve

- The in-memory `SpreadsheetApp` mock is duplicated across `Adapter_Sheets.delete.test.js`, `Adapter_Sheets.upsert.test.js`, and `Engine_DB.test.js`. 
- **Action Item for future**: Extract the `buildInMemorySheet` and `buildInMemorySpreadsheet` stubs into `jest.setup.js` or a `__mocks__/` utility file.

## 4. Learnings & Process Adjustments

- **Verification loop**: Relying purely on exit codes for `clasp push` is dangerous because clasp sometimes returns `0` when skipping a push due to out-of-sync manifests. The fix in `deploy.js` reading standard output is a permanent cure.
- **Audit Rules check**: QA-7 (Anti-Mockery) is critical. Mocking adapters superficially hides integration flaws. The tests are now structurally representative of the live SPA.

## 5. Next Steps

- Proceed with S5.2 or epic closure depending on the roadmap. The core Backend abstractions are now fully green and covered.
