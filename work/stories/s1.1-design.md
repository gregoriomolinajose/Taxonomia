# Design S1.1: Agregar saludo de usuario

## 1. What & Why
- **Problem**: La aplicación actual es impersonal después de iniciar sesión.
- **Value**: Aumenta la percepción de bienvenida y personalización para el usuario.

## 2. Approach
- **Solution approach**: Modificar la interfaz principal para leer el nombre del usuario activo (posiblemente de las propiedades de script o sesión en Google Apps Script) y renderizar un saludo en el header.
- **Components affected**: 
  - `DataView_UI.html` (Modification)
  - `API_Universal.gs` o controlador correspondiente si es necesario inyectar el nombre (Modification)

## 3. Examples
- **Dado** el usuario "Juan" que abre la aplicación web.
- **Cuándo** se carga el HTML principal (`DataView_UI`).
- **Entonces** se muestra un elemento visual con el texto "Hola, Juan".

## 4. Acceptance Criteria
- **MUST**: El saludo (Ej. "Hola, Juan") sea visible en todas las vistas principales del DataView.
- **MUST**: Que no ocurran errores si la información del usuario no se puede recuperar (fallback a "Usuario").
