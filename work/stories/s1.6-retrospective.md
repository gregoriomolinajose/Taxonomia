# Retrospective S1.6: Schema Audit & Defensive Programming

## What went well
- Auditing the schemas globally enabled us to quickly identify that entities like Portafolio and Unidad_Negocio lacked `section` constraints.
- We successfully injected the Graceful Fallback defensive logic in FormEngine_UI, rendering custom warnings rather than causing White Screens.

## What could be improved
- We missed catching the legacy `field.step` dependency inside the Subgrid UI generator during the initial audit, which triggered a crash when manually validating in the browser. A more exhaustive grep search during static analysis of UI dependencies is required for future blueprint changes.

## Action Items
- Deprecate `field.step` completely from the codebase and ensure all future layout calculations rely exclusively on `field.section`.
- Stabilize the Jest mocks (like `getLastRow()`) to allow CI pipelines to catch Google Apps Script issues before manual testing.
