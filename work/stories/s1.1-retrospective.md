# Retrospective S1.1: Agregar saludo de usuario

**Dates:** 2026-03-23  
**Estimated:** 20m  
**Actual:** 15m

## What went well / what to improve
- **Well:** La modificación del DOM fue directa y no requirió cambios complejos en el backend. Utilizamos directamente `response.email` para deducir el nombre, cumpliendo el principio KISS.
- **Improve:** Asegurar que los tests del backend (actualmente fallando por issues en `Adapter_Sheets.js`) se estabilicen priorizando su reparación técnica en otra historia.

## Heutagogical Checkpoint
1. **What did you learn?**
   - Al usar Vanilla JS con inyecciones de HTML globales (como `window.OriginalDashboardHTML`), es clave actualizar esa variable *después* de hidratar datos dinámicos como el saludo. Si no, al regresar a la vista de inicio se sobreescribe con la plantilla estática.
2. **What would you change about the process?**
   - Correr una prueba de regresión (`npm test`) antes de desarrollar para documentar un "baseline" real de fallos previstos y no confundirlos con regresiones introducidas en la historia actual.
3. **Are there improvements for the framework?**
   - Detectar si el CLI `rai` no está instalado localmente para omitir sugerencias de ejecución que fallarán (`rai pattern`, `rai signal`).
4. **What are you more capable of now?**
   - Comprendo mejor el ciclo de hidratación inicial de Ionic mezclado con Google Apps Script (`AuthManager`).

## Actions & Patterns
- **Hydrated UI Caching (Technical):** Al modificar el DOM durante la autenticación para inyectar datos del usuario, re-capturar el `innerHTML` principal y guardarlo como Snapshot (caché local) permite que la navegación emulada por Javascript restaure la vista con el estado hidratado correcto.
