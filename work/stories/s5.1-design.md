# S5.1 — Test Suite Coverage Gap Closure

**Epic:** E5 — Quality & Maintainability
**Sprint:** 5.1
**Complejidad:** Moderada (3-4 componentes, lógica de mocking no trivial)
**Puntos:** 5

---

## Problema & Valor

### Problema
El test suite tiene **cobertura estructural pero no comportamental** en las capas más críticas del backend:
- `Adapter_Sheets.list()` — **0% cobertura** (incluyendo el nuevo flag `includeAudit`)
- `Engine_DB.orchestrateNestedSave()` — cubierto con `jest.mock(Adapter_Sheets)` — **viola QA-7 (Anti-Mockery Rule)**
- `API_Universal.gs` — sin tests de contrato de serialización (se expuso como riesgo en S4.3)
- **2 suites con skip activo** — suprimen fallos reales silenciosamente

### Valor
Cerrar estos gaps activa la **red de seguridad completa** antes de la expansión a nuevas entidades. Cada story futura podrá confiar en que el upsert, el list y la orquestación anidada ya están validados.

---

## Approach

Agregar 4 nuevos archivos de test y refactorizar `Engine_DB.test.js` para cumplir QA-7:

| Archivo | Tipo | Cobertura |
|---------|------|-----------|
| `Adapter_Sheets.list.test.js` | Unit | `list()` normal, `includeAudit=true`, hoja vacía |
| `Adapter_Sheets.upsert.test.js` | Unit | create path (4 audit fields), update path (preserva created_*) |
| `Engine_DB.nested.test.js` | Integration | `orchestrateNestedSave` con Adapter real en memoria |
| `API_Universal.contract.test.js` | Contract | Serialización JSON sin objetos Date nativos |

**Decisión clave (D1):** `Engine_DB.nested.test.js` usará un mock manual de `SpreadsheetApp` (in-memory array), **no** `jest.mock(Adapter_Sheets)`, para cumplir QA-7.

## Ejemplos Concretos

### AC1 — `list()` normal
```js
// Mock sheet con encabezados + 2 filas
const result = Adapter_Sheets.list('Dominio', { SPREADSHEET_ID_DB: 'test' });
expect(result.headers).toContain('id_dominio');
expect(result.headers).not.toContain('created_at'); // slimmed por defecto
expect(result.rows.length).toBe(2);
```

### AC1 — `list()` con includeAudit
```js
const result = Adapter_Sheets.list('Dominio', config, 'objects', true);
expect(result.headers).toContain('created_at');
expect(result.headers).toContain('updated_by');
```

### AC2 — Audit trail en create
```js
const result = Adapter_Sheets.upsert('Dominio', { id_dominio: 'DOMI-NEW', n0_es: 'Test' });
const row = capturedSetValues[0][0];
expect(row[createdAtIdx]).toMatch(/^\d{4}-\d{2}-\d{2}T/); // ISO timestamp
expect(row[createdByIdx]).toBe('system@localhost');
expect(row[updatedAtIdx]).toMatch(/^\d{4}-\d{2}-\d{2}T/);
```

### AC2 — created_at preservado en update
```js
// Primera llamada → create
Adapter_Sheets.upsert('Dominio', { id_dominio: 'DOMI-EXIST', n0_es: 'Original' });
// Segunda llamada → update
Adapter_Sheets.upsert('Dominio', { id_dominio: 'DOMI-EXIST', n0_es: 'Modified' });
const updatedRow = capturedSetValues[1][0];
expect(updatedRow[createdAtIdx]).toBe(ORIGINAL_CREATED_AT); // inmutable
expect(updatedRow[updatedAtIdx]).not.toBe(ORIGINAL_CREATED_AT); // sí se actualiza
```

### AC4 — Contrato de serialización API
```js
const response = API_Universal.getInitialPayload('Dominio');
// No debe lanzar en JSON round-trip (Date nativo causaría fallo)
expect(() => JSON.parse(JSON.stringify(response))).not.toThrow();
```

## Acceptance Criteria

**MUST:**
- AC1: `list()` cubierto (normal + includeAudit + vacío)
- AC2: `upsert()` cubierto (create path + update path, audit trail)
- AC3: `orchestrateNestedSave()` con mock manual (no `jest.mock`)
- AC4: 2+ contract tests de serialización en `API_Universal`
- AC5: 0 suites con skip activo

**SHOULD:**
- AC6: Total passing ≥ 90 tests

**MUST NOT:**
- Usar `jest.mock(Adapter_Sheets)` en tests de `Engine_DB` (viola QA-7)
- Mockear la lógica de Soft Delete — debe usar el `remove()` real del Adapter
