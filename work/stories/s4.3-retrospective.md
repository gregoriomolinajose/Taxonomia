# Story S4.3 Retrospective

## Summary
- **Story IDs:** S4.3 (Architecture fixes for Motor Jerarquía / UI Coupling)
- **Estimated Time:** 60 minutes
- **Actual Time:** 90 minutes

## What went well / What to improve
- **Well:** The pure JS abstraction `Math_Engine.js` successfully resolved the semantic duplication (Beck's Rule H9) between the GUI tests and frontend injection scripts.
- **Improve:** Schema payload tests silently drafted when S5.5 Temporal Graph refactored `id_dominio_padre` to `id_nodo_padre`. This wasn't caught until we ran the whole test suite during this story implementation phase.

## Heutagogical Checkpoint
1. **What did you learn?**
   Caching logic tightly coupled within UI events (like `submitBtn.addEventListener`) causes severe maintenance issues. Pure functions should be strictly separated from DOM interactions.
2. **What would you change about the process?**
   When making architectural modifications in `Schema_Engine.gs` (e.g., changing field names or `triggers_refresh_of` relations), we must retroactively ensure all downstream test payloads are synchronized.
3. **Are there improvements for the framework?**
   Tests should load schema definitions dynamically rather than statically defining string keys to catch schema drift immediately before tests begin to execute.
4. **What are you more capable of now?**
   Fully applying the Golden Pattern of extracting environment-agnostic logic functions (`Math_Engine`) independent from the GAS UI DOM, ensuring Jest testability parity with Google Apps Script.

## Applied Improvements
- `Math_Engine` logic completely isolated for `buildOrdenPath` universal derivation.
- Zero-latency `_patchFrontendCache` method was cleanly abstracted in `FormEngine_UI.html` to separate DOM mutation from data injection.
- Synchronized `Dominio` component test payloads to match the modern `id_nodo_padre` adjacency list schema state.

## Patterns Added/Reinforced
- Added new architecture pattern governing the isolation of logic out of UI events into pure JavaScript engines.
