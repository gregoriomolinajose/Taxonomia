## Tasks

### T1 — Refactor `Engine_DB.test.js` to remove `jest.mock(Adapter_Sheets)` [HIGH RISK]

**TDD Cycle:** RED → GREEN → REFACTOR
**Size:** M | **Depends on:** none (do first — highest risk)

**Problem:** `Engine_DB.test.js:L14` uses `jest.mock('../src/Adapter_Sheets', ...)` to mock the `upsert` method. This violates `rules_qa.md` §7 (Anti-Mockery), which prohibits mocking `Adapter_Sheets` in `Engine_DB` integration tests.

**What to do:**
- Replace `jest.mock('../src/Adapter_Sheets')` with a manual in-memory SpreadsheetApp mock (same pattern as `Adapter_Sheets.delete.test.js`)
- The test must call the **real** `Adapter_Sheets.upsert` against a fake in-memory sheet
- Keep the Dual-Write, Idempotency, and Resiliency test intentions — only change the mocking strategy

**Files:**
- MODIFY `__tests__/Engine_DB.test.js`

**Verification:**
```bash
npx jest Engine_DB.test.js --no-coverage --verbose
```
All 3 existing tests must still pass (green) with real adapter.
