# User Story: S1.3 Blueprint V2 Refactoring & Persona Provisioning

## Connextra
**Como** Arquitecto de Plataforma
**Quiero** que todas las entidades operen bajo el esquema Blueprint V2 y que la entidad `Persona` esté completamente provisionada
**Para** garantizar que la renderización de formularios sea consistente, a prueba de fallos y que los lookups dependientes (como Scrum Master en Equipos) funcionen sin errores topológicos.

## Gherkin Acceptance Criteria

**Scenario:** Parsing a V2 Schema Entity
**Given** the Schema Engine configuration for `Portafolio`
**When** the frontend or backend requests the schema
**Then** it should return an object containing `primaryKey`, `titleField`, and an array of `fields`
**And** it should not use the deprecated dictionary-style field mapping.

**Scenario:** Persona Entity Lookup Integrity
**Given** the `Persona` entity schema is defined in V2 format
**When** a user tries to create an `Equipo`
**Then** the dropdown for `Scrum Master` and `Product Owner` should successfully fetch from `getPersonasOptions()` without raising an undefined object error.
