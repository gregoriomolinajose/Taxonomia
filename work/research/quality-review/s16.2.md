### Critical (fix before merge)

### Recommended (improve code quality)
1. **Semantic Bug - Exception Swallowing (JS Idiom):**
   - *File:* `Controller_Action.gs (L176)` en `bulkInsert(entityName, recordsArray)`
   - *Why:* El bloque `const ss = SpreadsheetApp.openById(config.SPREADSHEET_ID_DB); if (!ss) return { status: 'error'};` engaña la lectura. Si el `SPREADSHEET_ID_DB` está vacío o es inválido, el método `openById` no devuelve `null`, sino que dispara una excepción nativa in-lugar que detiene la ejecución, siendo atajada por el Catch global del router sin el control semántico esperado.
   - *Fix:* Envolver `openById` en un `try/catch` individual si quieres devolver un status asíncrono controlado, o validar `if (!config.SPREADSHEET_ID_DB) throw new Error(...)` antes de tocar la API.

2. **Semantic Safety - Truthiness Traps en Filtrado (JS Idiom):**
   - *File:* `Controller_Lookups.gs (L180)` en `getGenericOptions(...)`
   - *Why:* Al usar el filtro estricto `.filter(opt => opt.value && opt.label)`, Javascript desecha cualquier valor "Falsy". Dado que los IDs actualmente son Alfanuméricos (`UNID-9X`), funciona bien. Pero si en el futuro se asocia una tabla catálogo cuyo Primary Key sea numérico e incluya `0` (ej: `{value: 0, label: "Indefinido"}`), la validación `opt.value &&` evaluará a Falso y silenciará la opción, provocando pérdida de datos M:N fantasma.
   - *Fix:* Usar comprobación explícita de nulos como dicta Blueprint: `.filter(opt => opt.value !== undefined && opt.value !== null && opt.label)`
